# Day-one setup — flash-study agent crew

## 1. Install
Unzip this at the root of your repo (e.g. `bpr_lab/`). It creates:
```
CLAUDE.md
SETUP.md
.claude/
  settings.json
  agents/  architect.md  implementer.md  quiz-evaluator.md  reviewer.md  debugger.md
  scripts/ check-no-secrets.sh
```
Also drop your `docs/flash-study-spec.md` into the repo (CLAUDE.md references it).

## 2. Prerequisites
- Claude Code v2.1.32+ (agent teams). Check: `claude --version`
- `jq` installed (for the commit hook). Without it the hook just no-ops.

## 3. Load it
Restart Claude Code in the repo (subagents load at session start). The 5 agents now appear in `/agents` and via `@`-mention. Agent teams are enabled by `settings.json`.

Optional: set a default model for teammates in `/config` → "Default teammate model" (e.g. Sonnet for cheap parallel work; "leader's model" to follow Opus 4.8).

## 4. Everyday use (subagents — your default)
Plan, then implement, then verify:
```
Use the architect agent to plan the callLLM(provider) abstraction for Anthropic/OpenAI/Google.
```
(approve the plan, then:)
```
Use the implementer agent to build that plan.
@quiz-evaluator check the quizzes generated by the new genQuiz against the source.
@reviewer review the diff, focus on BYOK key handling.
```
On a bug:
```
Use the debugger agent: a Gemini response fails JSON.parse intermittently. Find the root cause.
```

## 5. When to spin up a TEAM (only these 3 moments)
Teams cost a lot more tokens — reserve them for genuinely parallel, independent work.

A) Quiz-quality as a scientific debate (your #1 risk):
```
Create an agent team to harden quiz generation. Spawn 3 teammates using the
quiz-evaluator agent type. Give each the SAME source text, have each generate
and critique 4-choice questions, then have them challenge each other's items
for hallucination and weak distractors. Converge on the single best change to
the genQuiz prompt and report it.
```

B) Parallel Phase 2/3 build (each owns different files — no overlap):
```
Create an agent team to build Phase 2. Three teammates:
- one on the PDF intake pipeline (pdf.js text extraction)
- one on the BYOK settings screen + callLLM provider branching
- one on the IndexedDB deck library
Each owns its own files. Require plan approval before any of them edit code.
```

C) Hard bug with no obvious cause:
```
Spawn a small agent team to investigate why <bug>. Each teammate takes a
different hypothesis and tries to disprove the others. Update a findings note
with the consensus.
```
Always `Clean up the team` when done. Split-pane view needs tmux or iTerm2; the default in-process mode works anywhere (Shift+Down cycles teammates).

## 6. Guardrails baked in
- `check-no-secrets.sh` blocks `git commit` if staged changes look like they contain an API key (Anthropic/OpenAI/Google patterns). Tighten as needed.
- `reviewer` and `quiz-evaluator` are read-only and have project memory, so they accumulate your recurring issues over time.
- `architect` runs in plan mode (read-only) so design happens before code.

## 7. Cost note
Subagents return a summary to the main session (cheaper). Each teammate is a full separate Claude instance (more tokens, scales with team size). Start with 3 teammates; three focused beat five scattered.
