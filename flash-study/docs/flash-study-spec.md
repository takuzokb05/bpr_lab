# フラッシュ学習アプリ 仕様書 (v0.1)

資格学習向け。文章を文節単位でフラッシュ表示（RSVP）して集中を維持し、最後に4択の「まとめ問題」で定着を確認するツール。NotebookLM的に、参考書から本文とクイズをAIで生成できることを目指す。

-----

## 1. 設計の背骨：デッキ = 1つのJSON

すべてのデータをこの1つの形式に統一する。これにより「API自動生成」「職場コピペ取込」「保存・再利用」がすべて同じ箱で処理できる。

```json
{
  "id": "uuid",
  "title": "固定資産税の基礎",
  "source": "原文テキスト（任意・PDF抽出含む）",
  "flashMode": "original | ai",
  "flashText": "フラッシュ表示する本文（原文 or AI再構成後）",
  "quiz": [
    {
      "q": "固定資産税の賦課期日はいつ？",
      "o": ["毎年4月1日", "毎年1月1日", "取得した日"],
      "a": 1,
      "e": "1月1日時点の所有者に課税される。"
    }
  ],
  "quizStatus": "ready | unset",
  "createdAt": "ISO8601"
}
```

- **API生成** → このJSONを出力
- **職場コピペ** → Copilotにこの形式で吐かせて貼り付け
- **保存** → このJSONを溜める（= デッキ集）
- アーティファクト内は「書き出し/読み込み（ファイル）」、PWA化で「IndexedDB保存」に昇格

-----

## 2. モードと状態

### 本文（フラッシュ対象）

- **原文モード**：入力テキスト/PDF抽出をそのまま流す（API不要）
- **AI再構成モード**：参考書本文をフラッシュ向けに要約・短文化して流す（API必要）
- 両者は**切替**で選べる

### クイズのセット方法

1. **API自動生成**（個人用）：本文からAIが4択を生成。問題数は本文量に応じてAIが自動決定
1. **JSON/テキスト貼り付け取込**（職場用）：Copilot等に作らせたものを貼る
1. **未セット**：クイズ無しでもフラッシュ学習は成立する状態。「あとで生成/貼付」を選べる

### 出題形式

- **4択（多肢選択）のみ**。即時フィードバック＋解説。
- 結果画面で「間違えた問題の論点だけ再フラッシュ」ループ。

-----

## 3. AI連携仕様

### 提供元（PWAでのBYOK）

- **Anthropic (Claude) / OpenAI (GPT) / Google (Gemini)** の3社対応
- キーは**端末保存**、ブラウザから各社APIを直接呼ぶ（各社のブラウザ利用許可フラグを使用）
- ⚠️ **正直メモ**：BYOK方式ではキーがクライアントに見える形になる。個人ローカル利用前提とし、共有・公開はしない。

### 開発・検証時（アーティファクト内）

- キー不要でClaude内API（`claude-sonnet-4`）が使える。Phase 1はこれで動かす。

### AIの2つの仕事（出力は必ずデッキJSON）

1. **本文再構成**：原文 → フラッシュ向けの短い文章列
1. **クイズ生成**：本文 → 4択配列（問題数は本文量で自動）

### PDF処理

- 個人（API有）：PDFをAPIに直接渡す or `pdf.js`でテキスト抽出
- 職場（API無）：`pdf.js`でローカル抽出 → 原文モードでフラッシュ（サーバ不要）

## 5. 職場モード用：Copilotに貼るプロンプト雛形

```
以下の本文から4択問題を作り、次のJSON配列だけを出力して（前後の説明文は不要）:
[{"q":"問題文","o":["選択肢1","選択肢2","選択肢3"],"a":正解の番号(0始まり),"e":"解説"}]
問題数は本文量に応じて適切に。
本文:
"""
（ここに本文）
"""
```

これで出た配列をアプリの「クイズ取込」に貼れば完成。API不要。

## 6. 段階リリース

- **Phase 1（今・アーティファクト）**：原文/AI再構成の切替 + Claude内APIでクイズ自動生成 + デッキJSONの書出/読込 + 4択 + 既存のフラッシュUI
- **Phase 2**：PDF取込（pdf.js）、職場用JSONコピペ取込、設定画面、デッキ詳細/ライブラリ
- **Phase 3**：PWA書き出し、3社BYOK、IndexedDBで本物のデッキ集

## 7. 既知の制約（正直メモ）

- アーティファクト内では `localStorage` 等が使えない → 保存はファイル書出/読込で代替、PWA化でIndexedDBに昇格
- BYOKはキーがクライアント可視 → 個人ローカル限定
- フラッシュ（速読）は速くしすぎると定着が落ちる → **本命はまとめ問題（能動的想起）**。フラッシュは入口、定着は問題、の役割分担を崩さない

-----

## 8. API設定スキーム（BYOK）※PWA/Claude Code側の機能

> アーティファクト内はキー無しの代理呼び出し専用のため、キー設定UIは持たない（持てない）。下記は**標準版（PWA）でのみ**実装する。

### 設定画面の項目

- **プロバイダ選択**：Anthropic / OpenAI / Google（ラジオ or ドロップダウン）
- **APIキー入力**：プロバイダごとに保持（マスク表示、貼付）
- **モデル選択**：プロバイダごとの既定モデル（実装時に最新名を確認）
- **接続テスト**ボタン：短い疎通リクエストで疎通＋キー有効性を確認
- **保存先**：端末（PWA=`localStorage`/IndexedDB）。サーバには送らない

### 各社のブラウザ直叩き作法（要点）

|提供元      |エンドポイント                                            |キーの渡し方                               |備考                                                                    |
|---------|---------------------------------------------------|-------------------------------------|----------------------------------------------------------------------|
|Anthropic|`POST /v1/messages`                                |`x-api-key` ヘッダ + `anthropic-version`|ブラウザからはCORS回避フラグ `anthropic-dangerous-direct-browser-access: true` が必要|
|OpenAI   |`POST /v1/chat/completions`                        |`Authorization: Bearer <key>`        |`response_format:{type:"json_object"}` でJSON強制が可能                     |
|Google   |`POST .../models/<model>:generateContent?key=<key>`|クエリ`key`                             |`generationConfig.responseMimeType:"application/json"` でJSON強制        |

### 抽象化方針

- 呼び出しは `callLLM(prompt, opts)` の1関数に集約し、内部でプロバイダ分岐。
- 上位の `genQuiz()` / `reconstruct()` / `categorize()` はプロバイダ非依存。
- レスポンスは各社差を吸収して「本文テキスト」に正規化してから既存パーサ（`parseQuizLoose`）へ。

### セキュリティ正直メモ

- BYOKは**キーがクライアントに見える**。個人ローカル端末限定、共有・公開・配布はしない。
- 公開配布したいなら、将来的にキーを預かる薄いプロキシ（自前VPS等）を挟む構成に変更が必要。

## 9. AIによるデッキ自動分類（Phase 2以降）

デッキが増えると「試験用」「通知文」等が混在して探しづらい問題への対応。

- **作成時の自動カテゴリ提案**：本文をAIが見て候補カテゴリを提示（採用/手動修正可）。
- **既存デッキの再分類**：溜まったデッキ一覧をAIに渡し、再グルーピングや統合候補を提案。
- いずれもAPI依存のため、`categorize()` として §8 の抽象化に乗せる。
- フォールバック：APIが無い/職場では**手動カテゴリ**（Phase 1で実装済み）。

> Phase 1ではまず手動カテゴリ＋カテゴリ別グループ表示までを実装済み。AI自動分類は土台の上に後付けする。
