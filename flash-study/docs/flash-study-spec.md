## 1. プロダクト概要

資格試験向けの学習ツール。学習素材（テキスト or PDF）を取り込み、

1. **フラッシュ読み（RSVP）** — 文章をフレーズ単位で一語ずつ高速提示し、集中を維持して通読する。
2. **要約クイズ（4択）** — 通読後にアクティブリコールで定着を確認する。

NotebookLM 的に「素材を貼る → AI がフラッシュ用テキストとクイズを生成」という体験を目指す。

**プロダクト原則:** フラッシュ読みは入口、クイズ（アクティブリコール）こそが定着の本体。
速度のためにクイズ品質を犠牲にしない。最大リスクは「もっともらしいが間違った AI クイズ」。

## 2. データの背骨：1デッキ = 1 JSON

すべてのフロー（AI 自動生成 / ワークモードの JSON 貼り付けインポート / 保存・再利用）は
この単一シェイプを生成・消費する。**第二の真実の源（second source of truth）を作らない。**

```json
{
  "id": "",
  "title": "",
  "source": "",
  "flashMode": "original|ai",
  "flashText": "",
  "quiz": [
    { "q": "", "o": ["", "", "", ""], "a": 0, "e": "" }
  ],
  "quizStatus": "ready|unset",
  "category": ""
}
```

| フィールド | 意味 |
|---|---|
| `id` | デッキ識別子（生成時に付与） |
| `title` | デッキ名 |
| `source` | 元素材の全文（クイズ採点の根拠＝グラウンドトゥルース） |
| `flashMode` | `original`=素材そのまま / `ai`=AI が再構成したフラッシュ用テキスト |
| `flashText` | フラッシュ表示に使う本文 |
| `quiz[]` | 4択問題の配列。`q`=設問, `o`=4選択肢, `a`=正解インデックス(0-3), `e`=解説 |
| `quizStatus` | `ready`=クイズあり / `unset`=未生成 |
| `category` | 分類（手動 or AI 自動分類） |

- ブラウザ内セッション：メモリ保持 ＋ ファイル export/import。
- PWA：同じシェイプを IndexedDB に永続化。

## 3. フラッシュ表示（RSVP）

- `flashText` をフレーズ単位のチャンクに分割し、一定間隔で1チャンクずつ提示。
- 速度はユーザー調整可（chunk/分 もしくは ms/chunk）。再生 / 一時停止 / 前後移動。
- **日本語には単語境界がない。** チャンカーは助詞や句読点だけの極小フラグメントを孤立させない
  こと（孤立は隠さずチャンカー側で直す）。

## 4. クイズ（4択 / アクティブリコール）

- 1問 = 設問1・選択肢4・正解1・解説1。
- 採点・出題は `deck.source`（元素材）を根拠とし、外部知識に依存しない。
- 品質ゲート：AI 生成クイズは `quiz-evaluator` エージェントで
  グラウンディング / 正解唯一性 / ディストラクタ品質 / 難易度・網羅 / 形式 を検査する。

## 5. LLM アクセス

- すべてのモデル呼び出しは **1つの関数 `callLLM(prompt, opts)`** を通し、プロバイダで分岐。
- 上位の `genQuiz()` / `reconstruct()` / `categorize()` はプロバイダ非依存で `callLLM` を呼ぶ。
- プロバイダ（BYOK・standalone/PWA のみ）：
  - **Anthropic** — `x-api-key` ＋ version ヘッダ ＋ `anthropic-dangerous-direct-browser-access` ヘッダ
  - **OpenAI** — `Authorization: Bearer`
  - **Google Gemini** — key をクエリパラメータで
  - 可能なプロバイダでは JSON 出力を強制する。
- **堅牢なパース:** レスポンスが途中で切れても、完結した JSON オブジェクトをサルベージする。

## 6. セキュリティ（譲れない）

- API キーは BYOK。実行時にブラウザストレージにのみ存在する。
- キーをログ出力・print・ハードコード・コミットしない。
  `.claude/scripts/check-no-secrets.sh` のコミットフックは最後の砦であって免罪符ではない。

## 7. フェーズ

- **Phase 1（完了・試作）:** フラッシュ UI、4択クイズフロー、デッキ JSON export/import、手動カテゴリ。
  試作は単一 HTML ファイル（`prototype/index.html`）。AI 呼び出しは無し。
- **Phase 2:** PDF 取り込み（pdf.js）、ワークモードの JSON 貼り付けインポート、設定画面、AI 自動分類。
- **Phase 3:** PWA 化、`callLLM` 経由の 3プロバイダ BYOK、IndexedDB デッキライブラリ。
