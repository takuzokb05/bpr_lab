## 要約
本番チームからの18のClaude Codeベストプラクティスパターン。計画ワークフローの重要性：「実装なし・計画だけ」のdraftを作り、エディタで注釈→「全ノートに対応・実装なし」で送り返す、すべての決定が解決するまで繰り返す。Claude Codeは明確なコンテキスト・規律ある計画・事前の構造化があれば最高の成果が出る。公式ドキュメントのbest-practicesとも整合する内容で、特にMCPによる外部ツール統合（issueトラッカー・DB・Figma・監視データ）の活用パターンを産業別に整理。2026年の重心はplan mode・並列探索・永続プロジェクトメモリ・構造化実行・長期セッション継続へ。計画→実行の明確な分離と反復精緻化がキーパターン。Claude Code活用初心者から上級者まで参照できるパターン集として有用。
