## 要約
KnowledgeWork社がClaude Codeを実務導入して月間400件超のPull Requestを生産した国内一次情報事例（Zenn）。開発チームが実際に計測した定量データを公開。主な知見：①エンジニア1人あたりの並列ブランチ数が従来の3〜5倍に増加、②CLAUDE.mdへのプロジェクト固有ルール集約でコンテキスト再設定コストが激減、③Subagentsによる並列worktree実行（4〜8個）でPR作成〜レビュー準備時間が約60%短縮。導入上の注意点としてモデルコストの急増とセキュリティレビューの強化が必要という実体験も記録。同社の実運用プロンプトテンプレートの一部も公開されており、同種の日本語コードベースを持つチームに直接参考になる高価値一次情報。
