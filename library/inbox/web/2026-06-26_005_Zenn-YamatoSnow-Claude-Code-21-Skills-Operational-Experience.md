## 要約
Zennユーザーyamato_snow氏による21個のSkillを実際に運用した経験から得た設計知見の公開。SKILL.mdの書き方・Progressive Disclosure原則（frontmatterは常駐/本文は起動時/スクリプトは必要時）・descriptionの重要性（90%の成否を左右する）を詳述。実際に作成・運用した21個のスキル一覧と各スキルの設計判断を公開。「完璧なスキルを最初から作ろうとしない」という運用哲学（3-5行で開始→使いながら育てる）と、失敗パターン（descriptionが曖昧→自動起動しない、スクリプト化すべき処理をSKILL.mdに詰め込む等）の具体例を含む実践的ガイド。
