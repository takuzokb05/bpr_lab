## 要約
DEV Community に掲載された CLAUDE.md ベストプラクティス完全ガイド（2026年版）。Anthropic 公式の推奨事項を踏まえた「200行以下に保つ」原則を中心に、10の必須セクション（コマンド一覧・プロジェクト概要・アーキテクチャ・コーディング規約・禁止事項・環境変数・テスト手順・デプロイ手順・既知の問題・用語集）の具体的な書き方を示す。「clean code を書くこと」のような曖昧ルールを避け、テスト可能な具体的指示（例: "Server components by default; add 'use client' only when truly needed"）を書くべきとする。ルールには「なぜ」の説明を必ず付けることで、コンテキストが変わった場面でも Claude が適切に判断できると解説。フォーマッターが強制するスタイルルールの重複記述は不要と明言。
