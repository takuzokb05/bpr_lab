## 要約
Build to LaunchによるClaude Codeプラグイン11本の実際テスト結果レビュー（2026年）。
実用評価で「本当に使えた」4本: ①Context7（ライブラリドキュメント精確参照・バージョン固定）、②rtk（CLIツール出力圧縮・トークン節約）、③Playwright Plugin（ブラウザ自動化テスト統合）、④GitHub Plugin（PR・Issue・Actions直接アクセス）。
除外した7本の理由も公開：機能重複・コンテキスト消費が大きい・動作が不安定・既存ワークフローと競合。
キーインサイト: 「プラグインは多ければ良いわけではなく、コンテキスト汚染のリスクがある。厳選した少数が最善」。
Context7はライブラリドキュメントをバージョン固定で提供し、Claudeが古いAPIを参照する問題を解決する点で特に評価が高い。
実際の使用コスト評価（コンテキスト消費量）も掲載されており、実践的な導入判断に役立つ記事。
