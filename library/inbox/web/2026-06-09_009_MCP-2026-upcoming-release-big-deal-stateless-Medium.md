## 要約
MCP 2026-07-28 Release Candidateの重要性を解説したMedium記事。プロトコルのステートレス化が最大の変更点：以前はスティッキーセッション・共有セッションストア・ゲートウェイでのディープパケットインスペクションが必要だったが、新仕様ではラウンドロビンロードバランサーが使用可能に。追加変更：Streamable HTTPにMcp-Method/Mcp-Nameヘッダーを必須化（ボディ検査なしでルーティング可能）、ttlMsとcacheScopeでキャッシュ制御、W3C Trace Contextによる分散トレース標準化。SDK Tier 1は10週間以内に対応義務。拡張フレームワーク・Tasks・MCP Apps・OAuth強化も含む。
