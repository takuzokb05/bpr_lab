## 要約
v2.1.172（2026年6月10日）で追加されたClaude Codeの5段階ネストサブエージェント機能の詳細解説。仕組み: 各サブエージェントはAgent toolを呼び出し自身の子エージェントを生成可能、上限5段階のハードキャップ。コンテキスト: 各レベルが独自のシステムプロンプト+モデルを持ち、親は子の要約のみ受け取る（中間レベルのトークンは消費されるが消える）。トークン計算: 1レベルあたり約200Kコンテキスト×7倍ブランチ相当。3つのアンチパターン: (1) 暴走ネスト（深すぎると指数的コスト増）、(2) Opusを全段階で使用（コスト最適化：末端ほど安価モデルへ）、(3) 型なしAgentのallowlist。実用推奨: 2〜3段階を上限として設計。ユースケース: 大規模リファクタリングの並列化・マルチリポジトリ横断作業・CI/CDパイプライン統合。
