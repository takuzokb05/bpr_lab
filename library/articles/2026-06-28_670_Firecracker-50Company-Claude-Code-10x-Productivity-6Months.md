## 投稿内容
北海道札幌市の50名規模・ウォーターフォール型システム開発会社がClaude Codeを採用した6ヶ月の記録（2025年9月〜2026年3月）。

**導入の段階的アプローチ:**
1. 現状評価フェーズ（2025年9月）: 業務プロセスの棚卸しと課題特定
2. 全社セミナー（2025年10月）: 全従業員を対象にした理解促進
3. 実案件PoC（2025年10〜12月）: 実際の顧客プロジェクトでの試験的適用
4. プロセス標準化（2026年1月〜）: AI中心のワークフローを全社標準に

**キーとなったワークフロー転換:**
PDFや既存資料をMarkdownに変換し、Claude Codeに設計書ドラフトを生成させ、人間がレビューするという流れを確立。重要な文化的転換点: 「AIを使わないと仕事が進まない」という強制力が全社採用を推進した。

**定量的成果:**
- 設計文書作成: 10倍の生産性向上
- 全チームで標準化されたAI駆動ワークフローを採用
- 社内AI推進チームを設立し持続的展開を実現

**成功の重要要因:**
単にツールを配布するのではなく、AIを中心にビジネスプロセス全体を再設計したこと。週次アライメントミーティングで継続的な調整と推進を実施。

ウォーターフォール型の伝統的な開発会社でも、適切な段階的アプローチとプロセス再設計により、Claude Codeを全社的に導入できることを示した具体的な事例。

## 要約
北海道・50名ウォーターフォール型システム開発会社のClaude Code6ヶ月導入事例（2025/9〜2026/3）。段階的アプローチ（評価→全社セミナー→PoC→標準化）でAI駆動開発を実現。PDF→Markdown変換＋設計書ドラフト生成のワークフローで設計文書作成10倍生産性向上。「使わないと仕事が進まない」強制力が採用を促進。成功要因はツール配布ではなくビジネスプロセス再設計と週次アライメント。社内推進チームを設立し持続的展開を確保。
