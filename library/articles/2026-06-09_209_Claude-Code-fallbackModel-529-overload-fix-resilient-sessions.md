## 投稿内容
Claude Code v2.1.166（2026年6月6日）で追加されたfallbackModel設定の詳細ガイド。

設定方法：
```json
{
  "model": "claude-sonnet-4-6-20260620",
  "fallbackModel": ["claude-haiku-4-5-20251001"]
}
```
最大3つのバックアップモデルを優先順に設定可能。

動作仕様：
- トリガー条件：過負荷（529）・予期しない非リトライ可能エラー
- 即座にエラー表面化（フォールバックしない）：認証エラー・レートリミット・リクエストサイズ・転送エラー
- バックグラウンドセッション（/bg、--detach）もフォールバック設定を継承（サイレント失敗なし）
- --fallback-model CLIフラグ：interactiveセッションでは無効、printモード(-p)・バックグラウンドで有効

v2.1.169での改善：primary model Not Found（モデルが見つからない）でもセッションをフォールバックモデルへ切り替えるよう変更。APIが予期しない非リトライ可能エラーを返した場合、フォールバックモデルで1回リトライ。

実用効果：長時間エージェントセッションでの中断を防止。Anthropic 529プラットフォームインシデント（5月・6月に複数回発生）でのセッション損失を回避。

## 要約
v2.1.166+で使えるfallbackModel設定のベストプラクティスガイド。529過負荷エラー発生時の自動フォールバックで長時間セッションの安定性が大幅向上。バックグラウンドセッション継承サポートで自動化パイプラインの耐障害性が強化された。設定値と動作条件を明確に解説した実践的なリファレンス。
