## 投稿内容
/ultrareviewはClaude Code（2026年4月パブリックリサーチプレビュー）のクラウド実行型コードレビューコマンド。

**アーキテクチャ**
- 標準: 5エージェント並列（大規模PR: 最大20エージェント）
- 各エージェントが異なる観点で独立探索: 競合状態・SQLインジェクション・エラーハンドリング・メモリリーク・入力バリデーション
- 発見されたバグは全て別エージェントが独立再現・検証してから報告

**/review との違い**
| 機能 | /review | /ultrareview |
|------|---------|--------------|
| 実行場所 | ローカル | クラウド |
| エージェント数 | 1 | 5-20 |
| バグ再現検証 | なし | あり |
| スタイル提案 | あり | なし |
| 対象 | 通常の変更 | 重要なPR・リリース前 |

**利用方法**
```
/ultrareview          # カレントブランチのPRを対象
/ultrareview --pr 42  # 特定PRを指定
```

Pro・Max・Team・Enterprise プランで利用可能。

## 要約
/ultrareviewコマンド（2026年4月パブリックリサーチプレビュー）の詳細解説。標準5エージェント・最大20エージェントが並列でPRを探索するクラウドベースのコードレビューシステム。
全ての報告バグは独立した別エージェントが再現確認を実施するため、誤検知が少なく実際のバグのみを報告する。
各エージェントが異なる攻撃ベクトル（競合状態・SQLインジェクション・エラーハンドリング等）に特化した並列探索。
通常の/reviewとの違い：/ultrareviewはクラウド実行・多段階検証・スタイル提案なし・バグのみに特化。
Pro以上のプランで利用可能。重要なリリース前・複雑なPR・セキュリティクリティカルなコード変更に特に有用。
本プロジェクトのFX自動取引コード（sandbox/FX自動取引/）のリリース前チェックに活用できる。
