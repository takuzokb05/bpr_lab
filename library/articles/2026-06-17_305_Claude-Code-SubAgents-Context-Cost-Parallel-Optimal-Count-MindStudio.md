## 要約
Claude Codeサブエージェントをコンテキスト・コスト・並列実行の観点から定量分析した解説記事。設計原則：各サブエージェントは独立コンテキストウィンドウを持ちメインセッションのトークン消費を抑制。最適並列数は3〜5台（超過するとサマリーマージのオーバーヘッドが増大）、Dynamic Workflows導入後は数十〜数百台に拡張可能。権限分離パターン：リサーチ型サブエージェントはファイル書き込み禁止、ライター型はProduction API呼び出し禁止。コスト試算：5サブエージェント並列でメインセッション比約35%のトークン削減例付き。
