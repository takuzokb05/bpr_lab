3. 今週の重要トピック

Claude Code

Artifacts — セッション成果物のライブHTML共有

6月18日ベータ開始。Claude Codeが生成する成果物をself-contained HTMLページとして共有可能に。セッション進行に伴いリアルタイム更新される。PRウォークスルー、テスト結果サマリー、インシデントダッシュボード等に最適。Team plan以上で利用可能。週次ダイジェスト自体もArtifacts的な出力形態と言える。

5段階エージェントネスト + Agent Teams

Agentツールが最大5段階のネストに対応し、ルートエージェント→ドメイン分析→ファイル探索→実装→テストの自律パイプラインが構築可能になった。さらにAgent Teams機能（環境変数 CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1）により、アーキテクト・実装者・テスター・ドキュメント作成者の4エージェント並列協調が実現。階層ごとにトークンコストが複利的に増加する点には注意が必要。

Routines + Remote Control = 夜間自律開発

Routinesの3トリガー（Scheduled/API/GitHub）により、ラップトップを閉じてもAnthropicクラウド上で継続実行される。Remote Controlでスマートフォンから進捗確認・承認が可能。定期バックテスト、日次レポート生成、PR自動レビューなどの「定型的だが重要な」タスクを完全自動化できる。

/rewind + fallbackModel — 安全性と信頼性の強化

/rewindコマンド（6月24日追加）で会話コンテキストとコード変更の両方を任意チェックポイントへ復元可能。従来の/clearが一方向リセットだったのに対し、双方向移動による安全ネットを提供。fallbackModelパラメータにより主モデル障害時の自動フォールバックが実現し、Fable 5停止のような事態への備えとなる。

JetBrains統合 + Skill設計知見の成熟

GitHub Copilot for JetBrainsでClaude Agentがパブリックプレビュー開始。VS Code以外の主要IDE統合として初。一方、Skill設計では「21個運用して分かった」実践知見が日本語で共有され、Progressive Disclosure原則（frontmatter→SKILL.md→scripts/）やdescriptionの書き方など具体的なベストプラクティスが確立された。

4. ドメイン横断分析

横断 「モデル停止リスク」がFX自動売買の設計要件に昇格

Fable 5停止は「AIモデルの政治リスク」という新しいカテゴリの障害を顕在化させた。FX自動売買でLLMを判断エンジンに組み込んでいる場合、モデルが数時間で利用不能になるリスクへの対策が設計要件に加わる。Claude Codeの新fallbackModelパラメータ、TrueFoundryのマルチプロバイダーゲートウェイ、TradingAgentsのリモートOllama対応は、いずれもこの問題への解を提供している。

横断 MCPステートレス化 × Claude Code Routines × FX自動売買

MCPのステートレス化（7月28日RC）は、Claude Code Routinesによる定期実行ワークフローと直接連動する。Routinesで夜間にFXバックテストを自動実行し、MCPサーバー経由でMT5のデータや取引履歴にアクセスするアーキテクチャが、ステートレス化によりスケーラブルかつ信頼性の高いものになる。ただし7月28日のBreaking Changesに対応しないと、この統合パイプラインが動かなくなるリスクがある。

横断 AI規制とエージェント運用のガバナンスギャップ

EU AI Act 8月施行、日本AI促進法HITL義務化、米EO 14409の三方向から「AIエージェントの監視・制御・監査」が求められている。しかし現実にはエンタープライズの60%にガバナンスギャップがある。FX自動売買のLLMエージェントについても、取引判断の説明可能性（なぜこのポジションを取ったか）、HITL介入ポイントの設計、監査ログの保全が、規制対応として必要になりつつある。Claude CodeのArtifactsやAgent Teamsは、エージェントの意思決定過程を可視化するツールとして規制対応にも活用できる。

横断 メタトレンド: 「開発の非同期化」と「判断の自律化」

今週のデータから見える最大のメタトレンドは、人間の関与タイミングが「実行中」から「事前設計と事後確認」に移行していることだ。Claude Codeは夜間にRoutinesで自律開発し翌朝スマホで確認。FXエージェントは3層記憶で過去の失敗を自律的に回避。MCPステートレス化でサーバーは人間のセッション管理なしに水平スケール。人間の役割は「何を・なぜ」を定義し、結果を監査することに集約されつつある。この流れは規制（HITL義務化）とも適合しうるが、「事後確認で間に合わないケース」（FX急変時のLLM判断遅延、エージェント連鎖障害）への対策が次の課題になる。

5. アクション提案

  • 最優先 MCPサーバーの7月28日RC対応を開始する
    検証ウィンドウ残り4週間。Session-Id依存コードの削除、_meta対応、Sampling代替実装の3点が必須。自作MCPサーバーだけでなく、利用中のサードパーティサーバーの対応状況も確認が必要。Graviteeの移行チェックリスト（記事639）とTokenMixの9変更一覧（記事626）を基に作業計画を立てること。
  • 最優先 FX自動売買のモデルフォールバック体制を構築する
    Fable 5停止（記事658）により、最新モデルが無通告で利用不能になるリスクが現実化した。fallbackModelパラメータの設定（Opus 4.8→Sonnet 4.6→ローカルLLM）を即実装。TradingAgentsのリモートOllama対応（記事597）を参考に、ローカルLLMによる「主権的フォールバック」も検討する。
  • 今週中 Agent SDKクレジット使用量を確認し予算計画を更新する
    6月15日から課金分離が適用済み（記事617, 636）。bpr_labの日次収集スクリプト・FX自動売買のバッチ処理でclaude -pやAgent SDKを使用している場合、月間クレジット消費量を確認し、Pro $20/月の枠内に収まるか検証する。超過時はAPI従量課金にフォールバックされるが、コスト管理のため事前に把握が必要。
  • 今週中 MT5+ONNX+Python統合の検証環境を構築する
    MT5+ONNXによるMLフィルター追加で利益+57%の実証結果（記事663）、3層記憶SQLiteアーキテクチャ（記事631）、TradingAgents v0.2.5のGround Truth統合（記事597）と、FX×LLMの実装基盤が一気に成熟した。Claude CodeのRoutinesで夜間にバックテストを自動実行し、翌朝スマホで結果確認するワークフローが現実的に構築可能。
  • 月内 Claude Code Skills体系を整備する（/rewind・Cavemanスキル導入）
    21個のSkill運用知見（記事650）とSKILL.md書き方ガイド（記事644）で設計パターンが確立された。Cavemanスキルによるトークン65%削減（記事618）はコスト最適化に直結。/rewind（記事642）はFX取引ロジックの試行錯誤時の安全ネットとして即活用すべき。Progressive Disclosure原則に基づくfrontmatter→SKILL.md→scripts/の3層設計で既存スキルをリファクタリングする。
