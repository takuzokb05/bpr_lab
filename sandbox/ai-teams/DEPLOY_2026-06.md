# AI Teams v3 デプロイ手順 / 稼働記録（個人利用・出先アクセス / 2026-06）

VPS（ConoHa Windows Server, FX と同居）に **uvicorn 単一プロセス**で同居配信し、
**cloudflared クイックトンネル**で即時公開する個人向け構成。**2026-06-02 に実デプロイ・疎通確認済み**。

## なぜ VPS か（Vercel ではない理由）

バックエンドは **常駐プロセス**が必須でサーバーレス（Vercel）に載らない:
- SSE の長時間ストリーム（討論は数分、floor-open は無期限待機）
- リクエストを越えて生きるバックグラウンドスレッド（producer 完走 / paused 待機）
- インメモリのセッション状態（`SESSIONS` 辞書・スレッド・条件変数）＝**単一ワーカー前提**

→ FastAPI バックエンドは VPS。フロントは**静的書き出しして同じ uvicorn が同居配信**するのが最も簡単
（Node 常駐不要・低RAM・CORS 不要）。

## 実際の構成（2026-06 稼働中）

```
[ブラウザ(出先)] --HTTPS--> [cloudflared quick tunnel (*.trycloudflare.com)]
                                   │ （ポート開放なし・自動HTTPS・SSE素通し）
                              [uvicorn :8000 (VPS, 単一ワーカー)]
                                   ├ /sessions /personas /presets /intake → FastAPI（Bearer 認証）
                                   ├ /health                              → FastAPI（無認証）
                                   └ / , /_next/* , その他                → web/out の静的SPA（無認証で配信）
```

- フロントは `next build`（`output:"export"`）で `web/out/` に純粋な静的ファイルを生成。
- `api/main.py` 末尾で `web/out` があれば `StaticFiles` を `"/"` にマウント。**全 API ルートの後**なので
  `/sessions` 等が優先され、それ以外（`/` や `/_next/*`）は SPA を返す。→ **1プロセス・同一オリジン**。
- フロントの API 呼び出しは `apiUrl()`（`NEXT_PUBLIC_API_BASE=""`＝相対パス）でバックエンド直結。
  **Next.js の rewrite は使わない**（SSE をバッファして「終わってから一括表示」になる既知の罠。実測済み）。
  cloudflared は SSE を素通しする。

## 認証モデル（多層防御）

- `AI_TEAMS_API_TOKEN`（VPS `.env`）を設定すると、認証MWが **`/sessions` `/personas` `/presets` `/intake`**
  のみ Bearer を要求。`/` と `/_next/*` と `/health` は無認証（SPA を読めるように開ける）。
- フロントは `NEXT_PUBLIC_API_TOKEN`（ビルド時に埋め込み）を全 fetch の `Authorization: Bearer` で送る。
  **2つのトークンは同一値**にする（ビルド済み out/ と VPS `.env` を一致させる）。
- ⚠️ **`NEXT_PUBLIC_*` はクライアント JS に埋め込まれる＝真の秘密ではない**。ページを開けばトークンは見える。
  実効的なアクセス制御は **(1) 推測困難なランダム URL（クイックトンネル）** が主、トークンは
  「URL を知らずに backend へ直当てする素朴なスキャナ」を弾く控え。**本当のログインゲートが要るなら
  → Cloudflare Access**（後述の名前付きトンネル＋Access。要 Cloudflare ログイン＝対話操作）。

## 環境変数

`sandbox/ai-teams/.env`（gitignore 済み・VPS にのみ実値）:
```
ANTHROPIC_API_KEY=sk-ant-...      # 実LLM/Web検索用
AI_TEAMS_MODEL=claude-opus-4-8    # 任意（既定 opus-4-8）
AI_TEAMS_API_TOKEN=<長いランダム> # 認証トークン（フロントと同一値）
```
`web/.env.local`（ローカルのビルド時のみ・gitignore 済み）:
```
NEXT_PUBLIC_API_BASE=            # 空＝相対パス＝同一オリジン
NEXT_PUBLIC_API_TOKEN=<同じトークン>
```

## 再現手順

### ローカル（ビルド）
```
cd sandbox/ai-teams/web
npm run build            # → out/（静的SPA。NEXT_PUBLIC_* が埋め込まれる）
```
### VPS へ反映
```
# コードは git（main.py の StaticFiles マウント等）
ssh ... "cd C:\bpr_lab; git pull origin <branch>"
# 静的フロントは out/ を scp（git 管理外）
scp -r web/out  Administrator@<vps>:C:/bpr_lab/sandbox/ai-teams/web/
# .env は scp（チャットにキー/トークンを貼らない）。AI_TEAMS_API_TOKEN を out/ と一致させる
```
### VPS で常駐起動（Task Scheduler / 単一ワーカー）
- Windows では **Task Scheduler** で常駐（ssh 越しの `Start-Process` は silent death する既知の罠 →
  `Start-ScheduledTask` 経由。`ExecutionTimeLimit=0`＝無制限にしないと既定 72H でハードkill）。
- 稼働中タスク:
  - **`AITeams_API`**: `python -m uvicorn api.main:app --port 8000`（WorkingDirectory = ai-teams）
  - **`AITeams_Tunnel`**: `cloudflared.exe tunnel --url http://localhost:8000 --logfile C:\bpr_lab\cloudflared.log`
  - いずれも AtLogOn トリガ / `ExecutionTimeLimit=0` / `RestartCount` 付き。
- `cloudflared.exe` は `C:\bpr_lab\cloudflared.exe`（GitHub releases から取得・git 管理外）。

## クイックトンネル URL について（重要な制約）

- 公開 URL は `https://<ランダム>.trycloudflare.com`。**トンネル再起動（再起動・クラッシュ復帰）ごとに変わる**
  （＝ブックマーク不可）。
- **現在の URL を知る**: VPS の `C:\bpr_lab\cloudflared.log` を見る:
  ```
  ssh ... "powershell -c \"(Select-String C:\bpr_lab\cloudflared.log -Pattern 'https://[a-z0-9-]+\.trycloudflare\.com').Matches.Value | Select -Last 1\""
  ```
- **安定 URL ＋ ログイン認証が欲しくなったら → 名前付きトンネル ＋ Cloudflare Access**（要 Cloudflare ドメイン・
  `cloudflared tunnel login`＝ブラウザ対話）。これで固定ドメイン＋Google/メールのログインゲートになる。

## コスト・運用の要点（個人利用でも必須）

1. **セッションは既定 mock（無料）**。実 LLM/Web 検索はトグル時のみ従量課金。出先で気軽に回すと乗る。
2. **公開 URL が漏れると**、実 LLM トグルで**あなたの Anthropic キーで課金**され得る。ランダム URL＋トークンで
   一次防御、真に塞ぐなら Cloudflare Access。
3. **単一ワーカー固定**（インメモリ状態共有のため `--workers` を付けない）。
4. **`.env` を git に乗せない**（gitignore 済み）。**チャットにトークン/キーを貼らない**。
