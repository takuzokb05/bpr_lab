# UI/UX レビュー & リデザイン指針 (2026-05)

対象: v2 `app.py`（Streamlit）の UI と、プロジェクト全体の視覚言語。
決定済みの方向性:
- **見た目** = 報道番組系の「静かな上質さ」（広い余白・端正なタイポ・無彩色＋1アクセント）
- **絵文字** = ほぼ全廃し、線アイコン＋モノグラム・アバターの体系へ
- **実装基盤** = 別スタック（Next.js 系）への移行を検討（core が UI 非依存なので可能）

## 2. デザイン原則（報道番組系の静かな上質さ）

1. **絵文字でなく余白とタイポで階層を作る。** 装飾は引き算。
2. **無彩色を基調に、アクセントは1色だけ。** 色は「意味」がある時だけ使う（ペルソナ識別・状態）。
3. **黒ベタ(#000)をやめ、インク（やや温かい黒）と生成り（off-white）で上質な紙面感。**
4. **重い影より、細いボーダーと適度な余白で構造を示す。**
5. **動きは最小限・上品に。** 発言は静かにフェードイン。派手なアニメは禁止。
6. **報道のトーン**: ラベルは名詞で端正に（「介入」「巻き戻し」「議事録」）。煽り・厨二語を排除。

## 3. デザイントークン（提案値）

```
— Color —
--ink            #1F1D1A   /* 本文（温かみのある黒） */
--ink-muted      #6B675F   /* 補助テキスト */
--paper          #FAFAF7   /* 背景（生成り） */
--surface        #FFFFFF   /* カード面 */
--line           #E7E4DD   /* 罫線 */
--accent         #2E5D7D   /* 唯一のアクセント（深い青＝報道の知的さ） */
--accent-weak    #EAF1F5

— ペルソナ識別用カテゴリ色（彩度低め）—
thinking     #5B7C8A   founders     #8A6D3B   philosophers #6E5B8A
facilitation #4A4A4A   chair        #3B6E5B

— Type —
本文 : 'Noto Sans JP', system-ui    14px / 1.7
見出し: 'Noto Serif JP'（明朝を少量だけ）→ 報道紙面の格を出す
等幅 : ui-monospace（モデル名・トークン数など数値）
type scale: 12 / 14 / 16 / 20 / 28（控えめなモジュラー）

— Space — 8px グリッド（8/16/24/40）。余白は広めに。
— Radius — 6px（カード）/ 999px（アバター）。角は控えめ。
— Shadow — ほぼ無し。使うなら 0 1px 2px rgba(0,0,0,.04)。
```

## 4. アイコン / アバター体系（絵文字全廃の具体策）

- **アバター = モノグラム**: `display_name` 先頭1–2文字を、ペルソナのカテゴリ色の円に白文字で。
  例: 論理担当→「論」、ジョブズ→「SJ」、ソクラテス→「ソ」。絵文字 `avatar` は廃止。
  → ペルソナ・スキーマに `accent`（または category 由来の色）を持たせる。`avatar`(絵文字)は後方互換で残してもUIでは使わない。
- **UI アイコン = 線アイコン1セット**（lucide 等）に統一: 送信/巻き戻し/設定/コピー/議事録 など。見出しやボタンから絵文字を撤去。
- **状態は色とラベルで**: 発言中＝アクセントの細い左ボーダー＋「発言中」。未発言などの絵文字バッジは廃止。

## 8. 実装基盤の推奨（別スタック移行）

**結論: Next.js(App Router) フロント + FastAPI バックエンド（core を SSE で配信）。**
core が UI 非依存の純Pythonなので、この分離はそのまま実現できる。

```
[Next.js + Tailwind + shadcn/ui + lucide]
        │  SSE (text/event-stream)
        ▼
[FastAPI]  POST /sessions {topic, persona_ids}
        │   → council.run(topic) を yield ごとに Turn(JSON) でストリーム
        ▼
[core/]  Council / RoundRobinScheduler / build_context  ← 変更不要
```

- **なぜ別スタック**: 「報道番組系の上質さ」は Streamlit のコンポーネント/CSS制約では到達しにくい（型・余白・アニメ・アバターの自由度）。React+Tailwind なら#3のトークンを忠実に再現できる。
- **ストリーミングが自然に解決**: `run()` の `yield` を SSE でそのまま流すso、v2の「途中経過が見えない」が構造的に消える。
- **移行ステップ**:
  1. `api/`（FastAPI）: `GET/POST /sessions` で `council.run()` を SSE 配信。ペルソナ一覧 `GET /personas`。
  2. `web/`（Next.js）: SSE購読→タイムライン描画。デザイントークンを Tailwind config に落とす。
  3. デザインシステム: アバター(モノグラム)・発言カード・成果パネルをコンポーネント化。
  4. 永続化（Phase 6）は API 裏に SQLite/Postgres で後付け。
- **暫定案（任意）**: すぐ動かしたいだけなら、まず Streamlit に#3トークンの簡易CSS＋絵文字撤去だけ当てて「マシな見た目」にしてから移行してもよい。ただし本命は Next.js。

## 9. 機能の取捨選択（v2機能の整理）

| v2機能 | 判定 | 理由 |
|---|---|---|
| 討論タイムライン（ストリーム） | **残す（主役）** | 中核体験。SSEで強化 |
| 議事録（成果） | **残す** | chairman統合の出力先。core側と直結 |
| 介入（司会に指示） | **残す（改名）** | 人間が舵を取れるのは価値。名前を上品に |
| この発言から巻き戻す | **残す（改名）** | 「死に戻り」は機能自体は良い。やり直し導線として |
| ペルソナ編成・モデル設定 | **残す（左に集約）** | v3の多様性設計と直結 |
| 品質レビュー（旧:システム監査） | **残す（後段・任意）** | discッション後の自己点検。MVP後でも可 |
| 構造図 / 知識建築家 | **MVPから外す** | 実験的・重い。後で別機能として再検討 |
| ToDo タブ | **議事録に統合** | 「ネクストアクション」は議事録の一節で足りる |
| Command Center / 最近のプロジェクト | **簡素化** | 起動＝新規セッション。履歴は左に小さく |
