# 自分の小さな「箱」から脱出する方法

## なぜこの本を読むのか

今日1日で3冊読んで、テクニックは揃った。解像度を上げる型（解像度本）、問いに変換して渡す型（話す力）、仮説ドリブンの構造化（イシュー本）。でも「なぜテクニックが使えないか」が残っている。

堀江さんとの差分を掘り下げた結果、たどり着いたのは「答えが見えた瞬間に解決済みになる」「相手が辿り着くプロセスに興味が湧かない」という問題。これはテクニックの問題ではなく**相手をどう見ているかの問題**。

この本の「箱」= **自己欺瞞（self-deception）**。箱の中にいると、相手を「人」ではなく「物（問題・障害・道具）」として見てしまう。あなたが相談を受けたとき、相手を「解くべきクイズ」として見ている状態は、まさに「箱の中」。

## 要点3行

1. **「箱」= 自己欺瞞の状態**。箱の中にいると、相手を人間としてではなく「物」（問題を持った人、答えを待っている人、時間を奪う人）として見てしまう
2. **箱に入るきっかけは「自分への裏切り」**。「こうしたほうがいい」と感じたのにやらなかったとき、その行動を正当化するために相手の欠点を見つけ始める。正当化が箱を作る
3. **箱から出る方法は「相手を人として見る」こと**。テクニックではない。相手にも自分と同じように希望・不安・ニーズがあると認識した瞬間、箱の外に出ている

## 章別ノート

### 第1部: 「箱」という名の自己欺瞞の世界（p17-60）

- **「もしかしたら、あなたこそが問題を引き起こしている張本人かもしれませんよ」** — 冒頭のこの一文が全て
- 箱の中: 自分は正しく、相手が間違っていると見える。自分の視界が歪んでいることに気づかない
- 箱の外: 相手を人間として見ている。相手にもニーズ・希望・不安がある
- **「箱の中にいる人は、自分が箱の中にいることに気づかない」** — これが自己欺瞞の本質

### 第2部: 人はどのようにして「箱」にはまるのか（p61-130）

- **「自分への裏切り」**: 自分が「こうすべきだ」と感じたことに背く行為
- 裏切り → 正当化 → 相手を歪めて見る → 箱に入る
- 箱の中にいると:
  - 自分の欠点は見えなくなる
  - 相手の欠点が誇張されて見える
  - 自分を被害者だと感じる
- **選択の図（図表）**: 「自分への裏切り」→ 箱の中 → 相手を物として見る → 共謀が始まる

### 第3部: 箱からどのように抜け出すのか（p131-260）

- **「箱から出るためにテクニックを使おうとすること自体が、箱の中にいる証拠」**
- 箱から出る唯一の方法: **相手を人として見ること**
- 「こうしたほうがいい」という自分の感覚に素直になった瞬間、箱の外に出ている
- **リーダーの役割**: 自分が箱の外に出ることで、周囲の人も箱の外に出やすくなる
- 「問題を引き起こしているのは自分かもしれない」という問いを常に持つ

## 内省ワーク（問い5つ）

1. **直近で相談を受けたとき、相手を「人」として見ていたか「クイズの出題者」として見ていたか？** 相手の名前・表情・声のトーンを思い出せるか？それとも「問題文」しか覚えていないか？

2. **「もっと聞いたほうがいいな」と思ったのに解を出してしまった場面はあるか？** その直後、自分をどう正当化した？「忙しそうだったから」「答えを知っているから教えてあげたほうが早い」「この人には難しいだろう」— どれが近い？

3. **堀江さんが「ってことは〜ですか？」と聞けるのは、テクニックではなく「箱の外にいる」からではないか？** 堀江さんは相手を「答えを知らない人」ではなく「一緒に考える人」として見ている。あなたが同じ問いを出せないのは、テクニック不足ではなく「箱の中にいる」からかもしれない

4. **あなたの「自分の弱みを先に見せる」スタイルは、箱の外にいるときの自然な行動ではないか？** 弱みを見せるとき、あなたは相手を「人」として見ている。相談を受けるとき、なぜそのモードが切れるのか？

5. **「共謀」の構造に気づいたことはあるか？** あなたが答えを出し続ける → チームメンバーが「久保さんに聞けばいい」と依存する → あなたは「やっぱり自分が答えないと回らない」と思う → ますます答えを出す。この循環を止めるには？

## 明日から使えるアクション

- [ ] 次に相談を受けたとき、最初に**相手の名前を心の中で呼ぶ**。「○○さんが困っている」— これだけで「クイズ」から「人」に切り替わる
- [ ] 解を出しそうになったとき、「自分は今、箱の中にいないか？」と1回だけ自問する
- [ ] 「この人は答えを待っている」と思ったとき、それが**正当化ストーリーではないか**確認する。本当に答えを待っているのか、一緒に考えたいのか
- [ ] チームメンバーが相談に来たとき、「答えを出す前に、この人のニーズは何か？」と考える。答えがほしいのか、聞いてほしいのか、一緒に考えてほしいのか
