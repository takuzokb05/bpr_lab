## 意思決定パターン

- **即断即決 → フィードバックループ**。事前に完璧な仕様を作るより、出てきたものに反応して改善する
- **技術は即採用、制度は徹底調査**。新ツールは「まず触る」。組織制度は入念にリサーチしてから動く
- **小さく試して実績で引っ張る**。いきなり大きな提案をせず、小さな成功事例を積み上げてから横展開（「助走レーン」思考）
- **選択肢を提示すると素早く判断する**。「Cにしようか」とテンポよく選ぶ
- **リスク許容度: 技術は高い、対人は慎重**

## コミュニケーションスタイル

- **口語で業務知識を一気に放出**し、整理・構造化はAI側に委ねる
- **語尾**: 「〜かなぁ」「〜よねぇ」「面白いけど」「まあいいか」「そうしよう」
- **Yes, And で返す**。相手の提案を否定せず受けてから自分の視点を追加
- **自分の弱みを先に見せる**。「自分くらいのレベル感だと」と言い、心理的安全性を作る
- **違和感は率直に指摘**する。「そもそもそういう使い方じゃないと思うんだけどな」
- **押しつけない**。「こうすべき」ではなく「こんな形もあり得るのかなと」

## AIとの協働スタイル（＝Claudeへのリクエスト）

### 役割分担
| 自分がやること | AIに任せること |
|---------------|---------------|
| 方向性の判断、ステークホルダー関係構築 | 調査・リサーチ、コード実装 |
| ドメイン知識の提供（口語で一気出し） | 文書の叩き台作成、構造化 |
| 組織政治的な判断 | 選択肢の網羅的列挙 |

### 好む対話の進め方
- **テンポよく進めてほしい**。自分で検証可能な情報の確認は不要。自律的に進める
- **選択肢は端的に提示**。長い説明より「A/B/C、どれにする？」
- **5分以上かかる大きな作業は事前にリスク併記で確認**を取る
- **成果物が出た後に視覚的・体験的に確認**してから方向修正を入れるスタイル
- **叩き台→自分の思考を乗せる→対話の素材にする**。AIの出力をそのまま使わず、自分の経験で加工して対話に持ち込む

### 避けてほしいこと
- 冗長な説明や毎回の要約（「diff読めるから」）
- 過度な確認（自律的に進めてOK）
- 過剰な丁寧語や絵文字

## 価値観

- **「すごい人」より「一緒にやる人」**。ヒーローになるより周囲が動けるようになることを重視
- **「もったいない」精神**。既存投資・データの活用を重視
- **後に残る仕組みへのこだわり**。「俺いなくなるからさ、なるべく後任が迷わないものを作りたい」
- **心理的安全性が最優先**。制度より文化、仕組みより信頼関係
- **段階的アプローチ**。0→10→30→60→100。各段階に「撤退の自由」を確保
- **先駆者になることに積極的**。「税畑のファーストペンギンになれそう」

## 思考の型

- **「速度の10x」と「方向の10x」を区別する**。AIは同じ方向に速く走れる（速度）。人との対話は走る方向自体を増やす（方向）。方向のほうが価値が高い
- **「答えを持つことの落とし穴」を自覚**。成果物よりプロセスを共有することで相手の参加余地を作る
- **GROWモデル的な整理**。Goalを問い直す力・現場の肌感覚・「本当にやるのか」を問う力は人の領域
