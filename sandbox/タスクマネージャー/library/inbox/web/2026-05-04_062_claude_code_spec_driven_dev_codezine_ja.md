## 要約
CodeZineによるClaude Codeを使った仕様駆動開発（Spec-Driven Development）の実践解説。「SPEC.md」「ARCHITECTURE.md」などの仕様文書をまず作成し、Claudeがそれに基づいて実装を行う開発手法を詳しく説明。3フェーズ開発（計画→実装→検証）のフレームワークと、Claude Codeの/planコマンドを活用した計画フェーズの効果的な進め方を解説。仕様書をClaude Codeのコンテキストに組み込む方法（@importを使ったCLAUDE.mdへの参照）、AIが仕様から外れた実装をした場合の軌道修正方法も含む。非エンジニアでも仕様書を書くことでAI開発に参加できるという民主化の視点も提示。CodeZineは翔泳社の開発者向けメディアで記事の技術的品質が高い。
