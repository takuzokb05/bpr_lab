## 要約
2026年4月29日公開のMQL5フォーラム技術記事。LLMとMQL5 EAを統合する本番稼働可能なアーキテクチャを詳解。4層構成：MQL5データパブリッシャー→Pythonミドルウェア（FastAPI）→LLM推論→実行ゲートウェイ。重要な設計原則：(1) JSON Schemaによる厳格なバリデーション—コントラクト違反は全拒否で幻覚トレードを防止。(2) 信頼度スコアリング（0.0〜1.0）によるポジションサイジング、0.55未満はノーエントリー。(3) EA（観察・実行）、ミドルウェア（推論）、LLM（推測）の責任分離。(4) 30〜60取引日のステートフルコンテキストウィンドウでin-context学習を実現。(5) 悪意あるデータ・極端値・エッジケースへの敵対的テスト実施。従来の「AI EA」はRSIラッパーに過ぎず真のLLM統合には非同期推論パイプラインと決定論的出力バリデーションが必須と主張。MT5自動取引とLLMの実装において最も技術的に詳細な一次資料の一つ。
