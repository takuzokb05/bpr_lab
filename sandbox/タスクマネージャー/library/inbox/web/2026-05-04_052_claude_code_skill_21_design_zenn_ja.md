## 要約
yamato_snow氏による21個のClaude Code Skillを実際に運用して得た設計知見のZenn記事。「スキルの作り方」だけでなく「育て方（継続的改善）」まで含む実践的なガイド。21個運用した経験から導き出した設計原則：①descriptionの重要性（最初の250文字が自動トリガーの精度を決定する）②スキルの肥大化防止（reference.mdへの詳細分離）③バージョン管理の必要性（スキル自体のGit管理）④チームでの共有ベストプラクティス（.claude/skills/のリポジトリ管理）⑤スキルの廃止基準（使用頻度・メンテナンスコスト評価）。実際に機能したスキルと失敗したスキルの具体例が含まれており、試行錯誤の現実的な情報として貴重。
