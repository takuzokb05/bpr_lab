## 要約
日常的にClaude Codeを使い込んで得た実践的なTips集。主要ポイント：(1) Plan Modeを複雑タスクで必ず使い、仮定を早期に検証する。(2) ステップ毎に実装を依頼しレビューを挟む。(3) `/rewind`・`/clear`コマンドで誤った方向に進んだ場合はリセット。(4) Previewでコード変更を自律的にテストさせる。(5) セッション後にClaude自身に「学んだこと」を振り返らせてCLAUDE.mdへ反映。(6) `/simplify`・`/review`をPRレビュー前に実行。(7) `@path/to/file.ts` 構文でファイルを直接コンテキストへロード。(8) Context7・Snyk MCPなど厳選した数個のツールのみ追加。(9) CLAUDE.mdは200行以内に収め業務ロジックを中心に記述。(10) 過剰な並列化・worktreeの乱用は避けてエラーを早期に検出する。
