## 要約
Jonathan Fulton氏によるClaude Code スキルを「チートコード」と表現した解説記事。スキルがなぜ強力なのか（毎回プロンプトを書かずに済む・チームで標準化できる・複雑なロジックを再利用できる）を明快に説明。特に「チームのやり方をエンコードする」Encoded Preference型スキルの価値を強調：NDレビュー・週次ステータスレポート・コミットメッセージフォーマット・コードレビューチェックリストなど、Claudeが本来できるタスクをチーム固有の方法で自動化する例が豊富。SKILL.mdのdescriptionフィールドが最重要（自動起動判断に使用・250文字で切り詰め）という具体的な設計知識も含む。個人スキル（~/.claude/skills/）とプロジェクトスキル（.claude/skills/）の使い分けも解説。
