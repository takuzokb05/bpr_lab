## 投稿内容
■ サマリ
xAIがGrokに外部サービスとの直接連携機能をリリース。Claude CodeやOpenAIのAgents SDKに続いて、Grokもツール統合の戦場に本格参入してきた

■ 主要機能
・Connectors: GitHub/Notion/Linear/Google/Microsoftとのネイティブ連携
・Remote MCP Tools: 外部MCPサーバーへの接続で、サードパーティのツールを自由に拡張
・Voice Agent API対応: 音声エージェントからもMCP経由でツール操作が可能

■ 背景
MCPがAIツール連携の事実上の標準規格になりつつある。Claude Code → OpenAI Agents SDK → Grokと、主要プレイヤーが全員MCP対応に動いた。対応してない方がリスクになるフェーズに入ってて、ここから先はコネクタの数と品質で差がつく

■ ポイント
カスタムMCPコネクタに対応してるのが地味にデカくて、企業が社内の独自ツールをGrokに繋げられるようになった。Claude Codeと完全に同じ土俵に立った形

## 要約
@latdayoが詳細を解説。xAI GrokがConnectorsとRemote MCP Toolsをリリース。ConnectorsはGitHub/Notion/Linear/Google/Microsoftとのネイティブ連携、Remote MCP Toolsは外部MCPサーバーへの接続で第三者ツールを自由に拡張、さらにVoice Agent API対応でMCP経由での音声エージェント統合も可能。Claude CodeとOpenAI Agents SDKに続き、GrokもMCPツール統合の競争に本格参入。エコシステムの水平展開が加速。
