## 投稿内容

Qiita の実践ガイド。スキル設計の3原則から運用フェーズの知見まで、21個運用した経験をもとに解説。

## 要約

スキルは「YAMLフロントマター付きのMarkdownファイル（SKILL.md）1つ」で Claude Code の振る舞いを根本から変えられる。設計の3原則：（1）1スキル1責務、（2）description は動詞始まりで具体的に（Claude の自動判断の精度が変わる）、（3）instructions は手順より意図を書く。description フィールドが最重要で、Claude がこの説明文を読んで「このスキルを使うべきか」を自動判断する。運用フェーズの知見：口頭指示を繰り返した場合はスキル化のサイン。セッションが変わっても記憶が持続するのがスキルの最大価値。21個運用した知見として、スキルが増えすぎたら description が曖昧なものを統合するリファクタリングが必要。スキルの粒度は「1回のタスクで1スキルが呼ばれる」程度が適切。
