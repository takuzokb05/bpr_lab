最近、X界隈で「Claude Codeを使い始めました！」という私のような非エンジニアの方の声をチラホラ聞くようになりました。

AIがファイルを作ってくれたり、調べものをしてくれたり。正直、めちゃくちゃ便利ですよね。

でも、使い始めてすぐに気づいたことがあります。

「このAI、私のファイルを消すこともできるのでは…？」

AIは賢い。でも、賢いからこそ、自分の判断でどんどん作業を進めてしまうことがある。こんな感じで間違った方向に暴走されたら困ります。

そこで私がやったのが、AIに「やっていいこと」と「やっちゃダメなこと」をちゃんと教えるための設定です。

設定ファイルはたった2つ。 どちらもテキストファイルなので、エンジニアじゃなくても大丈夫です。

この記事では、その2つのファイルの中身を全部見せながら、一行ずつ「これは何をしているか」を解説していきます。

# AIに「ルール」が必要な理由

Claude Codeは、普通のチャットAIとは違います。 ファイルを作る、移動する、ソフトをインストールする。パソコンの中で実際に手を動かしてくれるアシスタントです。

これ、言い換えると「パソコンの操作権限をAIに渡している」ということです。

便利な反面、リスクもあります。

たとえば「フォルダを整理して」と頼んだつもりが、大事なファイルを消されてしまったら？ 知らないソフトを勝手にインストールされたら？

人間のアシスタントを雇うときだって、最初に「これはやっていいけど、これはダメ」と伝えますよね。AIも同じです。

Claude Codeには、そのための仕組みがちゃんと用意されています。それが次の2つのファイルです。

- settings.local.json — AIの行動を「仕組みとして」制限する設定ファイル
- CLAUDE.md — AIに「なぜそのルールがあるか」を文章で伝えるルールファイル

どちらも .claude というフォルダの中に置きます。

# settings.local.json — AIに「鍵」をかける

まず1つ目。settings.local.json です。

このファイルは、AIの行動を「仕組みとして」コントロールします。日常に例えるなら、ドアに取り付ける「鍵」のようなものです。AIがルールを忘れても関係ない。鍵がかかっていれば、物理的に開かないのと同じです。

## allow（許可リスト）— 確認なしでやっていいこと

ここに書いた操作は、AIが「やっていいですか？」と聞かずに、自分の判断で実行できます。安全だと確認できたものだけを入れています。

■ファイル操作: mkdir, move, cp
■バージョンの確認: node --version, npm --version
■プロジェクト操作: npm init -y, npm start
■Webアクセス: Pexels, Pixabay, code.claude.com 等

## deny（禁止リスト）— 絶対にやってはダメなこと

- rm — ファイルを消す（Mac/Linux用）
- rm -rf — フォルダごと中身も全部強制削除する。最も危険な操作
- del — ファイルを消す（Windows用）
- rmdir — フォルダを消す（Windows用）
- Remove-Item — ファイルやフォルダを消す（Windowsの別のやり方）

# CLAUDE.md — AIに「ルール表」を見せる

2つ目のファイルが CLAUDE.md です。

settings.local.json が「鍵」なら、こちらは「壁に貼ったルール表」です。

## 運用ルール（0番）— 「どう振る舞ってほしいか」の基本姿勢

「すべて日本語で」「専門用語は使わない」「分かりやすく説明する」と明記。

## 作業スタイル（1〜6番）— 「どう仕事を進めるか」のルール

特に大事なのは「1. Plan ノード」の部分。複雑な作業をいきなり始めるのではなく、まず計画を立ててから動くようにしています。
「3. 自己改善ループ」も面白いところで、AIが自分のミスを記録して、同じ失敗を繰り返さないようにする仕組みです。

## コア原則 — AIの「仕事のポリシー」

3つだけ。シンプルにやれ、手を抜くな、余計なことはするな。

## 安全ルール（7〜10番）

- 7番：ファイルを書き換える前に「いいですか？」と聞く
- 8番：ファイルを消す操作は原則やらない
- 9番：ソフトを入れる前に「何を・なぜ・どこに」を説明する
- 10番：よくわからないコマンドは実行前に説明する

# なぜ2つとも必要なのか

CLAUDE.md にルールを書いておけばAIは守ってくれるはず。でも、AIは長い作業をしているうちにルールを見落とすことがある。

逆に、settings.local.json だけでは「やっていい・ダメ」しか伝えられない。理由は書けません。

- settings.local.json は包丁に安全カバーを付けること。物理的に刃に触れない
- CLAUDE.md は「包丁は危ないから気をつけて」と伝えること。応用が効く

両方やって、初めて安心できます。

# まとめ — 二重ロックで、安心してAIと仕事する

やることは2つだけです。

- .claude/settings.local.json を作って、許可と禁止のリストを書く
- .claude/CLAUDE.md を作って、AIへのルールと理由を文章で書く

settings.local.json は「鍵」、CLAUDE.md は「ルール表」。 片方だけではなく、両方を組み合わせた「二重ロック」がおすすめです。
