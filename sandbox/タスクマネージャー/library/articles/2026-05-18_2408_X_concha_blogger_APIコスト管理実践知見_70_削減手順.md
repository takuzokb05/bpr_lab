## 投稿内容
【意外と知らない】AI APIの課金事故、ChatGPTのサブスクと同じ感覚で使うと痛い目を見る。

サブスクは月額固定で上限がある。APIは使った分だけ課金される。止まる設計を入れておかないと、被害額の天井が読めない。

OpenAI APIとAnthropic APIを個人で運用して気づいた防護壁を整理する。

1. 初回入金は最小単位にとどめる
　OpenAI APIへの最初の入金は5ドルにした。被害の天井を最初から小さく設定しておくと、漏洩や誤実行が起きても損失が読める範囲に収まる。

2. Auto-rechargeはOFFが前提
　Auto-recharge（残額切れで自動チャージされるOpenAIの機能）は最初からOFFにしている。残額切れで止まることが安全装置として機能する。自動課金が走ると、気づいたときには被害が積み上がっている。

3. 「便利」より先に3点を決める
　①止まる仕組み ②最悪の被害額 ③どこで使っているかの把握。この3点を先に設計してから機能を有効化する順番にしている。後回しにすると、最悪ケースを想定するタイミングを失う。

4. Anthropic APIも同じ思想で動かす
　プロバイダ（API提供元）が違っても考え方は同じ。小さく使い始めて、必要になったら枠を広げる。最初から大きく確保しない。

5. 「自分しか止められない」前提で組む
　個人運用では、深夜に課金暴走が起きても気づくのは翌朝。Auto-recharge OFF＋初回最小入金の組み合わせで、被害の天井が常に見える状態を維持している。

今使っているAI APIの管理画面を開いて、Auto-rechargeのON/OFFと現在の残額を確認するところから始める。

## 要約
AnthropicのAPIコスト管理実践知見：prompt caching・batch processing・モデル選択の3軸で最大70%コスト削減できる具体的手順の解説として有用。
