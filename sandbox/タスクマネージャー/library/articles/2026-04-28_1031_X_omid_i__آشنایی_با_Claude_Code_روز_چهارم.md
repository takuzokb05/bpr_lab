آشنایی با Claude Code: روز چهارم

این MCP چیه و چه طور میشه به سادگی با Claude از ویژگی های MCP استفاده کرد؟

تو روزهای قبلی راجع به Skill و Sub-agent گفتیم. ولی یه چیز هست که اگه نباشه، همه‌ی این‌ها محدود می‌مونن: MCP!

یه زمانی هر دستگاه برقی، پلاگ مخصوص خودش رو داشت. یه پلاگ برای یخچال، یه پلاگ دیگه برای تلویزیون، یه شکل دیگه برای جاروبرقی. تا اینکه یه استاندارد جهانی به اسم «پریز برق» تعریف شد. حالا هر شرکتی توی دنیا، می‌دونه باید پلاگ دستگاهش رو با چه شکلی بسازه. تو هم به‌عنوان مصرف‌کننده، خیالت راحته که هر دستگاهی بخری، توی پریز خونه‌ت کار می‌کنه.

اینجاست که MCP  یا همون Model Context Protocol میدرخشه!
ساده بخوایم بگیم MCP همین پریز برقـه، فقط برای Claude.

یعنی MCP یه استاندارد جهانیه که Anthropic تعریف کرده تا هر ابزار و سرویسی توی دنیا بتونه به‌صورت یکپارچه به Claude وصل بشه.
الان Gmail، Slack، Notion، Figma، GitHub، Google Drive، Asana، Linear، HubSpot، دیتابیس‌های PostgreSQL و MongoDB، و صدها سرویس دیگه، همه «MCP Server» خودشون رو ساختن. یعنی پلاگشون رو با شکل MCP درست کردن.

ما فقط کافیه وصلشون کنیم تا بتونیم کارهایی مثل لیست پایین رو انجام بدیم:

- ایمیل‌های دیروزم رو خلاصه کن: Claude میره توی Gmail می‌خونه و خلاصه میده
- این فیچر رو تبدیلش کن به یه pull request: اینجا Claude خودش می‌ره وصل میشه به GitHub و یه PR می‌سازه
- از دیتابیس کوئری بزن ببین چند تا کاربر هفته‌ی پیش ثبت‌نام کردن: Claude مستقیم به دیتابیس متصل میشه و کوئری می‌زنه
- این صفحه‌ی Figma رو ببین و کامپوننت‌های React ـش رو بساز: Claude میره Figma، طراحی رو می‌بینه و کدش رو می‌نویسه

اتصال MCP Server های مختلف هم به Claude Code خیلی ساده است:
توی Claude Code دستور claude mcp add رو می‌زنی، اسم سرور رو میدی، احراز هویت می‌کنی (مثلاً برای Gmail، با حساب گوگلت لاگین می‌کنی) و از اون به بعد Claude می‌تونه باهاش کار کنه.

توی این لینک هم خود کلاود کلی از MCP Server هاشو گذاشته:
https://t.co/9VUic57Gvm

با دستور Claude mcp list هم میتونید ببینید در حال حاضر کلاودتون به چه MCP Server هایی وصله. مثالش رو از کامپیوتر خودم اینجا گذاشتم.
