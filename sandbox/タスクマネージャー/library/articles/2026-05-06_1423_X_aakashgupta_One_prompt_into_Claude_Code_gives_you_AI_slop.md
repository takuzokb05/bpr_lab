## 投稿内容
One prompt into Claude Code gives you AI slop.

21 specialized agents working together gives you a production app shipped to TestFlight in a single session.

The gap between those two outcomes is the entire skill curve PMs are currently climbing.

Most people loading up Claude Code and typing "build me an app" are operating at level zero. The output is unusable code, hallucinated dependencies, and spaghetti architecture that breaks the moment you touch it.

The 21-agent setup operates at a completely different level. You're not asking one model to do everything. You're orchestrating specialists.

System analyst agent writes the spec into Confluence. Designer agent generates the Figma Make prototype with brand guidelines baked in. Engineering agents pick up frontend tickets in Jira with the Figma links already attached. Build agents push to TestFlight. Each agent has a defined role, a defined input, and a defined output.

This is the part people miss. The agents only work because of the scaffolding underneath them.

The reason most AI builds collapse into spaghetti code is that there's no system architecture before the prompts start flying. The 21-agent approach forces you to define what a "good" system analyst looks like before any code gets written. What does a good PRD contain? What does a good ticket look like? What handoff format does the engineering agent expect from the designer?

Once those rails exist, the agents just execute. You can rebuild a hockey rules app, an internal tool, or a startup MVP with the same scaffolding.
The other piece worth extracting: dictation beats typing for AI specs.
Voice notes capture the texture of how you actually think about a feature. The asides, the trade-offs you considered, the edge cases you remembered halfway through. Typing forces you to compress before the model can see your reasoning. A 5-minute dictation often produces better output than a 500-word written prompt.

The career signal here is hard to ignore.

Classic PM skills (specs, scoping, requirements, acceptance criteria) are what make this stack work. The PMs panicking about AI replacing their job are the ones who skipped those skills in the first place.

The PMs who actually wrote good PRDs at their last job are about to become significantly more valuable, because the same skill that made them effective with engineers makes them effective with agent teams.
The full workflow demonstrated in this episode goes idea to prompt to design to app to TestFlight in one session.

That used to be a sprint. Now it's an afternoon.
