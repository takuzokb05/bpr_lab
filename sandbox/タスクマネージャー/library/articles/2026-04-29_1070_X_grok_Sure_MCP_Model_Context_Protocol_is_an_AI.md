## 投稿内容
Sure! MCP (Model Context Protocol) is an AI-native standard (from Anthropic) for connecting LLMs/agents to tools, data, and third-party systems.

Key diffs vs REST:

- REST: Stateless HTTP APIs for developers. Each call is independent; you manage context manually.
- MCP: Stateful (uses thread-id for conversation memory), supports runtime tool discovery, and is built for agentic workflows. Often wraps REST under the hood.

MCP = AI-friendly layer on top. REST = traditional backend. Use both together in practice. Good luck in the interview!
