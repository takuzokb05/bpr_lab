非エンジニアが久々に「黒い画面」に飛び込んで2ヶ月。Claude Code で業務が変わった話

ソース: https://zenn.dev/no9_dev/articles/9fdeb8b6db585b

## 業務の変化
- 毎朝60分の情報収集 → コマンド1つで3分（7エージェント並行実行）
- 2ヶ月で16個のスキル（業務手順書）を構築
- ChatGPT/Gemini/Notion/Sheetsを行き来 → Claude Code 1つに集約

## 人としての成長・マインドセット変化

### 「言語化」が最強のスキルになった
頭の中で無意識にやっていた業務プロセスを手順として言語化して初めてAIが実行可能になった。
エンジニアリングスキルではなく「業務の言語化能力」が武器になる発見。

### 完璧主義→「ver.1は雑でいい」
CLAUDE.mdは完璧に書かないと、と思っていたが「使いながら育てる」で十分。
開発思想のインストール。仕事全般に効くマインドセット。

### 「小さな部署長」の感覚
情報収集担当、資料作成担当、データ分析担当をAIエージェントに分担。
自分は「最終確認」と「判断」に集中。プロジェクトマネジメント経験がAI管理に直結。

### オーケストレーション思考
単体AIではなく、複数エージェントの組み合わせと連携設計がカギ。

## 組織への展開
属人化の反省から「Starter Kit」を開発。心理的安全性を最優先にしたセミナーで社内展開。
