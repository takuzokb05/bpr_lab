## 投稿内容
Coding was the fastest part of building this app. That one fact rewrites how a software company should be organized.

A 21-agent team running 4 parallel sprints inside Claude Code shipped from idea to App Store submission in 72 minutes. The actual code generation took less time than the spec dictation, less time than the design review, less time than the ticket-writing. The activity that has dominated every software project for 50 years became the throwaway step.

The labor pyramid that built every tech company was downstream of one assumption: engineers were the scarce resource because compiling intent into syntax was hard. Specs, tickets, and review existed to ration engineering hours. The entire org chart, sprint structure, code review process, and engineering manager hierarchy was a giant queue management system for one expensive resource.

That resource is no longer expensive. The second-order effects are wild. Sprint planning was a tool for queueing engineering throughput. Design handoff was a tool for spending engineering tokens carefully. Code review was insurance against expensive engineering mistakes. All three rituals were calibrated for a constraint that has now collapsed.

The new bottleneck is upstream. Whoever can spec, ticket, and design with precision is the new constraint. The work that used to be context for engineering became the entire workflow. Engineering is the cheap step that gets you 95% of the way there. The other 5% is wherever the spec was sloppy.

The code writes itself when everything before the code is right.
