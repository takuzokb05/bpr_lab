## 投稿内容
If you've been building with LLMs, you've probably heard both terms. They sound related, and they are — but they solve different problems.

Tool calls are a capability.

A tool call is a feature of the model itself. The LLM is trained to emit structured JSON saying "I want to invoke this function with these arguments." Your code reads that JSON, runs the function, and feeds the result back.
You define a function. The model emits a request to call it with specific arguments. You execute it. You return the result to the model.

Every tool is wired up by hand. You write the function, write the schema, register it with the SDK. If you want the same tool in three different apps, you implement it three times.
