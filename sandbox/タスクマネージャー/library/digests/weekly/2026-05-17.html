今週の重要トピック

Claude Code

大型機能追加: Agent View, /goal, /loop, Routines

今週のClaude Codeは、単なる機能追加ではなく「CLI対話ツール」から「マルチエージェント開発基盤」への構造的転換が起きた。Agent Viewで全セッション・全エージェントを一覧管理、/goalで目標達成まで自動継続、/loopで繰り返し実行、Background Sessionsでフォアグラウンドから独立した並列処理、Routinesでスケジュール・GitHubイベント・APIトリガー起動が可能になった。PCを閉じてもクラウドで自動実行が継続する世界が現実になっている。

Fast modeのOpus 4.7対応（出力2.5倍速、入力$30/出力$150 per MTok）も即戦力。5月14日からOpus 4.7がFast modeのデフォルトになり、同日のOpus 4.7全体デフォルト化と合わせて「Claude Codeの全力」がOpus 4.7ベースに統一された。

6/15課金変更: CI/CD自動化ユーザーに直撃

今週最大のインパクト。6月15日からclaude -p（非インタラクティブ）、Agent SDK、GitHub Actions、サードパーティSDKアプリがインタラクティブ利用枠とは別の専用クレジットから引かれる。クレジット額はPro $20/月、Max 5x $100/月、Max 20x $200/月。超過時はAPIレートで従量課金または停止。同日、claude-sonnet-4とclaude-opus-4がAPIから引退しSonnet 4.6・Opus 4.7が新デフォルトに。

隠しオプション--prefillが-pの代替として機能するとの報告もあるが、公式ドキュメント外のため動作検証は必須。

実践知見の蓄積: CLAUDE.md設計とhooks

6ヶ月間Claude Codeを使い込んだ開発者のガイドが複数公開された。共通するのは「禁止事項・承認フロー・スタイルガイドをCLAUDE.mdに明記する」ことで失敗率が激減するという知見。PreToolUseフックでwrite前に自動lint、PostToolUseでテスト実行を強制する構成が効果的。rtk CLIでコマンド出力をLLMコンテキスト投入前に圧縮するとコスト約40%削減も実現できる。5/16にはCLAUDE.mdベストプラクティス9則、MCP vs Skills vs Hooks解説も公開され、初心者から上級者まで参照できるリソースが整った。

接続: この実践知見はFX自動売買のClaude Code開発にも直接適用可能。CLAUDE.mdにトレード戦略の禁止事項（過剰レバレッジ等）を明記する運用が考えられる。

CLI戦争: Claude Code vs Codex vs Grok Build

xAIのGrok Build参入でCLIコーディングツールは三つ巴に。Grok Buildは200万トークンコンテキスト・最大8エージェント並列だが、$99/月は6ヶ月導入割引で通常$300/月。Claude Code Pro $20/月との5倍の価格差が焦点。一方MicrosoftがClaude Codeを6/30で社内廃止し、80-100席キャンセルする企業も出た。「Claude Code + Codexをスキル経由で統合」する実践パターンも登場しており、ツール間の壁が低くなっている。
