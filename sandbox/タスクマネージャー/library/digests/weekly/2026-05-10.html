03今週の重要トピック

CLAUDE CODE 開発ツールからAIオペレーティングシステムへ

Code w/ Claude 2026カンファレンス

5月6日サンフランシスコ開催。3つの革新的機能が発表された。Dreamingはエージェントが過去のセッションをオフラインでレビューし、失敗パターンを抽出して自己改善する機能。Outcomesはエージェントが成功基準を自ら定義し、達成するまで反復する機能。Managed Agentsはリードエージェントが専門エージェントに委任するマルチエージェント統合機能。API利用量は前年比17倍。

デスクトップアプリGA（マルチウィンドウ・フルスクリーン対応）、/ultrareview（並列マルチエージェントコードレビュー）、CI自動修正（GitHub Actions統合）も同時発表。東京開催は6月10日。

SpaceX提携によるレート制限2倍化

SpaceXAIのColossus 1（220,000+ NVIDIA GPU、300MW+）の提供により、Claude Codeの5時間レート制限が全プランで2倍に引き上げ。Opus API Tier 1は入力+1500%、出力+900%。ピーク時間の制限カットはPro/Maxで完全撤廃。長時間の自律コーディングが中断なく実行可能に。「重いタスクを遠慮なく投げられる時代」として実務者から歓迎。

Claude Codeの構造的優位性が学術的に確認

UCLのリバースエンジニアリング研究により、Claude Codeのコードベースのうちモデル（LLM）はわずか1.6%、残り98.4%はエージェントハーネス・基盤コードだと判明。競争優位はモデルではなく周辺のエージェントシステムにある。Boris Chernyは「Claude Codeは将来〜100行のコードに縮小する」と予測。Anthropicの全社コードが既にモデル生成であるという証言と合わせ、AIコーディングの「自己参照的な進化」が進行中。

v2.1.136リリースと実務的進化

hard_deny自動ルールの追加（ユーザー権限に関わらず強制ブロック）、MCPサーバーが/clearで消えなくなる改善、OAuth/ログイン競合の修正など。HookイベントがInstructionsLoaded、ConfigChange、PostCompactの追加で14→21に拡大。HTMLがMarkdownに代わるエージェント出力の標準フォーマットになりつつある（HTMLは生成速度2-4倍、共有に適する）。

接続: SpaceXの計算能力提供→レート制限2倍→長時間自律コーディングが実用化→Dreaming/Outcomesでエージェントが自己改善→開発者1人がClaude Codeで年間120万行を出力する生産性ループが完成。

04ドメイン横断分析

メタトレンド1: コンピュートが新たなボトルネックに

モデル性能の差は縮小し、計算資源の確保量が勝敗を決める構造に移行した。Anthropicの合計10GW確保（SpaceX+Amazon+Google+Microsoft+Fluidstack）は国家レベルの電力消費。この計算能力がClaude Codeのレート制限緩和となって開発者に還元され、生産性向上→ARR成長→さらなるコンピュート投資というフライホイールが回る。FX自動取引でも、レート制限緩和は長時間のバックテスト・最適化ループを中断なく実行できることを意味し、EA開発の質的転換点となる。

メタトレンド2: MCPが「AIのTCP/IP」として確立

97M SDKダウンロード、10,000+公開サーバー、主要プロバイダー全社採用。12ヶ月で「一企業のプロトコル」から「業界標準」に成長した速度はHTTPやREST APIの普及を大幅に上回る。影響はClaude Code（開発ツール連携）、FX取引（MT5+MCP公式ガイド）、エンタープライズ（AWS/Azure/Splunk/M365）、クリエイティブ（Blender/Adobe/Canva）と全ドメインに及ぶ。「MCPを持たないことがリスク」の時代。

メタトレンド3: エージェントの「ハーネス」が競争優位に

UCL研究のClaude Code分析（1.6% LLM vs 98.4%ハーネス）は、モデルそのものよりも周辺のオーケストレーション・ツール統合・メモリ管理が競争の鍵であることを実証した。21エージェントチームが72分でアプリを出荷できるのは、Claude Codeのハーネス（Skills/Hooks/MCP/並列実行）の成熟度による。FX取引でもTradingAgentsの7エージェント構成やQuantDingerのDocker Composeベースのセルフホスト型が、単一モデルのプロンプト一発より遥かに高い成果を出している。「新しいボトルネックは上流にある。仕様・チケット・設計を精密にできる者が新たな制約」。

メタトレンド4: 「人間不要」の証拠が積み上がる一方で、人間の役割が再定義されている

Boris Chernyの「Anthropicに人間が書いたコードはない」、Goldman Sachsの600→2人、882人/日の人員削減。しかし同時に「クラシックPMスキル（仕様、スコープ、要件）がエージェントスタックを動かす」「曖昧な指示はスロップを生む、具体的な計画+コンポーネント単位実行で10倍」という知見も蓄積された。人間の価値は「コードを書く」から「方向性を決め、品質基準を設定し、エージェントチームを設計する」に移行。ユーザーの「速度の10xと方向の10xを区別する」思考がまさに活きる局面。

  2. HIGH

    全プランのレート制限が2倍、Opus APIは入力+1500%/出力+900%。これまでレート制限で断念していた長時間の自律タスク（大規模リファクタリング、並列エージェント実行、バックテストの連続最適化）が実用化した。Skills/Hooks/MCPの設定を見直し、より積極的に自動化できるワークフローがないか棚卸しする好機。caveman skill（トークン75%削減）やContext Mode（98%圧縮）の導入でさらに効率化可能。

  4. MID
