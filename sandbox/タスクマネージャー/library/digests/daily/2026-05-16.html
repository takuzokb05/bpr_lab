Claude Code・エコシステム

CLAUDE.md運用の実践知

5月16日はCLAUDE.md管理に関する記事が集中した。TECHSY.ioの「9ルール」は即使える内容で、特に「200行を超えたらSkillに分離する」というルールが各所で繰り返し言及された。長すぎるCLAUDE.mdはモデルが選択的に指示を無視し始めるという警告は重要だ。「同じ指示を毎回プロンプトに打つなら、それはCLAUDE.mdかSkillに属する」という視点は、ヘビーユーザーが陥りがちな複雑化を防ぐ指針になる。[^1]

チーム規模での構成事例も公開された。30スキル・複数MCP・Self-Learning Hookの組み合わせで、セッション終了時にClaude自身が学んだことをCLAUDE.mdへ自動追記する仕組みが注目された。3ヶ月運用でCLAUDE.mdが600行超になり定期トリミングが必要になったという知見も共有されており、「育てすぎ」への備えが実務上の課題として浮上している。[^2]

Claude Code安全設定

「Claude Codeはファイルを書き換え・コマンドを実行・外部通信できる」という基本認識から出発した安全設定5項目スレッドが拡散した。`.env`ファイルをdenyリストと`.gitignore`の両方で二重保護する手順は、APIキー漏洩リスクへの具体的な対処として有効だ。片方だけでは不十分という指摘は見落としやすいポイントだ。MCPサーバーについても、GitHubスター数は安全性の保証にならないとして導入前に著者・ソース・依存ライブラリを5分確認する手順が推奨された。[^3][^4]

プラグイン・エコシステムの成熟

9,000超のClaude Codeプラグインが存在するが、production-readyなのは50〜100本程度と試算される。MCP・Skills・Hooksの使い分けの整理も進んでいる。MCPは外部システム接続、Skillsは繰り返しワークフローのスラッシュコマンド化、Hooksはイベント駆動で実行を「保証」する仕組み（Skillsと違いモデルが無視できない）。品質保証やセキュリティチェックにはHooksが適切だという整理が共有された。Claude Code+Codex役割分担でのオーケストレーション課題（永続状態・承認ゲート・失敗ルーティング）が次の実務テーマになっている。[^5][^6]

AIニュース・業界動向

Anthropic 6月15日課金変更（一次ソース確認済み）

Anthropicが2026-05-14に正式発表したAgent SDK課金分離は複数メディアで事実確認が取れている。6月15日以降、`claude -p`・Claude Agent SDK・OpenClaw等サードパーティエージェントの利用は通常の利用枠ではなく専用月次クレジットから消費される。クレジット上限はPro $20・Max 5x $100・Max 20x $200で、翌月繰り越し不可、超過後はAPI直接課金へ移行する。[^7]

GitHub Actions上でのclaude -p実行も同一プールを消費する点が見落とされやすい。Max 20x（$200クレジット）でOpus 4.7を使うと月間約2.2Mトークン相当しかない。CI/CD大量実行の場合はモデルをHaiku 4.5へダウングレードするか、プロンプトキャッシュ率を最大化する対策が現実的だ。claude.ai設定画面でAgent SDK利用量を今のうちに確認しておくことを強く推奨する。[^8]

Anthropicエコシステムの拡張

Claude for Small Business（Claude Coworkへの統合）が発表され、会計・請求・営業など15種類の業務ワークフローとQuickBooks MCP連携を提供する。Claude for Legalも12専門プラグインと20以上のツール連携で法律業界に本格参入し、ウェビナーに2万人の弁護士が参加したと報告されている。Anthropicのエンタープライズ市場シェアは過去12ヶ月で4倍になりB2BではOpenAIを逆転したとするデータも出回っている（未確認）。日本政府とメガバンクがClaude Mythosを使ったサイバー防御網を構築するという報道もあり（未確認）、公務員の立場でも注視したい動きだ。[^9]

Routines機能（スケジュール・GitHub event・API callでクラウドエージェントを定型起動）がClaude Code on the webで利用可能になった。Claude Platform Updatesに関しては、Opus 4.7のGA・fast modeデフォルト化は確認済みだが、Max/Team Premiumへの適用範囲は出典記事のみの主張で部分確認にとどまる。

EU AI Act期限の正確な情報

EU AI Actの規制変更について一部記事が「2026年8月2日がフルenforcement日」と報じているがこれは誤り。一次ソース確認済みの正確な情報：高リスクAI単独システムの期限→2027年12月2日に延長、AI組込み製品→2028年8月まで延長、AI生成コンテンツ透明性要件→2026年12月2日。8月2日は禁止用途（社会的スコアリング等）の施行日であり混同注意。日本のAIエージェントガイドラインv1.2（2026年3月31日）では、AIエージェントが初めて明記されHuman-in-the-Loopが必須化された点は、組織内でAIエージェントを展開する際のコンプライアンス要件として確認が必要だ。[^10]
