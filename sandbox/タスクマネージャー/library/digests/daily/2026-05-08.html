Anthropic Managed Agentsに「Dreaming（失敗から自律学習）」「Outcomes（目標定義→反復実行）」「Multi-agent Orchestration」の3機能が追加された（一次ソース確認済み）。AWS MCP ServerがGAとなり`call_aws`1コマンドでAWS全APIがClaude Codeから直接実行可能になった（一次ソース確認済み）。Claude CodeチームメンバーがHTML出力のMarkdown超の優位性を発信し「早期エージェント時代はMarkdown、今はHTML」という議論が加速している。

Claude Code・エコシステム

Skills vs Hooks ── 使い分けの正規理解が普及

Claude CodeのSkillsとHooksは発動原理が対照的だ。Skillsは「descriptionをClaudeが読んで自律判断する引き込み型」、Hooksは「ライフサイクルポイントで自動実行されClaudeは存在を認識しない押し付け型」。MindStudioとTECHSYのガイドが同日公開されたことで「コード品質の最後の砦はStop Hook、繰り返し作業はSkill」という実践的使い分け指針が定着しつつある。

Qiitaの実践ガイドでは21個のスキルを運用した知見として、descriptionフィールドの動詞始まり・具体的記述がClaudeの自動判断精度に直結し、スキルが増えすぎたらdescriptionが曖昧なものを統合するリファクタリングが必要だと報告されている。

AWS MCP Server GA ＆ 35 MCP必須サーバー（一次ソース確認済み）

AWS MCP ServerがGAとなり、`call_aws`1コマンドで15,000以上のAWS API操作に対応した。CloudFormation・EC2・S3などの複雑なインフラ操作を自然言語で直接実行できる。IAMベースのガードレール・CloudWatch・CloudTrail統合あり、追加料金なし（AWSリソース料金のみ）。MCPは2026年3月時点で9700万インストールを突破しており、コミュニティでは「35 MCP必須サーバー」リストが注目された。筆頭はfilesystem・github・gitlabの3サーバー。Qiitaの総合まとめではSEO記事作業時間を90分→15分に短縮した事例も報告されており、セキュリティ上の注意点としてMCP Tool Poisoning（悪意あるサーバーが隠しコマンドを埋め込む攻撃）への対策も解説されている。

Claude Code v2.1.136 リリース

`settings.autoMode.hard_deny`が追加され、Auto Modeの動作をユーザー許可に関わらず強制ブロックできるルール設定が可能になった。複数MCPサーバー同時OAuth更新時のトークン消失バグも修正され、エンタープライズ環境で毎日再認証が必要だった問題が解消。WSL2のクリップボード修正も含まれる。

Managed Agents 3新機能（一次ソース確認済み）

AnthropicがManaged Agentsに3機能を追加。Dreaming（Research Preview）は非稼働時間にエージェントが過去セッションを振り返り失敗パターンを学習し次回以降に反映する仕組みで、Harvey社では完了率6倍、Wisedocsでは50%の時間削減を達成。Outcomes（Public Beta）は「成功とはどういう状態か」をエージェントに定義させ目標達成まで反復実行する機能。Multi-agent Orchestration（Public Beta）はリードエージェントがタスクを分割し専門エージェントに委譲する設計で、NetflixはHundredsのビルドを並列処理している。DreamingとOutcomesを組み合わせることでエージェントの自律改善ループが完成する。

Claude Agent SDKとManaged Agentsの使い分けも整理された。「インタラクティブな開発作業→Claude Code、コードから自動実行→Agent SDK、長期エージェント運用→Managed Agents」という3分類が定着しつつある。エージェントフレームワーク比較ではMCP統合深度でClaude Agent SDKが最深（AnthropicがMCPを共同設計したため）、LangGraphは複雑なステートマシン向け、AWS Strands（2026年5月公開）はBedrock統合が強みと評価されている。
