## Claude Code・エコシステム

### 6/15課金変更：`-p`・Agent SDKがAPI従量課金へ

Claude CodeのインタラクティブなCLI利用とCoworkはサブスクのまま影響なし。しかし**6月15日**から
`claude -p`（非インタラクティブ）・Claude Agent SDK・Claude Code GitHub Actions・
サードパーティAgent SDKアプリが従量課金（API料金）に移行する。$200/月のMax Planユーザーも
programmatic usageには専用料金が発生するため、CI/CDや自動化スクリプトを組んでいる場合は
コスト構造の再計算が必要だ[^1][^2]。

なお`--prefill`という隠しオプションが`-p`の代替として機能すると報告されており、
@orca_buildプロジェクトでの数ヶ月の本番実績もある[^3]。公式ドキュメント外の情報のため
動作検証は必須だが、移行の緩衝策として覚えておきたい。

### Claude Code Routines：PCレス・クラウド自動実行

Claude Code Routinesが発表された。Anthropicのクラウドがプロンプトを処理するため、
ローカルPCの起動不要。トリガーは「定時実行」「APIコール」「GitHubイベント連動」の3種類で、
マルチエージェント並列オーケストレーションもRoutines内で動作する。Claude Code on the Webから設定でき、
毎晩のデータ集計→レポート生成→Slack通知という全自動パイプラインの構築が可能になる[^4][^5]。

### サブエージェントパターンが本格普及フェーズへ

サブエージェントが単なる実験から本番手法へと昇格しつつある[^6][^7][^8]。

なお公式ドキュメントの薄さが普及の壁との指摘も複数あり、コミュニティ事例への依存度は高い[^9]。

### MCPコスト・セキュリティの実測値

MCPサーバーは接続しているだけで1サーバーあたり2,000〜5,000トークンのスキーマベースラインを消費する。
単発スクリプトにはcurl/jqをインラインで書き、MCPはステートフルアクセスが本当に必要なワークフローに
限定するのが費用対効果の高い使い方だ[^10]。

セキュリティ面ではMCPのOAuthがクライアント↔MCPサーバー間のみを保護するスコープの狭さが指摘されており、
Googleトークンをリレーすると「confused deputy」攻撃と権限昇格リスクが生まれる[^11]。
