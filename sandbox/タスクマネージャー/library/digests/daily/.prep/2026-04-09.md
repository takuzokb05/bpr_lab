## Claude Code・エコシステム

### アーキテクチャ全体像：4レイヤーの使い分けが確立

今週はClaude Codeの拡張アーキテクチャを体系的に解説する記事が複数出揃い、理解の共通言語が整ってきた印象がある。[alexop.devの技術記事][1]はMCP・Skills・Subagents・Hooksの4レイヤーを「ライフサイクルイベント自動化（Hooks）→外部ツール接続（MCP）→再利用可能ワークフロー（Skills）→並列実行（Subagents）」という役割分担で整理している。実務では「2〜3 MCPサーバー＋数個カスタムSkill」が最もコスト効率が高いという結論は、小さく試して積み上げる運用スタイルに合致する。

[Towards AIの入門記事][3]はHooksの全12イベントにコード例を付けた網羅的な解説で、初めて設定する際の参照先として有用。[trenseeの上級パターン記事][2]はSkill Graph（複数Skillのトリガーチェーン）による「コード変更→テスト実行→PR作成」の完全自動フローを実装例として示しており、Claude Code 2.1.x系での動作確認済みとしている。

### 並列開発と組み込みスキル

サブエージェントを使った並列開発の実践ガイド[^4]では、git worktree分離による安全な並列実行と競合回避策を詳説。「フロントエンド＋バックエンド同時実装」パターンで4〜6倍の開発速度向上を報告している。

組み込みスキルの`/simplify`と`/batch`について[MindStudio公式ブログ][5]が詳解。`/batch`の内部フロー（コードベース調査→計画提示→承認→並列実行→テスト→PR作成）は「全ファイルの型注釈追加」「APIエンドポイント一括リファクタリング」等の大規模変更に有効。Anthropic公式ドキュメントに基づく信頼性の高い解説とされている。

### 設計パターンとCLAUDE.md運用

Anthropic公式33ページのAgent Skills設計ガイドを凝縮した[記事][6]では、コンテキスト消費最小化に最も効率的なパターンとして「ルーティングテーブル型（SKILL.md 100行以内で13参照ファイル管理）」が推奨されている。モデル指定型（`model: haiku`でコスト削減）も実用的。

チーム運用視点では[homula.jpの日本語ガイド][7]がCLAUDE.md・AGENTS.mdの責任分担パターン（プロジェクト共通＋個人用~/.claude/CLAUDE.md）と「200行超えたら分割」という具体的な管理基準を提示。MCP対応ツール一覧も付属している。

### v2.1.7アップデートと Hooks全量リファレンス

[一創による v2.1.7 解説][8]によると、主な変更点は新フックイベント追加（TeammateIdle・TaskCompleted・PermissionDenied）と--resumeの高速化。特に`PermissionDenied`フックで`{retry: true}`を返してモデルに再試行を指示できる新機能は、ヘッドレス運用の細かい制御に直結する。

[pixelmojo.ioの全12イベントリファレンス][9]は、各イベントの引数・戻り値を表形式で整理したもの。本番CI/CDでの「PreToolUseでgit状態確認→Stopで自動lint実行→PostCompactで圧縮後コンテキスト検証」パイプラインは参考になる。

### MCPエコシステムとAgent SDK

MCP v2.1の新仕様[^11]では**Server Cards**（`/.well-known/mcp.json`経由でCapabilities・認証要件・利用可能ツールを自動検出）が追加。累計9700万インストールを超え、登録サーバー数も1万件を突破（週次新規300件超）[^12]。AWS 54種・Salesforce・OktaなどエンタープライズのMCP採用が本格化している。

Claude Agent SDK（旧Claude Code SDK）の[深掘り記事][13]では、ライブラリとしてClaude Codeを使う際の組み込みツール・MCP統合・18フックイベントの活用法を解説。`context-1m-2025-08-07`ベータが2026年4月30日廃止予定で、Sonnet/Opus 4.6での1Mコンテキスト標準移行への対応が必要。[Claude API本番運用ガイド][10]はプロンプトキャッシュで最大90%コスト削減、max_tokens 300K対応などプロダクション知見も整理されている。

## AIニュース・業界動向

### LG EXAONE 4.5 正式発表（一次ソース確認済み）

LG AI Researchが2026年4月9日、33B VLMの「EXAONE 4.5」を[正式発表][14]。STEMベンチマーク平均スコア77.3でGPT-5-mini（73.5）とClaude 4.5 Sonnet（74.6）を上回ると主張（LG自社計測値）。Hybrid Attention architectureとマルチトークン予測による高速推論を実現し、6言語対応（日本語含む）。**ただしライセンスは非商用（研究・学術利用のみ）のため、プロダクション利用は不可**。FX自動取引システムへの組み込みには利用できない点に注意。

### Meta Muse Spark — オープンソース路線から転換（一次ソース確認済み）

Metaが2026年4月8日、Llama以降初のプロプライエタリモデル「Muse Spark」を[発表][15]（内部コードネーム：Avocado）。Alexandr Wang率いるMeta Superintelligence Labsが主導し、マルチモーダル推論・ヘルスケア・エージェントタスクで競合に対抗する。同日Zuckerberg氏がLlama 5のオープンソース継続も発表し、クローズドとオープンの二本立て戦略へ転換した。

### Claude Mythos（Capybara）リーク情報

[The Decoder報道][16]によると、3月27日のAnthropicのCMS設定ミスで未発表モデル「Claude Mythos」（内部コード名Capybara）の存在が漏洩。ソフトウェアコーディング・学術推論・サイバーセキュリティでOpus 4.6を「劇的に上回るスコア」とAnthropicは「step change」と表現。現時点では約50の限定パートナー組織のみアクセス可能な「Project Glasswing」ゲートプログラムで提供中。**fact_checkは未実施のため情報の信頼性は暫定扱い**。

### 米国AI州法：州vs連邦の権限争いが本格化（一次ソース確認済み）

[Troutman Pepperの週次レポート][17]によると、Oregon（SB 1546・チャットボット開示義務）・Idaho（S 1297）・Tennessee（医療AI・AIが精神科医資格詐称を禁止）の各法律が相次いで署名成立（2027-01-01施行）。[Q1 2026規制総括][18]ではトランプ政権の「ライトタッチ」AI政策フレームワークとDOJ AI Litigation Task Forceによる違憲州法挑戦の動きが並行している。公務員の立場では今後の国内AI利活用ガイドライン策定に向けて米国の動向を注視する価値がある。
