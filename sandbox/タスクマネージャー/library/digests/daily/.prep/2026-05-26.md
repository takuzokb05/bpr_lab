## Claude Code・エコシステム

### Agent SDK課金分離：6月15日のXデー（一次ソース確認済み）

Anthropicが5月13日に発表した課金分離ポリシーの実装まで約3週間となり、実務影響分析が集中した。6月15日以降、`claude -p`・Claude Code GitHub Actions・Agent SDK経由の呼び出しはサブスクリプション枠から切り離され、Pro $20/月・Max 5x $100/月・Max 20x $200/月の独立クレジットプールへ移行する（ロールオーバーなし）。軽量ワークロードで実効12倍、重量ワークロードで最大175倍の価格上昇との試算もある。[^1]

**インタラクティブな手動Claude Code利用（ターミナル対話）はサブスクリプション枠のまま変わらない**点が鍵だ。自動化スクリプト・GitHub Actionsとして呼ぶ「非インタラクティブ利用」だけが影響を受ける。対策として推奨されているのは①直接APIキーへの切り替え、②Haikuへのダウングレード、③バッチ処理化による呼び出し削減、④予算上限の設定の4点。6月15日前のクレジット申請手続きも公式推奨されている。[^2][^3]

### MicrosoftのClaude Code大規模縮小（一次ソース確認済み）

MicrosoftがExperiences+Devices部門を中心にClaude Codeライセンスを6月30日までに大部分キャンセルし、GitHub Copilot CLIへ移行する方針が複数メディアで確認された。ただし完全廃止ではなく、Copilot CLI内部ではClaudeモデルが継続利用される構造でAnthropicへの支払い自体は変わらないとの見方もある。[^4] エージェント型AIは通常の生成AIの数倍トークンを消費し、Uberではわずか4ヶ月で年間AI予算を使い切った例も報告されており、コスト急増が業界全体の共通課題として顕在化してきた。

### MCPエコシステム：Google Cloud GA・Stainless買収

Google CloudがNext '26でBigQuery・Spanner・Cloud SQL・Vertex AIなど50以上のマネージドMCPサーバーをGA公開した（一次ソース確認済み）[^5]。GCP既存IAMで認証が完結するため、インフラ管理コストがゼロになる。Claude Code・Cursor等から即座に接続可能。

AnthropicがStainless（SDK・MCPサーバーツール専業）を約$300Mで買収した。StainlessはAnthropicのSDK全体を初期から支えてきた企業で、MCPエコシステムの内製化が進む。同時期にMCP Server Registry v2.0（2,500以上の検証済みコミュニティサーバー）もリリースされており、Anthropic主導のMCP標準化とLinux Foundation推進のA2A（Agent-to-Agent Protocol）の覇権争いが2026年のAI基盤の焦点になっている。[^6][^7]

### Claude Skills・Anthropic Academyの充実

70本以上のClaude Skillsを実際に構築・テストした実践ガイドによると、GitHub PR自動化・Dockerfileレビュー・/code-review・/loopが実用性で高評価を得た。Skillsはトリガー時のみロードされる軽量マークダウン命令セットで、CLAUDE.md・supporting filesとの組み合わせで専用アシスタントを構築できる。[^8]

Anthropicが Academy経由で17コースを無料公開している。「Building with the Claude API」は84レッスン・8時間以上の動画で修了証付き。組織内でClaude展開を進める際の研修資材として活用できる構成だ。

### エージェント封じ込め設計

Anthropicのエンジニアリングブログが「エージェントへのアクセス権限はその能力の成長に合わせて進化させるべき」とする内容を公開。Claude Code・Claude Coworkでのサンドボックス設計（破壊的アクションのスコープを限定する仕組み）の詳細が解説されている。非技術者でも「CLAUDE.mdに承認なしに変更を加えるな」と記述するだけで同様の安全策を実装できるという事例紹介も複数のXポストで共有された。

## AIトレーディング

**TradingAgents**（arxiv:2412.20138）は証券会社の組織構造を模倣したマルチエージェントFXフレームワーク。ファンダメンタル・センチメント・テクニカルの各分析エージェントがBull/Bear研究者とリスク管理チームと議論し、複数リスクプロファイルのトレーダーエージェントが最終判断する設計。LangGraphベースでAnthropicを含む主要LLM全対応。単一モデルとの比較でCumulative Returns・Sharpe Ratio・Maximum Drawdownの全指標で優位という実験結果が報告されており、既存の自動売買EAとのアーキテクチャ比較の参考になる。[^9]

**EA Agent Studio**（今週末公開予定）がアプリ上からバックテストできる機能を追加。EAをサーバー側のMT5で処理し、AIがパフォーマンスを判断・分析する仕組みで、スマホでEA作成後にすぐ検証できる設計。現行の開発→検証サイクルを短縮できるか注目したい。
