## Claude Code・エコシステム

### SpaceX Colossus 1提携：使用制限の壁がなくなる日（一次ソース確認済み）

本日最大のニュースは、AnthropicがSpaceXのColossus 1データセンター（NVIDIA GPU 22万基・300MW超）と契約し、
Claude Code 5時間レート制限をPro/Max/Team/Enterprise全プランで**即日2倍**に拡張したことだ。
Pro/Maxのピーク時間帯制限も完全廃止。Tier 1のClaude Opus APIは入力トークン/分が1500%増・出力900%増と桁が違う引き上げになった。
「すぐ上限に当たる」という長年の不満に対する直接的な回答であり、エンタープライズCI/CD統合が一気に現実的になる。
Anthropicは将来的に「複数ギガワット規模の軌道上コンピュート」への展開を示唆しており、規模感はもはや国家プロジェクトに近い。

### Code w/ Claude 2026（SF）：Dreaming・Outcomes・CI自動修正が登場（一次ソース確認済み）

同日SF開催のAnthropicの年次開発者イベントでは、Claude Code・Claude Managed Agentsの新機能が大量発表された。

### MCPがデファクト標準に：AWS GA・Grok Connectors・Splunk・Mistralが同日参入

本日だけで複数のMCP関連ニュースが重なった。**AWS MCP Server**がGAになり、
IAMガードレール・CloudWatch・CloudTrailを使ったセキュアかつ監査可能なAWSサービス操作が追加費用なしで利用可能になった。
**Grok Connectors**（xAI）もMCPを採用、**Splunk**はMCPサーバーとホスティングモデルを同時リリース、
**Mistral Workflows**もMCP採用によりAnthropicが設定した標準に従う形となった。

開発者の洞察として注目は「MCPサーバーよりCLIツールの方がエージェントネイティブ」という指摘：
MCPはスキーマ全体をコンテキストに乗せるが、CLIはエージェントが`--help`でオンデマンドに能力を発見できる。
GISツール（QGIS）の実践でも、MCPよりPythonスクリプト生成→コンソール実行の方が効率的と確認されている。

## AIトレーディング

FX・量化運用の視点から本日の注目ニュースをまとめる。

**Paris Trading Expo**でAI（5モデル）対プロのディスクリショナリートレーダーのライブ対決が開催。
「AIが着実に追いついている」という評価は、ルールベース運用からAIハイブリッドへの移行が現実的になってきた証拠だ。

ゴールドマン・サックスが2017年に600人のトレーダーを**わずか2人**に削減し、200人のITエンジニアが自動取引アルゴリズムを運用している事実は、
AI取引との競争について重要な視座を与える。介入も「押し目」として処理されるロジックの世界で戦っているということだ。

**AI trading agent**がTrading SMART Benchmarkで80/100（グレードA・172エントリー中54位）を記録。
スコア内訳はデータ14/インテリジェンス13/分析17/決断18/リスク18で、リスク管理とデシジョン領域の評価が高い。

**Polly Nulla**（ローカルAI使用・自律型取引ハーネス）は「ルールなしで$0.01株を買い始めた」という逸話が示す通り、
制約設計がAI取引エージェントの最重要課題だ。現在FreqtradeでオーケストレーターとしてClaudeを使っているユーザー報告とも符合する。

MT5対応の自己ホスト型AI量化システム**QuantDinger**（Docker Compose一発・IBKR統合対応）もGitHubで公開されており、
小さく試してローカルで検証してから本番導入するアプローチに向いている。
