## Claude Code・エコシステム

### Routines: クラウド常駐の自律エージェント

Anthropicが2026年4月14日に公式発表したClaude Code「Routines」（**一次ソース確認済み**）は、AIエージェントの運用パラダイムを根本から変える機能だ。スケジュール・API・GitHub Eventsの3トリガーに対応し、**ローカルPCがオフラインでもAnthropicクラウド上で継続実行**される。夜間バックログのトリアージ、デプロイ後スモークテスト、PR自動レビューが代表的なユースケースだ。実行上限はPro=5件/日・Max=15件/日・Team/Enterprise=25件/日。[^1]

The Registerは「賢いcronジョブ」と批評的に表現するが、Qiitaの独自分析が指摘するように「定期実行ではなく常駐実行レイヤー」という設計思想は従来のcronと本質的に異なる——文脈理解・適応的実行・エラー回復能力を持つ点が決定的な差だ。Pro=5件/日という上限が実用上のボトルネックになりうる点は、Max以上のプラン検討材料になる。[^4][^10]

同日リリースのデスクトップ刷新も重要だ。統合ターミナル（外部ターミナル不要）・高速diffビューア・アプリ内ファイルエディタ・再配置可能ペインにより、Claude Codeがターミナル専用ツールからオールインワンのエージェントコーディング環境へ進化した転換点となる。[^3]

### サブエージェント・ワークフロー設計の実践知

公式ブログがサブエージェント活用ガイドを公開した。並列化可能タスク（バルクリファクタ・マイグレーション・テスト生成）に最適で、密結合な順次ワークフローには不向きという原則が示される。重要な制約として**サブエージェントからSkillツールを呼び出せない**点（GitHub issue #38719で改善議論中）があり、設計時の考慮が必要だ。[^2]

MindStudioが公開した5パターンガイドでは、順次→条件分岐→並列→階層型（オーケストレーター＋ワーカー）→完全自律への設計段階を整理し、「2026年はプロンプトよりコンテキストアーキテクチャが重要」と指摘する。Agent Teamsによる共有タスクリスト並列実行は実験的機能として利用可能で、設計コストとトークンコスト増加のトレードオフを考慮したい。[^5][^6]

### 公式スキル17個とCLAUDE.md設計

Claude Code公式スキル17個（パワポ・PDF・Excel・スライド生成を含む）の全解説記事が登場し、チームへの普及ガイドとして活用できる。CLAUDE.md設計は規模別5パターン（Minimal 50行以下〜Enterprise）で整理されており、rulesサブディレクトリ分離によるProgressive Disclosure実装の参考になる。[^11][^12]

### Claude Mythos PreviewとAzure MCP 2.0

Glasswing限定公開のClaude Mythos PreviewはSWEbench Verified 93.9%・CyberGym 83.1%を達成（**一次ソース確認済み**、ただしGlasswingメンバーの一部表記に差異あり——正確にはGoogle・JPMorgan Chase・Nvidiaが含まれる）。英国AISIが独立評価を実施し、その結果が限定公開判断に影響したとされる。政府機関によるフロンティアAIの事前安全評価という前例ない規制モデルの事例として注目すべきだ。[^13][^14]

Azure MCP Server 2.0安定版（4月10日リリース）はMicrosoftがエンタープライズMCPエコシステムを本格整備したことを示す。Claude CodeやCursorからAzureサービスへのMCP接続が安定化し、エンタープライズSSO・監査ログも強化。AWSに続く主要クラウドのMCP標準化加速を示す出来事だ。[^15]

## AIニュース・業界動向

### Stanford AI Index 2026: 業界の転換点を数字で捉える

Stanford HAIとIEEE Spectrumが示す2026年AI業界の現状は衝撃的な変容を記録している。SWEbenchスコアが1年で60%→約100%へ急上昇しベンチマーク飽和が加速。米中パフォーマンス差は9.26%→1.70%に急縮小し、オープンソースモデルが独自モデルに匹敵する水準に達した。一方でFoundation Model透明性指数は58→40に低下——能力向上と透明性低下が同時進行している。エージェントAIが本番環境でスケールしている事実も確認された。[^17][^18]

### 国産LLM「LLM-jp-4」: 日本語でGPT-4oを超える

NII（国立情報学研究所）が4月3日に公式公開した「LLM-jp-4」（**一次ソース確認済み**）は12兆トークン学習の8Bモデルと32B-A3B（MoEアーキテクチャ）をオープンソースで提供する。日本語MT-Bench 7.54（8B）/7.82（32B-A3B）はGPT-4oの7.29を上回り、英語でも同等以上を達成。2026年度中に332B-A31B（MoE超大規模）の公開も予定されており、国産LLM自立化の歴史的マイルストーンとなった。[^19]

### AI規制の動向: 経産省・EU・米国

経産省が4月9日に公表した「AI民事責任手引き」v1.0は、補助型AI（判断支援）と代替型AI（判断代替）の2類型で不法行為法・製造物責任法上の責任分界を整理した実務的ガイドライン。組織でのAI導入における法的予測可能性の観点から必読の一次情報だ。[^20]

EU AI Act高リスクシステム要件は2026年8月施行（2027年12月への延期提案あり）。米国ではトランプ政権が州法を優越する国家AIフレームワークを提案中。日米欧でAI規制の方向性が分岐しつつあり、企業コンプライアンス戦略の設計が急務となっている。[^21]

## AIトレーディング

2026年FX AIトレーディングの主流は、ルールベースから**目標指向型自律エージェント**への移行だ。AgentTradeX（LangChain+GPT-4o、ボラティリティ相場で25%超過収益）・TradingAgents（マルチエージェントLLM）・LLM-TradeBot（DeepSeek）の比較が参考になる。MT5統合方法はPython APIブリッジ・MQL5+LLM APIコール・MetaQuotes Cloudの3方式。評価指標はシャープレシオ・ソルティノレシオ・最大ドローダウンが基準で、Routinesをスモークテスト自動化に転用する設計も示唆される。[^16]
