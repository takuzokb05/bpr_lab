## Claude Code・エコシステム

### Anthropic、Stainlessを約3億ドルで買収——SDK・MCP基盤を内製化（一次ソース確認済み）

Anthropicが2026-05-18、SDK・MCPサーバー生成ツーリング企業Stainlessを約3億ドルで買収した。StainlessはTypeScript・Python・Go・Java・KotlinのSDKをAPIスペックから自動生成してきた企業で、OpenAI・Google・Cloudflare・MetaもSDK生成に依存していた。今回の買収でAnthropicはMCPプロトコル（オープン標準）とその主流実装ツールチェーンを同時に掌握したことになる。競合他社は既存SDKを保持するが自動同期機能を失い、Anthropicが開発者インフラの垂直統合で主導権を握った格好だ。なお、ホステッドプロダクトは廃止予定（[WesRoth](https://x.com/WesRoth/status/2056585641937207544)）。

これに関連して、Claude Managed Agentsに「Self-hosted Sandboxes（public beta）」と「MCP Tunnels（research preview）」が追加された。MCPトンネルは`tunnel.anthropic.com`経由でセキュアなチャネルを提供し、内部サーバーをパブリックインターネットに公開せずにエージェントからアクセスできる。法務・セキュリティ部門がネックだったエンタープライズ採用の障壁を正面突破する機能として評価できる（[LifeMakersCom](https://x.com/LifeMakersCom/status/2056838580786581888)）。

金融業界では、CAISがAnthropicとのMCPサーバー統合を発表。アドバイザーがClaudeの画面から直接ファンドデータ・パフォーマンス分析にアクセスできるようになった（[CAISgroup](https://x.com/CAISgroup/status/2056724059312779455)）。

### Claude Codeの実践知見：Plan-first・Context7・並列worktree

**Planモード活用が主流化**：大規模プロジェクトへの機能追加前に「この実装で触らないファイル一覧」をPlanモードで出力させ、実装範囲を確定してから着手するワークフローが海外開発者コミュニティで広がっている。Claude Codeがデフォルトで「エンジニアリングらしい解」を選びがちな傾向に対し、plan-firstワークフローで不要なDBテーブル3件・キャッシュ層1件を事前排除した実績も報告された（[ura_unico](https://x.com/ura_unico/status/2056843603897667782)、[dmshirochenko](https://x.com/dmshirochenko/status/2056844404254720429)）。

**ツール連携の充実**：Venice MCP Serverが31ツール（チャット・画像・動画・音声・Web検索）を搭載して正式リリース。Context7はAPIドキュメントをクエリ時点でリアルタイム注入する仕組みでGitHub 55,000スター（[agentxagi](https://x.com/agentxagi/status/2056825328073425152)）。ForgeAILab/forgeはRust製・MITライセンスのエージェントワークフローエンジンで作成3日で39スター、タスクごとに独立したgit worktreeを割り当てCI＋レビューゲートを義務化する（[BeauJohnson89](https://x.com/BeauJohnson89/status/2056843925101391957)）。

**Skills・Hooks・MCP 6ヶ月実運用報告**：Freqtrade（暗号通貨アルゴ取引）スタックでClaude Code+skills+hooks+MCPを6ヶ月間実運用した報告によると、「10倍の生産性向上はモデル単体ではなく永続メモリ＋適切なハーネスから生まれる」。足場が仕事にフィットした段階でレバレッジが本物になるという観察はFX自動取引運用にも示唆深い（[lliu54827](https://x.com/lliu54827/status/2056832250860413419)）。

**Claude Code Routines**（リサーチプレビュー）の公式ドキュメントが整備された。スケジュール・Webhook・GitHubイベントの3種トリガーで無人クラウドエージェントを実行する仕組みで、PRコメント評価など「判断を要する作業」をGitHub Actionsと分担できる（[公式ドキュメント](https://code.claude.com/docs/en/routines)）。

## AIトレーディング

### MossAIの2層アーキテクチャ：AI推論と決定論的執行の分離

MossAIが採用する「AIが直接取引を執行しない」設計が注目を集めた。AIは「戦略作成・パラメータ最適化レイヤー」、執行は「数学的ルール+事前定義リスク限度の決定論的エンジン」に分離することでハルシネーションによる危険なトレードを防止する。LLMベース取引システム設計の参照パターンとして価値がある（[0xJoell](https://x.com/0xJoell/status/2056843908949369237)）。

### SMC AI自動売買稼働ログ（EURUSD/USDJPY M15 / 5日目）

@fxmethodologyによるSMC+AI自動売買の検証ログが公開された。EURUSD・USDJPYのM15足でLiquidity Sweep反発・CHoCH反転をシグナルに、ML確率（p=0.53〜0.55）・期待値（EV=+0.39〜+1.39）でエントリー。稼働5日・26トレード・累計損益-11,334円・勝率89%（ほぼ全SL決済で累計損失中）。検証フェーズにおけるリアルな損益推移として、FX自動取引を運用する立場では手法設計の参考になる（[fxmethodology](https://x.com/fxmethodology/status/2056732924196884722)）。

TradingAgentsフレームワーク（UCLA+MIT・LangGraph基盤・オープンソース）のレビューも公開。7役割AIエージェント協調型で3ヶ月バックテストで6〜25%改善報告があるが、Sharpe比5〜8は「統計的異常」と著者が明記、ライブ取引未実施。v0.2.0でClaude 4.xを含むマルチプロバイダー対応（[Medium](https://medium.com/data-science-in-your-pocket/tradingagents-best-ai-agent-for-financial-trading-c0bd6e790bf0)）。
