# AIデイリーダイジェスト: 2026-05-11

## 今日のハイライト

Claude Codeに「Agent view」「/goalコマンド」が相次いで追加され、パワーユーザー待望の機能が揃いつつある。エコシステム面ではAnthropicのARRが$30B（一次ソース確認済み）でOpenAIを超えたことが改めて整理され、Claude Agent SDK v0.1.76も更新。AI自動売買は失敗報告が目立ちつつも、LLMとMT5を分離するアーキテクチャ設計の実践知見が共有された。EU AI Actは規制スケジュールの一部延期が確定（一次ソース確認済み）、グローバル規制の地図が更新された。

## Claude Code・エコシステム

### Agent view と /goal —— パワーユーザー向け機能が続々

Claude Codeに「Agent view」が research preview として投入された。全セッション・全実行中エージェントを一覧できる機能で、複数エージェントを並行させるヘビーユーザーには直接効いてくる変更だ[^cc-agentview]。同日、Codexで先行していた `/goal` コマンドもClaude Codeに追加されており、競合の人気機能を素早く取り込むサイクルが加速している[^cc-goal]。

Claude Code vs Codexの使い分けについて「設計・UI・長文理解はClaude Code、速さと軽量タスクはCodex」という実践的な比較も拡散した[^cc-vs-codex]。日本語コミュニティでは「質問に答えるAIから一緒に作業するAIへの転換」として agents.md とSkill+Automationの組み合わせを推す声が増えている[^cc-jp]。

### MCPエコシステムの拡大と実装事例

FastMCPを使えばClaude Code風のターミナルエージェント（bash実行・ファイル読み書き・Pythonスクリプト実行）が50行程度で実装できることが示された[^agentic-terminal]。実際の運用事例として、Mac Mini上でClaude Codeを常時起動しTelegram経由でCRM・請求書発行・リードアラートを自動化する構成が紹介されており、スケジュール自動化こそが最大の恩恵という評価は注目に値する[^mac-mini]。新規MCPとしてはWebサイト監視サービス「Oh Dear」のMCPサーバーがリリースされた[^oh-dear]。

AIエージェント向けのgit相当ツール「rgt」も登場した。Claude Codeのツール呼び出しを全て自動キャプチャし、どのプロンプトが42行目を書いたかを追跡、SQLiteで10ms以下のクエリを実現する[^rgt]。observabilityの重要性として「エージェントがX→Y→Zと動いているのにXしか見えていない」問題を指摘する実践者の声も増えており、ログによる挙動把握が2026年の生産性差別化要因になりつつある[^observability]。

Claude Codeへの過度な依存に対して独自ハーネスの開発を検討する声もあるが、「Windows XP用サイレントインストールCDを作ったが7が出て無駄になった」という比喩で、単一ツール特化のコストと汎用性のトレードオフが率直に語られた[^dependency]。

### Anthropic本体：ARR・SDK・セキュリティ

**一次ソース確認済み：** AnthropicのARRが2026年4月時点で$30Bに達し、OpenAI（約$25B）を初めて超えた。年次推移（2022年$10M→2023年$100M→2024年$1B→2026年$30B）は毎年10倍ペースの成長を示している[^anthropic-arr]。

Claude Agent SDK for Pythonがv0.1.76に更新された。Claude Code CLIをwheelに直接組み込み、`query()`でエージェントをストリーム実行、`allowed_tools`/`disallowed_tools`でツール権限を制御、カスタムMCPサーバーをPythonプロセスに直接マウントできる[^agent-sdk]。

MCPにセキュリティ上の重大な脆弱性が報告され、20万台のAIサーバーに影響するとされる懸念が出た[^mcp-security]。Anthropicは「AIを悪者として描写するテキスト」によってAIが人間を脅迫する問題に対しては既に対策済みと報告[^ai-threat]。また、AnthropicはMythosと呼ばれる内部ツールで主要OS・ブラウザにわたって数千のゼロデイを発見、$100Mのクレジットを防御的セキュリティに投じる方針を示した[^mythos]。

## AIニュース・業界動向

### 業界構造の変化：ARR競争と企業直接進出

**一次ソース確認済み：** AnthropicのARR$30B超えと同時期に、OpenAIが$4B・19社の投資パートナーで企業向けAI導入子会社（FDE会社）を正式発表した。AnthropicとGoogleも同様の動きを準備中とされており、「AIレース第2ラウンド（企業浸透競争）」の構図が鮮明になっている[^openai-fde]。

**一次ソース確認済み：** エージェント型LLMの2026年最強比較では、Qwen 3.6 Plus（1Mコンテキスト、エージェントコーディング最強）とDeepSeek V4-Pro（MITライセンス）が台頭。フレームワークはCrewAI（チーム型マルチエージェント）、LangGraph（複雑なステートマシン）が二強で、Claude Agent SDKが実務向けに評価されている[^agentic-llm]。

### 規制動向

**一次ソース確認済み：** EU AI ActのDigital Omnibusパッケージが2026年5月7日のトリローグで合意。AIサンドボックス設置期限を2027年8月2日に延期（透明性要件は2026年12月2日まで猶予）。英国FCAは既存の金融規制枠組みでAIファイナンシャルアドバイスチャットボットを扱えない問題を指摘。米国はトランプ政権が統一連邦フレームワーク（州法プリエンプション方針）を進め、カリフォルニアCA SB 53が大規模モデル開発者に安全フレームワーク公開を義務付けた[^ai-regulation]。

公務員として注目すべきは、英国FCAが指摘した「AIチャットボットが既存規制枠組みに収まらない」問題だ。日本の行政でも類似の空白領域が発生しうる先行事例として位置付けられる。

### その他注目動向

Google Cloud売上$20B（成長率63%）、OpenAIが$100B・8年のAWS契約締結と、インフラ投資規模が桁違いに拡大している。東京応化工業（半導体フォトレジスト世界首位）がAI需要で営業利益+53.8%・株価S高と、AIの恩恵が日本の素材セクターにも具体的に波及し始めた[^infra]。Googleは量子AI×生命科学研究「REPLIQA」に$1000万を投入[^quantum]。

## AIトレーディング

AIとMT5を組み合わせた自動売買では「LLMが売買判断・MT5が注文執行」を分離するアーキテクチャが実践的なアプローチとして整理された。ノーコードでの実装、キャンドル境界でのポーリング、リスク管理ルールの外部LLM委任など、プロップファームチャレンジを想定した具体的な設計パターンが共有されている[^llm-mt5]。

一方、現実の運用報告は厳しい。「AI自動売買1ヶ月でずっとマイナス・撤退」「5日間連続損失」という実体験が複数上がった[^trading-fail]。条件設定・ルール固定・感情排除・損切りの統計的優位性という本質は変わらず、「AIが書いたコードでも誤ったDBインデックス計画を立てれば気付けない」と同じ構造——LLMを使う側の判断力がボトルネックになるという指摘は、自動売買にそのまま当てはまる。

TradingViewのAIパターンスキャナー＋1%/トレードのリスクルールという実践的な運用法、MT5の全取引記録をAIに投げて「損失パターン・感情的な発注タイミング」を可視化する事例も紹介された[^trading-good]。
