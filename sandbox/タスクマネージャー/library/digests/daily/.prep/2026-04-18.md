## Claude Code・エコシステム

### Claude Codeヘビーユーザーたちの「設定公開」が相次ぐ

4ヶ月間毎日Claude Codeを使い続けたエンジニアが詳細なセットアップを公開した[^007]。使用MCPサーバーの構成から、lint自動実行・コミット前テスト・セキュリティチェックを組み込んだカスタムフック、スキルセットの設計思想までを網羅している。同氏が示した原則「MCPは接続性、スキルは方法論、フックは保証」は、3つのレイヤーを整理する上でわかりやすい指針だ。

同日に公開された上級者向け11テクニック記事[^008]はこの考え方をさらに踏み込んで実証する。フック×CLAUDE.md組み合わせがCLAUDE.mdだけでは得られない強制力（70%→100%）を生むこと、CLAUDE.mdは200行を超えると効果が落ちる実測データを示している点は、ヘビーユーザーほど見逃せない知見だ。Claude Code Skillsの体系については、Qiitaで日本語チーム（5人規模）での展開事例を交えた完全ガイドが登場[^019]。個人の最適化から skills-registry の設計・チームオンボーディングへの組み込み方まで一気通貫で整理されている。

### スキル・MCP設計の「公式答え合わせ」

カスタムインストラクションに詰め込みすぎることの弊害を解説した記事[^009]が、設計の整理に役立つ。長い指示はClaudeが一部を無視し始めるという問題に対し、繰り返し手順はスキル化・外部システムアクセスはMCP・プロジェクト規約はCLAUDE.md（簡潔に）・動的データはMCPリソースと使い分ける体系が提示された。Anthropic公式が配布するスキル構築完全ガイドPDF[^010]は、この設計を「公式答え合わせ」する資料として最も信頼性が高い。エンタープライズ向けのバージョン管理・チーム共有設計まで踏み込んでいる。コミュニティキュレーションリスト awesome-claude-code[^002]は、スキル・フック・エージェントの全体像を把握するインデックスとして引き続き参照価値が高い。

### Claude エコシステム：SDK・API・MCPの整備が加速

MCP 2026公式ロードマップ[^001]が示す4優先領域（トランスポートスケーラビリティ・エージェント間通信・ガバナンス成熟・エンタープライズ対応）は、プロトコルがAnthropicの独占からオープン標準へ移行する道筋を明確にした（一次ソース確認済み）。A2Aプロトコルとの統合が進む点は、異プロバイダー間のエージェント連携をどう組み立てるかの設計判断に影響する。

Anthropic API側では4つの新機能がエージェント開発のインフラを整備した[^013]：①サンドボックスでのPythonネイティブ実行、②MCPコネクター（URLを追加するだけで外部ツール即時利用）、③Files API（会話をまたいだファイル参照）、④拡張プロンプトキャッシュ（最大12倍のトークンコスト削減）。Claude Agent SDK 2026ガイド[^011]では、get_context_usage()によるトークン使用量追跡や、プロンプトキャッシュ無効化バグの修正（入力コスト最大12倍削減に直結）といった4月18日リリース分の変更が解説されている。Managed Agentsパブリックベータ[^012]は、エージェントループ・ツール実行・サンドボックス・状態永続性をREST API一本で提供するアーキテクチャであり、「まず触ってみる」スタイルなら5分で動くサンプルから試せる。

## AIニュース・業界動向

### Claude Opus 4.7：「全次元で1段上」の評価が固まる

Claude Opus 4.7（4/16リリース）についてBloomberg[^024]が速報、Latent Space[^004]が技術的詳細を分析した。注目点はMythosの限定公開（約50社）からわずか1週間での一般公開という製品戦略で、フロンティアモデルと広く利用可能なモデルの二層構造を維持するAnthropicの姿勢が明確だ。技術面ではxhighエフォートレベル（highとmaxの中間）追加、タスクバジェットによるエージェントループのトークン推定制御、ビジョン解像度向上（最大2576px）が実用的なアップグレードとなる。Opus 4.6からの移行はドロップイン対応で、価格も据え置き（入力$5/MTok・出力$25/MTok）。ただし包括ガイド[^003]は一次ソース確認済みの情報として「トークナイザー更新により実コストは最大1.35倍増の可能性」を指摘しており、実運用コスト計算時は注意が必要だ（一次ソース確認済み）。

### 4月のLLMリリースラッシュと競合環境

4月は史上最もモデルリリースが多い月との評価も出ている[^014]。Claude Opus 4.7に加え、GPT-5 Turbo（4/7）・Mistral Medium 3（EU AI法コンプライアンスメタデータ内蔵）・GLM-5.1（Zhipu AI・MITライセンス・SWE-Bench ProでOpus 4.6超えを複数ソースが確認[^014]）・Gemma 4・LLM-jp-4（日本語特化）が相次いでいる。OpenAI Agents SDKの大型アップデート[^006]はサンドボックス実行・モデルネイティブハーネスなどClaude Agent SDKと直接競合する機能を追加。Cloudflare Agents Week[^005]ではエッジでのエージェントインフラフルスタック（MCPネイティブサポート含む）が発表された。

### AI規制と社会インフラ

AIコンパニオン規制が米国各州で本格化している[^016]。ワシントン州の開示義務・NYの自傷検出プロトコル義務・メイン州の全面禁止法案と対応が割れており、EU AI Act（8月完全適用）との整合性問題も浮上している。MIT Technology Review[^020]が予測する5大トレンドのうち「規制の二極化（EU厳格化vs米国軽タッチ）」と「AIエネルギー問題の顕在化」は現在進行形で確認できる。NVIDIAとアイダホ国立研究所が原子炉建設期間を数十年から数年に短縮する「Project Prometheus」を立ち上げた[^021]のは、エネルギー問題への具体的な動きとして注目される。Perplexity Personal Computer for Mac[^022]やResolve AI Labs $40Mシリーズ拡張[^023]はAIファーストのデスクトップ環境・インシデント対応自動化という成長領域への投資継続を示す。

## AIトレーディング

FX自動売買とAIの統合実践が日英両言語で記録されている。MQL5の解説記事[^017]は、AI搭載EAが従来型に勝る5つの理由としてLLMコンファメーションレイヤー（MLシグナル→LLMで市場コンテキスト照合の2段階検証）を挙げており、GPTをH4・日足のマスターブレイン、DeepSeekを高頻度ロジック評価に使い分ける実装例はJSON Modeを活用したMQL5側のパース安定性確保まで詳細だ。日本語圏でのLLM×MT5統合事例[^018]は、MT5 EAがChatGPT APIを定期呼び出ししてJSON形式で売買判断を受け取る具体的な実装を公開している。バックテスト不可という制約と実運用リスクの考察が含まれており、現在進行形でFX自動取引を運用している立場からは、プロンプト設計の依存度と実行環境の安定性に関する記述が参考になる。
