## Claude Code・エコシステム

### MCPが本番フェーズへ突入、脆弱性リスクも同時顕在化

AnthropicがMCPリモートサーバーの正式サポートを発表し、これまでローカル環境に限られていたMCPがクラウド上で動作可能になった[^16]。企業が社内カスタムMCPを構築し全社員がClaude経由で社内データにアクセスしたり、SaaSがMCPを公開してユーザーがAIから直接操作するシナリオが現実化しつつある。Claude Managed Agentsとの組み合わせで「SaaSプラットフォームとしてのClaude基盤」が形になってきた。

一方で、MCPのSDK設計自体に起因するRCE（遠隔コード実行）脆弱性が深刻化している[^14][^15]。個別の実装ミスではなくアダプタの設計レベルの問題であり、LiteLLM・LangChain・Flowiseの10件以上のCVEがすでに発行済み。OX Securityの調査では20万以上の脆弱なMCPインスタンスが存在するとされる。MCPを本番環境で使う場合は、最小権限原則の徹底と入力サニタイゼーション強化が急務だ。

### 公式ポストモーテムとスキルエコシステムの成熟

Anthropicが4月23日付でClaude Codeの品質低下ポストモーテムを公式公開した[^4]。SonnetとOpusで確認されていたコード生成品質の劣化と推論退行について原因と修正を詳説しており、透明性のある対応姿勢として評価できる。修正後の挙動変化はCHANGELOGで継続追跡を推奨する。

スキルエコシステムは実用段階に入っている。Anthropic公式プラグイン「claude-code-setup」がプロジェクト自動分析→Hooks・Skills・MCPサーバーの最適設定提案→ステップガイドを提供する[^12]。OSS側では「agent-session-resume」がClaude Code・Codex・OpenCode等で使えるセッション再開スキルとして公開され[^9]、Chrome MCPを使ったWebサイト1プロンプト複製スキルも注目を集めた[^10]。エンタープライズ向けには24時間でCoworkスキル15本を組織展開した事例が報告されており、QuickBooks・DocuSign・Squareなど10+コネクタを接続する実装知見が共有されている[^11]。

### GTMチームへの横展開と競合圧力

「2026 Claude Code GTM Report」は、マーケティング・営業・オペレーション部門でのClaude Code利用が急増していることを示す[^6]。CLAUDE.mdによるプロセス文書化やSkillsによる業務自動化が「コーディングツール」の枠を超えた使われ方に発展している。一方で高インパクトの投稿（1,512いいね）が「Claude CodeはCodex + GPT-5.5に王座を奪われた」と宣言しており[^19]、スマートネス・Computer Use・総合機能の面で競合が追い上げていることも事実だ。ユースケース別の使い分けを意識する局面になっている。

Claude Code 45 Tipsコレクションでは、Gemini CLIをサブエージェントとして使うコスト最適化手法や、カスタムステータスラインでのコンテキスト使用量可視化などの実践的テクニックが公開されている[^7]。

## AIニュース・業界動向

### DeepSeek V4：価格破壊と性能の同時実現（一次ソース確認済み）

DeepSeek V4-ProはMoEアーキテクチャで1.6兆パラメータ、デフォルト100万トークンコンテキスト窓を持ち、SWE-bench Verified 80.6%（Claude Opus 4.6の80.8%にほぼ並ぶ）・Codeforces 3,206（全モデル最高）を達成した[^1]。価格はGPT-5.5の約7分の1で、西側プロプライエタリモデルの価格優位性に直接挑戦する。@0xSeroは「2026年最重要AIリリース」と評し、大規模ラボ・企業・政府のAIコスト構造を根本から変えると分析する[^20]。*注：一次ソース確認済みだが価格表記に一部不整合あり（V4-Pro/Flashの混同疑い）。*

テンセントと阿里巴巴はそれぞれ「世界モデル（World Model）」を発表した[^8]。物理世界をシミュレートし「次に何が起こるか」を予測するアーキテクチャで、LLMの次の技術競争軸を示す。金融市場の「次状態」予測への応用は中長期的なトレーディングAIアーキテクチャに影響しうる視点だ。

### AI規制：連邦法 vs 州法の緊張最高潮（一次ソース確認済み）

米議会でAIディープフェイク取締法案が提出された（Ted Lieu（D-CA）主導）[^2]。非合意的なディープフェイク画像・動画の配布規制と内部告発者保護を柱とし、2026年中間選挙前の成立を目指す。超党派で動いており実現可能性は比較的高い。Colorado・New YorkでのAI法改正動向も注目される一方[^3]、DOJの「AI訴訟タスクフォース」が州法を連邦先取りで提訴する構えを見せており、連邦vs州の緊張が高まっている。

Metaが8,000人（全社員の約1割）を解雇する一方、残存社員のPC操作をモニタリングしAIに学習させるソフトウェアを導入したという報告が注目を集めた[^22]。CTOは「AIが作業をやり、人間は指示・レビュー・改善の役割に回る」と社内に明言している。OpenAIがAI専用スマートフォンを2028年発売目標で開発中という噂も流れているが、現時点では初期段階の情報として扱うべきだ[^21]。

### Anthropic：時価総額1兆ドル突破（一次ソース確認済み）

二次市場での取引でAnthropicの時価総額が1兆ドルを突破し、OpenAIを上回った[^13]。3ヶ月前の3,800億ドルから90日で約3倍の急騰で、年間収益の急増（233%増）が主因とされる。二次市場の評価額は流動性が低い点に留意が必要だが、AmazonからのAmazonからの追加出資と合わせてR&D・製品投資の財務余力を示している。

## AIトレーディング

マルチエージェントLLM金融取引フレームワーク「TradingAgents」がv0.2.4にアップデートされた[^5]。LangGraphチェックポイント再開機能（長時間タスクの中断→再開）、DeepSeek・Qwen・Azure OpenAI対応、Docker環境分離が追加されている。DeepSeek V4-Proが7倍低コストであることを考慮すると、バックエンドLLMとしてのDeepSeek選択肢がコスト効率面で現実的になった。

Claude Swarm + ObsidianでMT5を最適化し8ヶ月で700%超のリターンを達成したという報告もある[^18]が、バックテストと実運用の区別が不明確であり数値は要検証だ。MT5/NT8/TradingView対応22戦略にClaude/ChatGPT/Codexエージェントプロンプト集を組み合わせたパッケージも公開されている[^17]。

ゴールドマンサックスが2000年頃に600人いたトレーダーを2017年には2人に削減した歴史的事実が改めて取り上げられた[^23]。LLMトレーディングエージェントの普及はその「第二波」に当たる。AI自動売買の開発・運用側に立つことが、この変化を乗りこなす最も実践的な選択肢だ。
