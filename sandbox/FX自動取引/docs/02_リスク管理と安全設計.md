# リスク管理と安全設計 -- 安全に取引するための仕組み

> 調査日: 2026-02-13
> 調査の問い: 「安全に取引するためのリスク管理とは何か」

---

## 1. エグゼクティブサマリー

FX自動売買における最大のリスクは「利益を出せないこと」ではなく、「資金を失うこと」である。ESMA（欧州証券市場監督局）の開示義務データによれば、**リテール（個人）トレーダーの74〜89%が損失を出している**（出典: [InvestingGoal](https://investingoal.com/esma-broker-client-success-rate-stats/)、[BabyPips](https://www.babypips.com/news/almost-80-percent-of-retail-traders-are-unprofitable)）。この数字は「安全設計なき取引がいかに危険か」を示している。

本ドキュメントでは、FX自動売買システムに組み込むべきリスク管理の仕組みを体系的に整理する。主要な知見は以下の通り:

- **1トレードあたりのリスクは口座資金の1〜2%以内**に抑えるのが業界標準
- **損切り戦略はATRベース**が自動売買と最も相性が良い（市場ボラティリティに自動適応）
- **最大ドローダウンは20〜30%を上限**とし、10%到達時点でポジションサイズを半減させる段階的制御が推奨される
- **日次損失上限（口座の2〜3%）**に達したら自動停止するキルスイッチが不可欠
- **バックテストと実取引のギャップ**（スリッページ、スプレッド拡大、流動性低下）を過小評価してはならない
- **自動売買の最大の優位性は「感情の排除」**だが、人間がシステムを手動停止してしまうリスクも存在する

**注意: 本ドキュメントに記載された情報は教育・研究目的であり、投資助言ではない。FX取引には元本を失うリスクがある。**

## 2. 資金管理の基本原則

### 2.1 1〜2%ルール（Per-Trade Risk）

資金管理の最も基本的な原則は、**1回のトレードで口座資金の1〜2%を超えるリスクを取らない**ことである。

| 口座資金 | リスク1% | リスク2% |
|---------|---------|---------|
| 100万円 | 1万円 | 2万円 |
| 500万円 | 5万円 | 10万円 |
| 1,000万円 | 10万円 | 20万円 |

このルールの利点は**スケーラビリティ**にある。口座が増えればリスク額も自然に増え、口座が減ればリスク額も自動的に縮小する（出典: [StockForecastToday](https://www.stockforecasttoday.com/post/position-sizing-strategies-the-2-rule-and-stock-trading-risk-management)、[MH Markets Risk Management Playbook 2026](https://mhmarkets.com/global/uz/mh-academy/learn/forex-trading-risk-management-playbook-2026)）。

> 「90%のリテールトレーダーが口座を破綻させる原因は、エントリーやイグジットの精度ではなく、欲やリベンジトレードに駆られた過剰なポジションサイズである」
> -- MH Markets, Risk Management Playbook 2026

### 2.2 資金管理手法の比較

| 手法 | 概要 | メリット | デメリット |
|------|------|---------|----------|
| **固定ロット** | 常に同じロットで取引 | シンプル、初心者向け | 口座サイズの変動に対応しない |
| **固定比率（%リスク）** | 口座の一定%をリスクに設定 | 口座サイズに自動適応 | 大きなドローダウン後の回復が遅い |
| **ケリー基準** | 勝率とリスクリワード比から最適ベット比率を算出 | 数学的に最適な長期成長率 | フルケリーは変動が激しすぎる |
| **マーチンゲール** | 負けたら倍賭け | 短期的に回復しやすい | **破産リスクが極めて高い（非推奨）** |

### 2.3 ケリー基準の詳細

ケリー基準の公式:

```
Kelly % = 勝率 - (1 - 勝率) / リスクリワード比
```

例: 勝率55%、リスクリワード比1.5の場合
```
Kelly % = 0.55 - (0.45 / 1.5) = 0.55 - 0.30 = 0.25 (25%)
```

ただし、**フルケリーはドローダウンが非常に大きくなる**ため、実務では**ハーフケリー（50%）**が推奨される。ハーフケリーは最適成長率の約75%を維持しつつ、ドローダウンリスクを大幅に低減する（出典: [LiteFinance](https://www.litefinance.org/blog/for-beginners/best-technical-indicators/kelly-criterion-trading/)、[Medium - Olsonngula](https://medium.com/@olsonngula/position-sizing-using-the-kelly-criterion-e6b381bb6bb9)）。

### 2.4 リスクリワード比と勝率の関係

| リスクリワード比 | 損益分岐勝率 | 勝率60%時の期待値（1万円リスク） |
|-----------------|------------|-------------------------------|
| 1:1 | 50.0% | +2,000円/トレード |
| 1:2 | 33.3% | +8,000円/トレード |
| 1:3 | 25.0% | +14,000円/トレード |

勝率40%でもリスクリワード比1:3なら利益が出る。逆に勝率70%でもリスクリワード比が悪ければ損失になりうる。**自動売買設計では、勝率とリスクリワード比のバランスを意識した戦略設計が不可欠**である（出典: [LuxAlgo](https://www.luxalgo.com/blog/win-rate-and-riskreward-connection-explained/)、[A1 Trading](https://www.a1trading.com/whats-the-best-risk-to-reward-ratio-to-use-in-forex-trading/)）。

## 3. 損切り戦略の比較

### 3.1 損切り手法一覧

| 手法 | 概要 | 適用場面 | 自動売買との相性 |
|------|------|---------|----------------|
| **固定pips** | 常に一定のpips幅で損切り | シンプルな戦略 | 中（ボラティリティ無視） |
| **ATRベース** | ATR（平均真の範囲）の倍数で損切り幅を動的設定 | ボラティリティ変動がある市場 | **高（推奨）** |
| **サポート/レジスタンスベース** | テクニカル水準に基づく | 裁量要素が強い戦略 | 低〜中（自動判定が複雑） |
| **トレーリングストップ** | 利益方向に追従して損切り位置を移動 | トレンドフォロー戦略 | 高 |
| **ATRトレーリングストップ** | ATRに基づく動的トレーリング | トレンド+ボラティリティ対応 | **最高（推奨）** |

### 3.2 ATRベース損切りの詳細

ATR（Average True Range）は、一定期間の価格変動幅の平均を測定する指標である。

**計算方法:**

True Range = max(当日高値 - 当日安値, |当日高値 - 前日終値|, |前日終値 - 当日安値|)

ATR = True Rangeの期間平均（一般的に14期間）

**ストップロス設定:**

```
ロングの場合: ストップロス = エントリー価格 - (ATR × 倍率)
ショートの場合: ストップロス = エントリー価格 + (ATR × 倍率)
```

一般的なATR倍率:
- **1.5〜2.0倍**: 短期トレード向け
- **2.0〜3.0倍**: スイングトレード向け
- **3.0倍以上**: 長期トレンドフォロー向け

ATR期間の選択:
- **7〜10期間**: ボラティリティの高い市場向け
- **14〜21期間**: 安定したトレンド市場向け

（出典: [LuxAlgo - 5 ATR Stop-Loss Strategies](https://www.luxalgo.com/blog/5-atr-stop-loss-strategies-for-risk-control/)、[TradersPost](https://blog.traderspost.io/article/stop-loss-strategies-algorithmic-trading)、[PineConnector](https://www.pineconnector.com/blogs/pico-blog/mastering-the-atr-trailing-stop-what-it-is-and-how-it-can-boost-your-forex-strategy)）

### 3.3 ATRトレーリングストップの利点

ATRトレーリングストップはATRベースの損切りとトレーリングストップを組み合わせたもので、以下の利点がある:

1. **ボラティリティ適応**: 市場のボラティリティに応じて動的にストップ距離が調整される
2. **利益確定**: トレンドに追従しながら利益をロックインする
3. **早期損切り防止**: ボラティリティが高い局面で不必要な損切りを回避できる

（出典: [StrategyQuant](https://strategyquant.com/blog/the-atr-trailing-stops-indicator-when-and-how-to-use-it-for-effective-trading/)）

### 3.4 87種の損切り戦略テストの結果

ある検証では87種類の損切り戦略がテストされ、ATRベースの手法が一貫して上位にランクインしている。固定pipsによるストップは、市場環境の変化に対応できず、長期的にはパフォーマンスが劣化する傾向が確認されている（出典: [Paper to Profit](https://papertoprofit.substack.com/p/i-tested-87-different-stop-loss-strategies)）。

## 4. ポジションサイジング

### 4.1 基本計算式

```
ロットサイズ = リスク金額 / (ストップロスのpips数 x 1pipあたりの価値)
```

**具体例:**
- 口座資金: 100万円
- リスク率: 1%（= 10,000円）
- ストップロス: 30 pips
- 通貨ペア: USD/JPY（1 pip = 1,000円/標準ロット）

```
ロットサイズ = 10,000円 / (30 pips x 1,000円) = 0.33ロット（= 33,000通貨）
```

**重要: レバレッジはマージン（証拠金）に影響するが、リスクには影響しない。リスクはストップロスとロットサイズのみで決定される**（出典: [MT4Copier](https://www.mt4copier.com/leverage-margin-and-pip-values-in-forex/)、[OANDA Japan](https://www.oanda.jp/lab-education/beginners/aboutfx/moneymanagement2/)）。

### 4.2 レバレッジとの関係

| レバレッジ | 100万円口座で持てる最大ポジション | 備考 |
|-----------|-------------------------------|------|
| 1倍 | 100万円分 | リスク最小 |
| 10倍 | 1,000万円分 | 保守的 |
| 25倍 | 2,500万円分 | 日本国内規制上限 |
| 100倍 | 1億円分 | 海外ブローカー（リスク高） |

レバレッジが高いほどマージンコールまでの余裕が少なくなる。マージンコールは通常、証拠金維持率が80〜100%を下回った時点で発動する（出典: [FOREX.com](https://www.forex.com/jp/tradercenter/trading-ideas/risk-management/)）。

### 4.3 最大同時ポジション数の制限

自動売買システムでは、最大同時ポジション数を制限することが重要である。推奨される制限:

- **相関のある通貨ペア（例: EUR/USD と GBP/USD）**: 合計で口座リスクの3〜5%以内
- **全ポジション合計リスク**: 口座資金の5〜10%以内
- **最大同時ポジション数**: 3〜5ポジション程度

複数ポジションを同時に持つ場合、通貨間の相関を考慮しないと、見かけ上は分散しているが実際には同方向のリスクを集中させてしまう危険がある（出典: [LuxAlgo - Risk Management Strategies for Algo Trading](https://www.luxalgo.com/blog/risk-management-strategies-for-algo-trading/)）。

## 5. ドローダウン制御

### 5.1 ドローダウンの計算

```
ドローダウン(%) = (ピーク残高 - トラフ残高) / ピーク残高 x 100
```

例: 口座残高が100万円から80万円に下落した場合、ドローダウンは20%。

### 5.2 ドローダウンからの回復に必要な利益率

ドローダウンと回復の関係は**非対称**であり、これがリスク管理の最大の理由である:

| ドローダウン | 回復に必要な利益率 | 備考 |
|------------|------------------|------|
| 5% | 5.3% | 回復容易 |
| 10% | 11.1% | まだ管理可能 |
| 20% | 25.0% | 警戒レベル |
| 30% | 42.9% | 深刻 |
| 40% | 66.7% | 非常に深刻 |
| 50% | **100.0%** | **資金を倍にしなければ回復不可** |
| 70% | 233.3% | 実質的に回復困難 |
| 90% | 900.0% | ほぼ破産 |

この表が示す通り、**50%のドローダウンから回復するには資金を倍にする必要がある**。これが「損失を小さく抑えること」が利益を追求すること以上に重要な理由である（出典: [Volity](https://volity.io/forex/drawdown-in-trading/)、[Keenbase Trading](https://www.keenbase-trading.com/tools/drawdown-recovery-calculator/)、[BabyPips](https://www.babypips.com/learn/forex/drawdown-and-maximum-drawdown)）。

### 5.3 最大ドローダウンの目安

| トレーダーカテゴリ | 推奨最大ドローダウン |
|------------------|-------------------|
| ヘッジファンド | 10%未満 |
| プロップファーム | 5〜10%（日次2%上限） |
| 保守的な個人トレーダー | 10〜20% |
| 許容範囲の上限 | 30% |
| **危険ゾーン** | **30%超** |

（出典: [Volity](https://volity.io/forex/drawdown-in-trading/)、[IG証券](https://www.ig.com/jp/trading-strategies/what-is-a-drawdown-in-trading-and-how-to-handle-it-250220)）

### 5.4 段階的ドローダウン対応プラン

自動売買システムに組み込むべき段階的な対応:

| ドローダウン段階 | アクション |
|---------------|---------|
| **0〜5%** | 通常運用 |
| **5〜10%** | ポジションサイズを75%に縮小 |
| **10〜15%** | ポジションサイズを50%に縮小、新規エントリーを高確度シグナルのみに限定 |
| **15〜20%** | ポジションサイズを25%に縮小、戦略の再検証を開始 |
| **20%超** | **全ポジションクローズ、システム停止、完全な戦略レビュー** |

回復基準: ドローダウン最安値から5%回復したら、ポジションサイズを75%まで段階的に戻す（出典: [note.com - 日本個別株DDセンター](https://note.com/tatsuya_sabato/n/nd14d7a2a65af)、[Tradetron](https://tradetron.tech/blog/reducing-drawdown-7-risk-management-techniques-for-algo-traders)）。

### 5.5 日次・週次・月次の損失上限

| 期間 | 推奨損失上限 | 到達時のアクション |
|------|------------|------------------|
| **日次** | 口座の2〜3% | 当日の取引を自動停止 |
| **週次** | 口座の5〜6% | 週末までシステム停止、パラメータ検証 |
| **月次** | 口座の10% | システム完全停止、戦略見直し |

プロップファーム（自己資金運用会社）では、日次損失上限2%がスタンダードとなっている（出典: [Alpha Ex Capital](https://www.alphaexcapital.com/prop-trading/what-is-prop-trading/prop-trading-jargon-and-concepts/max-daily-loss-explained)、[TradeThatSwing](https://tradethatswing.com/setting-a-daily-loss-limit-when-day-trading/)）。

## 6. 自動停止機構（キルスイッチ）

### 6.1 キルスイッチの種類

| 種類 | トリガー条件 | アクション |
|------|------------|---------|
| **損失ベース** | 日次/週次/月次の損失上限到達 | 全ポジションクローズ、新規注文停止 |
| **連続負けベース** | N回連続で損失（例: 5連敗） | 取引一時停止、戦略パラメータの再確認 |
| **ボラティリティベース** | ATRが閾値を超過（通常の2〜3倍） | 新規エントリー停止 |
| **スプレッドベース** | スプレッドが通常の3倍以上に拡大 | 注文見送り |
| **API接続ベース** | API切断検知 | 既存ポジションにサーバーサイドSL設定、再接続試行 |
| **市場イベントベース** | 重要経済指標発表前後 | 指標発表前30分〜後15分は取引停止 |

### 6.2 取引所レベルのサーキットブレーカー

取引所レベルでも保護機構が存在する:

- **NYSE（ニューヨーク証券取引所）**: S&P 500が前日終値から7%下落でLevel 1（15分間取引停止）、13%下落でLevel 2（15分間取引停止）、20%下落でLevel 3（当日取引終了）
- **LULD（Limit Up-Limit Down）**: 個別銘柄の急激な価格変動を制限

2025年4月7日には、米国の関税政策発表を受けて**日本（東証・日経225先物等）および台湾**でサーキットブレーカーが発動した。米国市場（S&P 500）では4月4日に6%下落しLevel 1（7%）に接近したが、発動には至らなかった（出典: [CNBC](https://www.cnbc.com/2025/04/06/sp-500-circuit-breaker-on-tariff-worries-what-that-means.html)、[Nasdaq](https://www.nasdaq.com/articles/stock-market-circuit-breakers-what-you-need-know)）。

### 6.3 API切断時の対応設計

FX自動売買システムにおけるAPI切断は最も危険な障害の一つである。以下の対策が必要:

1. **サーバーサイドの損切り注文**: エントリーと同時にブローカー側にストップロス注文を送信（クライアント側のストップのみに依存しない）
2. **ハートビート監視**: 一定間隔でAPIの接続状態を確認し、応答がなければアラート発報
3. **自動再接続ロジック**: 指数バックオフ付きの再接続を試行
4. **フォールバック機構**: メインAPI切断時にバックアップブローカーへのフェイルオーバー
5. **ポジション照合**: 再接続後にローカル状態とブローカー側のポジションを照合

（出典: [MetaApi](https://metaapi.cloud/)、[FOREX.com API](https://www.forex.com/en/trading-tools/api-trading/)）

### 6.4 連続損失カウンター

アルゴリズムに「連続損失カウンター」を組み込み、指定回数の連続損失後に取引を一時停止する仕組みが推奨される。これにより、戦略が現在の市場環境に適合していない場合の損失拡大を防ぐ（出典: [LuxAlgo](https://www.luxalgo.com/blog/risk-management-strategies-for-algo-trading/)）。

## 7. バックテストと実取引のギャップ

### 7.1 ギャップの主要因

| 要因 | バックテスト | 実取引 | 影響度 |
|------|------------|--------|-------|
| **スリッページ** | なし（完全約定を前提） | 0.5〜数pips（急変時は数十pips） | 大 |
| **スプレッド** | 固定（多くの場合） | 変動（ニュース時に3〜10倍に拡大） | 大 |
| **約定力** | 100%約定 | 部分約定、リジェクト、遅延あり | 中〜大 |
| **流動性** | 無制限を前提 | 時間帯・イベントにより大幅変動 | 大 |
| **ギャップ（窓開け）** | 多くの場合モデル化されない | 週明け、イベント後に発生 | 中 |

（出典: [LuxAlgo - Backtesting Limitations](https://www.luxalgo.com/blog/backtesting-limitations-slippage-and-liquidity-explained/)、[MQL5](https://www.mql5.com/en/blogs/post/762963)、[PineConnector](https://www.pineconnector.com/blogs/pico-blog/backtesting-vs-live-trading-bridging-the-gap-between-strategy-and-reality)）

### 7.2 スリッページのモデル化

スリッページと市場インパクトは**非線形現象**であり、二次関数によるモデリングが最も正確とされる。実務的には以下の対策を取る:

- **バックテスト時にスリッページコストを追加**: 通常0.5〜1 pipを見込む（ボラティリティの高い時間帯は2〜3 pip）
- **約定遅延の考慮**: MetaTrader 5の場合、平均約定速度は32ミリ秒前後だが、遅延時には数百ミリ秒に達することがある
- **スプレッドの現実的設定**: 固定スプレッドではなく、時間帯別の変動スプレッドを使用

（出典: [QuantStart](https://www.quantstart.com/articles/Successful-Backtesting-of-Algorithmic-Trading-Strategies-Part-II/)、[Capitalise.ai](https://support.capitalise.ai/en/articles/5963164-trading-slippage-and-how-it-affects-live-trading-simulated-trading-and-backtests)）

### 7.3 オーバーフィッティング（過剰最適化）の危険性

バックテストにおける最大の罠は**オーバーフィッティング（カーブフィッティング）**である。

> 「オーバーフィッティングは取引戦略のサイレントキラーである。過去データのノイズをモデル化してしまい、華麗なバックテスト結果が実取引で崩壊する」
> -- LuxAlgo

**実例**: Knight Capitalは2012年にソフトウェアのデプロイミス（8台中1台に新コードが未適用、廃止済みの旧テストコードが再活性化）により、**わずか45分間で4億4,000万ドル（約660億円）の損失**を出した。これはオーバーフィッティングではなくデプロイメント管理の失敗だが、自動売買システムの品質管理がいかに重要かを示す（詳細はセクション8.5参照）。

**対策:**

| 対策 | 内容 |
|------|------|
| **アウトオブサンプルテスト** | データの70%で最適化、残り30%で検証 |
| **ウォークフォワード分析** | ローリングウィンドウで最適化と検証を繰り返す |
| **クロスペア検証** | 複数通貨ペアで戦略の頑健性を確認 |
| **パラメータ数の抑制** | ルールが少ないほどオーバーフィッティングのリスクが低い |
| **モンテカルロシミュレーション** | ランダムな順序変更で戦略の堅牢性を検証 |

（出典: [LuxAlgo - Overfitting](https://www.luxalgo.com/blog/what-is-overfitting-in-trading-strategies/)、[QuantifiedStrategies](https://www.quantifiedstrategies.com/curve-fitting-trading/)、[Logical Invest](https://logical-invest.com/walk-forward-testing-avoid-curve-fitting-backtesting/)、[Build Alpha](https://www.buildalpha.com/robustness-testing-guide/)）

### 7.4 流動性リスク

流動性は時間帯によって大きく変動する:

- **最も流動性が高い時間帯**: ロンドン〜ニューヨーク重複時間（日本時間21:00〜翌1:00頃）
- **最も流動性が低い時間帯**: アジアセッション後半（日本時間早朝5:00〜7:00頃）
- **流動性が急低下するイベント**: 祝日（特に年末年始）、重要経済指標発表直前

2019年1月のフラッシュクラッシュは、日本の祝日で流動性が薄い時間帯に発生した（後述）。

## 8. 失敗事例と教訓

### 8.1 スイスフランショック（2015年1月15日）

**概要**: スイス国立銀行（SNB）がEUR/CHFの1.20下限維持政策を突然撤廃。

**影響**:
- EUR/CHFが**1分以内に20%以上暴落**（1.20から1.00未満へ）
- 発表後1分以内に主要流動性プロバイダーが全てのクォートを撤回
- 5分後には2つの流動性プロバイダー間のクォート差が**5,382 pips**に達した（特定2社間の異常値であり、通常のビッド・アスクスプレッドとは性質が異なる）
- FXCM（当時最大手のFXブローカー）: 顧客の**マイナス残高が2億2,500万ドル**に達し、約3億ドルの緊急融資が必要になった
- SNB自身も2015年上半期に外貨ポジションの評価損として**472億フラン**を計上（通年の最終損失は230億フラン。「為替取引の損失」ではなく保有外貨資産のスイスフラン建て評価額の減少）

**教訓**:
- ストップロス注文があっても、**流動性がなければ約定しない**（ギャップリスク）
- 単一通貨ペアへの過度な集中は壊滅的損失につながる
- 中央銀行の政策変更は予測不可能な「ブラックスワン」イベントとなりうる

（出典: [LeapRate](https://www.leaprate.com/news/how-forex-brokers-went-bankrupt-overnight-amid-eurchf-flash-crash-infographic/)、[FTMO](https://ftmo.com/en/the-biggest-flash-crash-in-forex-can-a-similar-situation-happen-again/)、[Finance Magnates](https://www.financemagnates.com/forex/brokers/fxcm-publishes-data-of-snb-mishandling-of-the-swiss-franc/)）

### 8.2 円フラッシュクラッシュ（2019年1月3日）

**概要**: 日本の祝日期間中、薄い流動性下で自動売買の連鎖が暴落を引き起こした。

**影響**:
- USD/JPYが数分間で**約4%急騰**（円高方向）
- AUD/JPYは**約8%暴落**（2009年以来の安値）
- AUD/USDも4%以上下落
- トルコリラ/円は**10%下落**

**メカニズム**:
1. 円キャリートレードのポジション（「ミセス・ワタナベ」）が膨張
2. Appleの業績下方修正でリスクオフ心理が広がる
3. 薄い流動性下で自動ストップロスが連鎖的に発動
4. モメンタムベースのアルゴリズムが下落を増幅
5. 各ブローカーのマージンコールによる強制決済がさらに加速

**教訓**:
- 流動性の薄い時間帯（祝日、早朝）のリスクを過小評価してはならない
- 自動売買のストップロスが連鎖的に発動すると「雪崩」を引き起こす
- キャリートレードは平時には安定しているが、急変時に壊滅的損失を生む

（出典: [CNBC](https://www.cnbc.com/2019/01/03/yen-surges-against-global-currencies-after-flash-crash.html)、[Deutsche Bank](https://www.db.com/news/detail/20190104-thursday-s-currency-flash-crash-explained?language_id=1)、[RBA](https://www.rba.gov.au/publications/smp/2019/feb/box-b-the-recent-japanese-yen-flash-event.html)）

### 8.3 ポンドフラッシュクラッシュ（2016年10月7日）

**概要**: GBP/USDがわずか数分で**約6%暴落**。

原因はアルゴリズム取引の連鎖と、アジア時間帯の薄い流動性が組み合わさったとされる。EU離脱（Brexit）に関するネガティブなニュースヘッドラインにアルゴリズムが反応し、売りが売りを呼ぶ展開となった。

（出典: [Headway](https://hw.online/blog/what-is-a-flash-crash/)、[EarnForex](https://www.earnforex.com/guides/can-forex-crash/)）

### 8.4 ストップハンティング（ロスカット狩り）

**概要**: 大口参加者（機関投資家、マーケットメーカー）が意図的に個人トレーダーのストップロス注文が集中する価格帯を突破し、流動性を獲得する行為。

- **被害規模**: 一部のFX教育サイトでは、リテールトレーダーの最大**40%が年に1回以上**ストップハンティングの被害を受けているとされるが、学術論文や規制当局による一次ソースは確認できておらず、推定値として扱うべきである
- **2025年の状況**: プロップファーム、ソーシャルトレーディングの普及により、ストップ注文の集中する価格帯が予測しやすくなり、ストップハンティングはより精緻化・頻繁化している

**対策**:
- 明らかなサポート/レジスタンスラインの直下/直上にストップを置かない
- ATRベースのストップで市場のノイズに耐える余裕を持たせる
- 固定的なストップ位置パターンを避ける

（出典: [EBC Financial Group](https://www.ebc.com/forex/liquidity-sweep-in-forex-how-to-trade-the-trap)、[ACY](https://acy.com/en/market-news/education/market-education-stop-hunting-trading-swing-highs-lows-liquidity-j-o-20250822-095643/)、[FXSSI](https://fxssi.com/forex-stop-hunting)）

### 8.5 Knight Capital事件（2012年8月1日）

株式市場の事例だが、自動売買のリスクを示す極めて重要なケース。ソフトウェアのデプロイミスにより、**45分間で4億4,000万ドルの損失**を計上。適切なキルスイッチが実装されていなかったことが主因。

（出典: [LuxAlgo - Overfitting](https://www.luxalgo.com/blog/what-is-overfitting-in-trading-strategies/)）

## 8.A 心理的リスクと自動売買の優位性

### 人間のバイアス vs 機械的執行

| 心理的バイアス | 内容 | 自動売買での対処 |
|--------------|------|----------------|
| **損失回避バイアス** | 損失の痛みは利益の喜びの約2倍と感じる | 感情なく損切りを執行 |
| **サンクコスト効果** | 既に投じたコストにこだわり損切りを先延ばしにする | 機械的にストップロスを執行 |
| **リベンジトレード** | 損失後に大きなリスクを取って取り返そうとする | ルールベースでポジションサイズを一定に保つ |
| **群集心理（追随売買）** | 他者の行動に追従して売買する | 事前に定義したシグナルのみに基づく |
| **確証バイアス** | 自分に都合の良い情報だけを集める | データのみに基づく客観的判断 |
| **過信バイアス** | 連勝後にリスクを過大に取る | 固定%ルールで規律を維持 |

> 「自動売買の最大の価値は、優れた戦略の執行ではなく、健全なアプローチを妨害する感情的干渉の排除にある」
> -- TradersPost

**具体データ**: 手動でのオーバーライド（システムの判断を人間が覆す行為）は、**1契約あたり平均85ドルのコスト増**につながるとされる（出典: [LuxAlgo - Automated vs Manual Trading](https://www.luxalgo.com/blog/automated-trading-vs-manual-trading-pros-and-cons/)）。ただし、この数値の一次ソースは確認できておらず、参考値として扱うこと。

### 自動売買でも残る心理的リスク

自動売買は感情を排除するが、**人間はシステムを止める権限を持っている**。以下の行動が自動売買の利点を無効化する:

1. **ドローダウン中にシステムを手動停止する**: 連続損失への不安から
2. **利益が出ている時にパラメータを変更する**: 「もっと稼げるはず」という欲から
3. **損失後にレバレッジを上げる**: リベンジ心理

（出典: [TradersPost - Psychology of Automated Trading](https://blog.traderspost.io/article/psychology-automated-trading-control)、[Robots4Forex](https://robots4forex.com/smart-trading-with-algorithms/understanding-the-psychology-behind-algorithmic-trading/)、[Aron Groups](https://arongroups.co/technical-analyze/loss-aversion-bias-in-trading/)）
