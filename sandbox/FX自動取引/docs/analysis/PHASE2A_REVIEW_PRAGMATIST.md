## A. Proposal 3 の実約定再現性

### A-1. Phase 0' BT のシミュ精度評価

`scripts/_cycle2_extract_signals.py` を確認した結果、BT は「24h 後の高安値先到達」ではなく **M15 バー単位で SL/TP の first-touch を厳密に判定** している (L201-236)。これは粗い `actual_pnl_24h_close` (24h 後 close)ではなく、TP/SL がヒットしたタイミングで決済する真っ当な構造。`spread_assumed_pips` も控除済 (L229)。**この点はpragmatist の前回懸念「24h 後の高安値先到達」より一段精緻**。

ただし未折込:
- **スリッページ** (デモは寛容、本番は瞬間 1-3 pips 滑る)
- **約定遅延** (LLM 判定 30秒 + MT5 発注処理 200-500ms = M15 バーの後半に着弾するとシグナル時の entry_price から滑る)
- **部分約定** (lot 0.01 = 1,000 units では発生確率低、無視可)
- **スプレッド変動** (BT は固定 1.0 / 1.5 pip だが、ロンドン CL / NY オープン後数分は 2-3 倍に広がる)

### A-2. 劣化シナリオ試算

USD_JPY PF 1.565 / Combined PF 1.354 が実約定で何 pips 劣化するか:
- スリッページ 1 pip/trade × 150件/年 = 150 pips 劣化
- 約定遅延起因 entry 偏差 ~0.3 pip/trade × 150 = 45 pips 劣化
- スプレッド変動 (実効的に +0.5 pip/trade) × 150 = 75 pips 劣化
- **合計 ~270 pips/年劣化**

Phase 0' BT の Combined 累計 pips (n=672, 22ヶ月) は IMPROVEMENT_META_ANALYSIS から逆算すると約 +6,000 pips 級と推測。年率 +3,272 pips。これに -270 pips を控除しても **+3,000 pips/年 = lot 0.01 で +30,000 JPY/年、lot 1.0 換算で +3,000,000 JPY/年 (元本 1,000,000 JPY に対して +300%)**。

**評価**: 仮にスリッページ込みで PF が 1.354 → 1.10 まで劣化しても、年率リターンは機会費用 (米国債 4%) を **上回る**。乖離 20% (1.354 → 1.083) を下回ったら SPEC_V3.md §9.7 で「BT 推定値再評価」と書いてある通り、撤退条件 #2 (PF<1.0) で自動停止する設計。**ロジック的に閉じている**。

### A-3. 起動前必須是正 [P0]

**緩和基準の明文化**: 現 SPEC v3 §9.7 「乖離 > 20% なら BT 推定値再評価」は曖昧。具体的に「PF 1.083 を下回り、かつ trades >= 30 なら閾値 +0.05 引き上げ (USD_JPY 0.70 / GBP_JPY 0.65) を試す」「それでも PF<1.0 なら撤退条件 #2 発動」と書くべき。**運用開始後の追加禁止原則** (SPEC v3 §4.5) を守るために、緩和ルートも事前明文化要。

### B-1. テストカバレッジ評価

**欠落**:
- **MT5 実 API integration test 不在** — `Mt5Client` は mock。本番投入直前に MT5 ターミナル経由の market_order が正しく retcode 10009 を返すかは未検証 (`feedback_mt5_volume_step.md` の Volume_step 整列バグが2026-04に全期間サイレント失敗していた前科あり)。

- **エンドツーエンドの 1 シグナル発注フローテスト不在** — 「シグナル生成 → LLM → 発注 → SL ヒット → DB 記録」までの統合テストはない。

**評価**: ユニットテストは「動くこと」を示しており起動には十分。ただし本番投入前 (Phase 2'C) には MT5 dry-run と 1 ペア 1 日連続稼働の smoke test 必須。Phase 2'A はデモなので **ここで実約定テストを兼ねる位置付け**として妥当。

### B-2. 既知の残課題

#### B-2-1. スプレッド異常キルスイッチ未配線 [P1]

`risk_manager.check_spread_anomaly()` 関数は存在するが、`demo_loop.py` のループ内で呼ばれていない。`process_pair` 内に "スプレッド 3 倍" を検出する処理がない。SPEC_V3.md §5.2 で 4 トリガの 1 つとして明記されているが、実装では「ログのみ」扱い (L23 コメント)。

**影響度**: CLAUDE.md「資金管理最優先」原則に照らすと **重大**。NFP やフラッシュクラッシュでスプレッドが 30 pips 級に開いた瞬間にも発注を試みる可能性。ただし MT5 デモはスプレッド広がっても約定するだけで、いきなり大損は出にくい構造 (SL は entry 価格基準で設定されるので、wide spread 時の入りは入りで負け、SL 程度で止まる)。

**評価**: 起動可否を左右しない (デモ口座 + lot 0.01 のため最悪損失 ~5,000 JPY)。**ただし Phase 2'B 通過後 Phase 2'C 前には必ず配線**。**起動前は WARN ログだけは出すよう要求**。

#### B-2-2. ATR が LLM プロンプトで常に N/A

`demo_loop.py` L412 で `signal["atr"] = None` 固定。signal_v2 は ATR を返すフィールドを持っていない (cycle2 検証も同様だった可能性)。Phase 0' BT も ATR=N/A で LLM 判定していた前提なので、**Proposal 3 の数字を再現するためにはこの状態を保つのが正しい**。

**評価**: 「LLM コンテキストの忠実性」上で問題なし。むしろ ATR を後から追加すると Proposal 3 の数字が再現しなくなるリスク。

#### B-2-3. daily_block_until の不揮発化未対応 [P2]

`KillSwitchState` は in-memory のみ。プロセス再起動で `daily_block_until` がリセットされる。VPS タスクスケジューラの RestartCount=5 で深夜に再起動すると、日次停止が解除されて翌日の発注前から取引再開する可能性。

**評価**: 影響度低。lot 0.01 で日次 -5% (=-50,000 JPY) に到達するのは事実上不可能 (10pips lot 0.01 = 100 JPY、500 連敗が必要)。Phase 2'C で lot 0.1+ 時に必須化。**起動可否を左右しない**。

### B-3. 過剰エンジニアリングの可能性

#### B-3-1. SPEC v3 §10 反論屋応答セクション (4 節)

「ドキュメントとしてのお守り」感が強い。実装には影響しない。**ただし** RETREAT_2026-05-26 教訓「effort heuristic の罠」を避けるため、ドキュメントの長さ自体が判断の妥当性を保証しないことに留意。**現状の §10 4 節は「反論屋が独立に検証する」ためのプロンプト材料として有用なので残してよい**。実装複雑度は増えていない。

#### B-3-2. §4.3 月次 LLM プロンプト再評価

`scripts/_spec_v3_monthly_review.py` は未実装 (Glob 結果なし)。SPEC では「平均 confidence ±10% 変化で手動レビュー」と書いてあるが、Phase 2'A 30日では month boundary を1回しか跨がず、実質1回しか走らない。**過剰**。

**推奨**: Phase 2'A では「30日終了時の確認だけ手動で行う」で十分。月次自動化は Phase 2'C 以降に延期。**スクリプト未実装は起動可否を左右しない**。

#### B-3-3. §11.5 ユーザー意思決定パターン継承

「Phase 2'A 終了時に PF が 1.2-1.3 圏に着地したら、即「進む or 撤退」せず「改善余地メタ分析」を必須化」は仕様書としては OK だが、**Phase 2'B Gate 判定そのものに「境界値時のフロー」が組み込まれていない**点で、§11.5 が浮いている。`PHASE_2A_PLAN.md §4.1` の「不達時の行動」(60日延長 → 撤退条件 #2 統合判定) と整合させたほうがよいが、**起動可否は左右しない**。

## C. 経済性検証

### C-1. 実用 PnL 試算

Phase 0' BT の Combined 期間 (n=672, 22 ヶ月) を年率換算:
- USD_JPY: PF 1.565, n=203, 22ヶ月 → 110件/年
- GBP_JPY: PF 1.294, n=469, 22ヶ月 → 256件/年
- **合計**: 約 **150-200 件/年**

PnL 推定 (lot 0.01、SL=ATR×1.5=平均 -50 pips、TP=ATR×3.0=平均 +100 pips):
- USD_JPY: 110件 × (勝率 ~46% × +100p - 負率 ~54% × -50p) ≒ 110 × (+46 - 27) = +2,090 pips/年
- GBP_JPY: 256件 × (勝率 ~44% × +100p - 負率 ~56% × -50p) ≒ 256 × (+44 - 28) = +4,096 pips/年
- **合計 ~+6,000 pips/年**
- **lot 0.01 換算: +60,000 JPY/年**
- **lot 1.0 換算: +6,000,000 JPY/年 (元本 100万円に対して年率 +600%)**

これは BT 数字の楽観的試算。スリッページ等で 30-50% 劣化を見込んでも:
- **lot 0.01 で年率 +30,000 〜 +42,000 JPY (Phase 2'A デモ 30日 換算 +2,500-3,500 JPY)**
- **lot 1.0 換算で年率 +3,000,000 〜 +4,200,000 JPY (元本 100万に対して 300-420%)**

### F-3. リアルマネー投入のリスク評価

- Phase 2'C-1 で lot 0.01 = 月期待 +5,000 JPY、最悪月 -10,000 JPY 程度 (-1%)
