# CONTRARIAN PRAGMATIST — 経済性ベース反論

**作成日**: 2026-05-25
**観点**: 過剰エンジニアリング検出 + 実用性検証
**結論先出し**: **SPEC v2 PoC は経済的に成立しない。本番投入したら銀行預金より儲からないどころか元本毀損する。**

---

## 0. ユーザー指摘の核心

> 「数百万運用して1年間で数千円の儲け、とかなら銀行に預けるほうがマシ」

これは **経済性ベンチマーク** という、Step A〜C の検証フレームに欠落している軸を突いている。
A/C エージェントは「YZ_vol 閾値の統計的妥当性」「観察期間延長」で結論を出したが、**「期待 PnL がいくらか」を一度も計算していない**。

本ドキュメントは過去データだけで、その欠落を 60 分で埋める。

## 3. 経済性ベース判定 (機会費用)

### 表 3-1: 100 万円口座、2 年運用シナリオ (lot 1.0 換算)

| 運用先 | 2 年累計 JPY | 2 年累計 % | 年率 % |
|---|---:|---:|---:|
| 銀行預金 (0.05%) | +1,000 | +0.10% | +0.05% |
| 個人向け国債 変動10年 (~0.7%想定) | +14,000 | +1.40% | +0.70% |
| 米国債 ETF (4%) | +80,000 | +8.00% | +4.00% |
| 全世界株式 (8%) | +160,000 | +16.00% | +8.00% |
| **signal_v2 BT (lot 1.0、スプレッド込)** | **-385,811** | **-38.58%** | **-19.29%** |
| signal_v2 BT (スプレッドなし理論値) | +46,200 | +4.62% | +2.31% |

### 結論

- 銀行預金は 2 年で +0.10%
- signal_v2 を 100 万円口座でロット 1.0 で 2 年回したら **-38.58%、▲38 万円失う**
- **どんなに観察期間を延ばしても、シグナルが PF 0.95 のままなら、運転時間に比例して損失が増えるだけ**
- ユーザーの指摘「数千円の儲けなら銀行のほうがマシ」は **誤りで、実態はもっと悪い。儲けではなく数十万単位の損失**

## 4. A/C エージェントへの反論

### 反論 1: 観察期間延長は無意味な延命策

A エージェントは「90 日観察期間に延長」を結論したが、**観察対象 (signal_v2 × YZ_vol 二層) の期待 PnL が -2,894 JPY/年 (lot 0.01) で確定している**。
これを 90 日延長しても、期間に比例して損失が拡大するだけ。**「観察」とは経済性が立つ可能性を待つ行為であって、確定マイナスを継続する行為ではない**。

### 反論 2: 統計的妥当性 ≠ 経済的妥当性

C エージェントは「閾値が Spearman ρ で統計的に妥当」「Permutation test p<0.05」と結論した。
だが **本 BT が示すのは、YZ_vol 二層判定で VOLATILE を切り出しても、その中で打つ M15 ブレイクアウトは PF 0.95**。
**閾値の統計的有意性は「VOLATILE 判定がランダムでない」ことを意味するだけで、「VOLATILE 中にエッジが立つ」ことを保証しない**。
A/C はこの区別を曖昧にして「閾値妥当 → PoC GO」と結論した。

### 反論 3: 検証フレーム自体が経済性盲目

SPEC v2 § 2-1 (季節判定) は **「VOLATILE が観測される」** を目的にし、**「VOLATILE 中の戦略 PnL がプラスである」** を検証していない。
これは setup-only 検証であって、execution-with-edge 検証ではない。
A/C が経済性を語らないのは個人の落ち度ではなく、**SPEC v2 の検証フレーム自体に PnL ゲートが存在しない構造的欠陥**。

## 5. 過剰エンジニアリング指摘

### 5-1: 投じた精緻化と、それが防げなかったもの

| 投資 (高度な統計検証) | 防げた問題 |
|---|---|
| Permutation Test | × VOLATILE 中の戦略 PnL 不在は検出できない |
| Bonferroni N=606 | × 「閾値の偽陽性」は防ぐが「戦略の経済性」は射程外 |
| Romano-Wolf | × 同上 |
| Mode A vs Mode B 比較 | × YZ_vol の優越性は示すが、絶対的経済性は示さない |
| 単峰性検定 | × 分布形状検証で、PnL ゲートではない |
| Block bootstrap CI | × 閾値推定精度の話、戦略エッジの話ではない |
| 「★★★★★」「AAA」評価 | × **完成度の見せ方** であって、**儲かるかどうかの答え** ではない |

これら全部が、`scripts/_contrarian_bt.py` (216 行、実行 30 秒) で出る **「PF 0.95、年率 -2,894 JPY」** の一行に勝てない。

### 5-2: 簡単な代替手段

「過去 2 年で signal_v2 を回したら年間いくら儲かるか」 — この一問だけで PoC GO/NOGO は決まる。
やったのは:
- M15 5 万バーに YZ_vol を計算
- VOLATILE 判定
- ブレイクアウト+ATR×1.5/3.0 シミュレート
- スプレッド -2 pips/trade を引く
- 累計 PnL を出す

これだけ。**1 ファイル、30 分、ブートストラップも順列検定も要らない**。
PoC が 90 日かけて拾うべき情報は、既に過去データに書いてある。

### 5-3: 過剰エンジニアリングの典型構造

- 「未確定」の signal_v2 と「確定」の SeasonalDetector を分離した結果、**シグナル単独の経済性検証がフレーム外に追いやられた**
- 「Step A: 閾値妥当 → Step B: シグナル設計 → Step C: 統合検証」という直列フレームが、**Step A の段階で経済性ゲートが効かない構造**を作った
- 「★★★★★ Step A 完了」と言える代わりに、**Step C 直前で「実は赤字だった」と気づく**リスクを引き受けている

## 6. 推奨アクション

### 即時 (今日)

1. **PoC 起動は停止**。現在の構成 (signal_v2 + SeasonalDetector) は経済的に成立しない
2. `scripts/_contrarian_bt.py` の結果を SPEC v2 § 3-1 の合格基準に明示的に組み込む:
   - **GO 条件**: BT で年率 PnL > 米国債利回り × ボラ補正、PF > 1.3、シャープ > 0.8
   - 現状 PF 0.95、シャープ -0.39 → **NOGO**

### 短期 (1 週間)

3. signal_v2 を「プレースホルダではなく仕様候補」に格上げし、**ブレイクアウト以外の代替** (RSI ダイバージェンス、平均回帰、複数足コンフルエンス等) を 2-3 個 BT 比較
4. 「VOLATILE 中で打つ何か」のうち PF > 1.3 が立つものを **過去データで先に見つけてから** PoC を起動

### 中期 (1 ヶ月)

5. SPEC v2 § 2-1 の評価指標に「VOLATILE 期間中の素朴 momentum / mean-reversion PnL」を追加し、**ボラ判定が経済性に翻訳されるパスを設計上組み込む**
6. Step A 〜 C のフレームに「経済性ゲート」を挿入。統計的検証通過 → 過去 BT で年率 +5% 以上のエッジ確認 → 初めて PoC ペーパートレード
