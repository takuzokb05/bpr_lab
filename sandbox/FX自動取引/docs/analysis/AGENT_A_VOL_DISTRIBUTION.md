## 6. 結論

**両方の要因が同時に効いている**。比率は以下と判定:

1. **「現状の vol が異常に低い」要因 (主因、寄与度 ~70%)**
   - PoC 中央値 0.000693 = 5年分布の **9.5%ile** (下位 1割)
   - PoC max 0.001312 = 5年分布の **71.6%ile** (上位 30% 圏外)
   - 直近5年で 0.00175 を超えるバーは平均 10.03%、直近1年は 2.04% に縮小
   - 直近6か月で月 H1 max が 0.00131 以下だったのは 2025-12 のみ (該当6か月中1か月 = 17%)
   - **5/12 を除けば PoC 期間は過去5年で下位 10% に入る低ボラ局面、5/23 以降はさらに沈静化**

2. **「閾値が構造的に過大」要因 (副因、寄与度 ~30%)**
   - 5年分布で 0.00175 は p90 相当 (高ボラ局面の上位 10%) で設計通り
   - しかし PoC 内分布での p99 でも 0.001307、p100 で 0.001312 と「閾値到達がそもそも構造的に起こらない期間長」が存在する
   - Rolling 14日 p95 が 74-75% で固定化している点は「過去14日の最大級 vol を踏襲しても閾値に届かない期間」が存在することを示し、**動的閾値 (rolling p95 等) でも現状期間では VOLATILE を生めない**
   - これは「閾値が誤っている」のではなく「絶対閾値ベースの設計が、低ボラ体制下では完全沈黙する」という設計脆弱性

**示唆 (実装判断は C 担当)**:
- 半月のサンプルで「過大」と断じるのは早い。**過去5年で 11% の頻度で起きるパターン**を引いただけの可能性が高い
- ただし「VOLATILE 0 件期間」が 2-4 週間続く構造的可能性は織り込んだ運用設計が必要 (transitional モードを稼働させる、または絶対閾値を rolling 動的閾値に切り替える等)
- **判定の確証を得るには**: PoC 期間を最低 90 日まで延長 (= 過去5年分布で over 10% を 1 回でも引く期待値が現実的になる規模)。半月のデータで閾値変更を決断するのはサンプル不足。
