## 3. 観察期間延長 90 日は数学的に救済にならない

A と C の処方箋は揃って「90 日延長」。しかし以下の計算で **延長しても判断材料が増えない**ことが示せる:

### 期待 VOLATILE 発火回数 (Y5 ヒット率 1.99% を採用)

- H1 バー数: 90 日 × 24 = 2,160 バー
- H1 上抜け期待値: 2,160 × 1.99% = **43 バー**
- M15 上抜け同時刻成立 (PoC 実測 M15 above 18.4%): 43 × 18.4% = **約 8 バー**
- VOLATILE 判定 → エントリー想定: **約 8 件**

### サンプル不足

- 戦略評価に必要な最小取引数: 一般に 30-50 件 (バリュー検定で標準偏差を信頼区間化するため)
- 90 日延長で得られる: **8 件 (期待値)**
- **必要サンプル達成には 90日 × (50/8) = 約 1.5 年延長**が必要

### Agent A の主張の致命的弱点

Agent A は「5/12 のピーク (0.001312) でも閾値の 75%」「Rolling 14 日 p95 が 74-75% で固定化」と書いた。**つまり Agent A 自身の数字が「現在の vol 体制では rolling p95 でも到達しない」と示している**。

その上で「90 日延長を推奨」と結ぶのは **論理矛盾**。延長してもデータ生成過程が変わらない限り、**観察対象の事象がそもそも発生しない**。

**反論ポイント**:
- 「サンプルを増やせば判断できる」が成り立つのは、母集団が正常に観測される場合のみ
- 現在は「観察対象 (= VOLATILE 局面) が消失している」状態。延長は **無を増やすだけ**

## 7. 具体的予言 — このまま進めると何が起きるか

### シナリオ A: Y6 が低ボラ年継続 (確率 60%, Y1/Y3/Y5 の 3/5 ベース)

- PoC 90 日延長 → VOLATILE 0-3 件発火
- 「サンプル不足で再延長」コール → 累計 6 か月運用
- **経済成果: ゼロ または マイナス数百円**
- **機会費用: 100 万円ベースで -40,000 〜 -80,000 JPY/年 (米国債/S&P500 比)**
- 結論: 「観察延長」が文化となり、撤退判断もできず半年-1年溶ける

### シナリオ B: Y6 が高ボラ年復帰 (確率 40%, Y2/Y4 ベース)

- VOLATILE 50-100 件発火、placeholder 戦略が取引を回す
- placeholder の本性 = M15 ブレイクアウト × ATR×1.5 SL × RR=2.0
- 過去 GBP_JPY 本番実績勝率 35.1% を踏襲する場合の期待値 = `0.351×2 - 0.649×1 = +0.05R / trade`
- スプレッド 3pip = 0.5R 相当のコスト消化 → **期待値ほぼゼロ or 微マイナス**
- 100 件 × 0R = **±数千円**
- 結論: **★★★★★ が「経済的にゼロ」と確定し、SPEC v2 全体の再起草**

### どちらに転んでも

- **「銀行預金 (0.5%) で +5,000 JPY」を上回れない**
- **「米国債 (4%) で +40,000 JPY」を上回れない**
- ユーザーの懸念「数百万運用して年数千円なら銀行のほうがマシ」は **100% 現実化する**

## 8. では何をすべきか — 反論屋からの提案

「批判だけして代案ナシ」を避けるため、最低限の歯止めを 3 つ示す:

### 提案 1: **経済性 Gate を SPEC v2 ロードマップに正式追加** (最優先)

PoC 起動の前提条件として、以下を追加せよ:

```
[Gate E] 経済性事前バックテスト
- 過去 5 年 GBP_JPY OHLC で signal_v2.py を回し、PF ≥ 1.5, 勝率 ≥ 40% を確認
- スプレッド 3pip / スリッページ 0.5pip を控除した実効リターンで判定
- PF < 1.5 ならシグナル再設計 (PoC 起動却下)
```

これは 2026-05-07 の C 案 Gate FAIL と同じ仕組み。**SPEC v2 でも踏襲すべき**だった。

### 提案 2: **signal_v2.py を placeholder のまま PoC を回さない**

placeholder のまま観察を続けても、何が観察できているのか不明 (§ 4 参照)。
- SPEC v2 § 3-1 を確定させる
- 最低でも「過去 5 年 GBP_JPY で PF≥1.5 が出る暫定シグナル」を組み込む
- placeholder のまま回すなら、**「これは SeasonalDetector のフィルタ精度測定のみが目的」と明記し、PnL ゼロ前提で運用** (= デモ口座でもエントリーさせない)

### 提案 3: **撤退条件を先に書く**

PoC 開始時点で「以下を満たしたら撤退」を SPEC に明記:

```
[撤退条件]
- 90 日経過時点で trades < 5 件 → 撤退 (観察不能)
- 90 日経過時点で PF < 1.0 → 撤退
- 累計 PnL < -3,000 JPY → 撤退
- 機会費用 (年率 4% = 月 3,300 JPY @ 100万) を 3 か月連続で下回る → 撤退
```

撤退条件を **先に固定する**ことで、「物語の自己永続化」(memory: `feedback_baseline_check_failure.md`) を物理的に防ぐ。
