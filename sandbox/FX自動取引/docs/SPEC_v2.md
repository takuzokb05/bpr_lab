> 「SPEC が新しい逃避先」になっていないことを構造で証明する（karen 反論屋指摘 D-3 への抗体）。

## SPEC v2 の作業ルール

1. **数字には必ず根拠を付ける**: バックテスト結果 / walk-forward 検証 / 実トレード観察 / 文献参照 のいずれか
2. **「亡き者」化した既存パラメータ（PF 2.05 等）は引き継がない**: PREMISE.md に従う
3. **数字を確定したら次のフィールドを記入**:
   - 値
   - 根拠（出処）
   - 検証日付
   - 再検証期限（推奨6ヶ月）
4. **数字が空のうちは運用に降ろせない**: 空欄が残るスキームは本番投入禁止

### ★ Step B 文献調査ラベル (2026-05-08)

🚨 **本節の全採用閾値は本番投入禁止 (Step C 完了まで)**。理由:

- **★Deflation 未補正**: 試行数 N=360-600 推定（指標 K × length 4 × 時間軸 3 × 閾値グリッド ~10）。
  Bailey-López de Prado (2014) False Strategy Theorem に従い、PF/Sharpe を log(N) オーダーで deflate する処理が未実施。
- **★文献整合 部分**: 採用基準が自前用語 TR>1.0 で、文献標準 (Profit Factor BCa CI 下限>1.0 + Romano-Wolf 補正) に置換待ち。
- **★単一分割 WFA**: 現代標準は CPCV または rolling WFA × 5+ fold。現状は k=1 相当 (Pardo 2008 入門レベル)。
