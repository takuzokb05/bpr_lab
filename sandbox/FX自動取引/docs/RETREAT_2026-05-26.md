## 撤退理由 — 5つの構造バグ

3反論屋が独立に発見した構造バグの統合:

### バグ① 検証範囲と運用範囲のミスマッチ
Step C は **「季節判定器 (M15+H1 YZ_vol) が統計的に意味あるか」** を検証して ★★★★★ を取った。これは正しい。しかし PoC は **「分類器 + 戦略 + 実発注」** の3層を稼働させていて、戦略本体 (signal_v2) は `placeholder, 勝率未検証` のまま。**第1層の検証だけで5層先の運用判断をした**。

### バグ② 撤退済 GBP_JPY を3日後に再採用 (自己一貫性崩壊)
亡き者の世界で `data/fx_trading_prod_snapshot.db` に **GBP_JPY 37件 / 勝率 35.1% / PF 0.868 / -1,239 JPY (10日) / 年換算 -45,000 JPY** という事実数字が残っていた。2026-05-07 に撤退決定、**3日後の 5/10 に PoC 唯一の通貨として再採用**。`docs/SPEC_v2.md` L252 で USD_JPY には亡き者注記があるのに GBP_JPY には注記なし — **自分が書いた文書内で認識の非対称が発生**。

### バグ③ 経済性 Gate が SPEC v2 ロードマップに不在
2026-05-07 に「PF<1.5 で却下」という Gate を適用したばかり。なのに 3日後の PoC 起動で同じ Gate を再適用しなかった。SPEC v2 のロードマップは Step A→B→C→PoC→本番投入 という直列で、**経済性検証 (PF / Sharpe / 機会費用) のステップが構造的に欠落**。

### バグ④ PoC の三重定義
1つの `poc_loop.py` の中に「分類器の検証」「戦略の検証」「実発注の検証」が同居していて、どれを検証しているかが定義不明。`signal_v2.py` 冒頭で「§3-1 確定までのプレースホルダ」と自認しているのに PoC 実発注を起動した = **未確定戦略 × 未発火フィルタの二重不確定**で「何が観察できているのか不明」。

### バグ⑤ 4時間損切りのミスマッチ
亡き者 GBP_JPY 実取引の保有時間中央値 **234.4分**、**40.5% が >240分**。PoC 設計の `POC_MAX_HOLDING_MINUTES = 240` は **平均的取引の4割を強制決済する**。亡き者の挙動データを参照せずに設定された安全装置が、戦略の評価そのものを歪めている。

## なぜこの状態に至ったか — メタ要因

### effort heuristic の罠
SPEC v2 § 2-1 で Permutation / Bonferroni / Romano-Wolf / Mode A/B / 単峰性 / 介入実験 / 実用 WFA まで詰めた。**手続きの完成度がそのまま「正しさ」の証拠と感じられる**錯覚。Pragmatist の30秒 BT (PF 0.95) がこの錯覚を破壊した。

### 自分が書いた memory を自分で破った
`memory/feedback_intervention_required_for_causation.md` に「**採用候補は実用 WFA を経てから** (2026-05-10 SPEC v2 Q1/Q2 v1 撤回事例)」と書いた**まさに同じ日**に、別の論点で PoC を起動した。memory は事後反省記録としては機能していたが、事前チェックリストとしては機能していなかった。

### フレーム延命バイアス (AI 側の認知)
AI (Claude) が反論屋3体を起動した後、それらの結論を **「フレーム内補強案」** として消化してしまった (α/β/γ/δ の4選択肢)。**「畳む」「ゼロからやり直す」という選択肢が見えなくなった**。ユーザーがこの認知の浅さを補正した = フレーム自体を撤退する判断。

## 次フェーズ — プロポーザル方式

### 原則
- **手法 → フレーム**: 先に PF > 0.95 を示す手法を選び、それに合わせて新フレーム (SPEC v3) を書く

- **撤退条件を先に書く**: 90日 trades<5 / PF<1.0 / 累計-3,000 JPY のいずれかで即撤退

### Phase
- **Phase 0**: 候補手法ロングリスト (memory 実践者知見 + 文献 + 内部アイデア)

- **Phase 2**: 上位候補の精査 BT (WFA / OOS / Deflated Sharpe)
- **Phase 3**: 採用判断 + SPEC v3 起草
- **Phase 4**: PoC 再起動 (採用後)
