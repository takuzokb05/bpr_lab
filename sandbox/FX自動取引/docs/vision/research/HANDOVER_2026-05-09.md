### Q1. 指標–リターン単峰性 (H2 検証) — 優先度: 最高

**問い**: M15 YZ_vol の値域と future_return の関係は単峰か二峰か?

**背景**: P1-1b で M15 YZ_vol 3ペアの Mode B 最良閾値が **10%ile / 90%ile に二極化**。
これは指標–リターン関係が単峰でない可能性を示唆する。
- 単峰なら: Mode A 30/80%ile は単に「中庸の悪い値」、最適は端点近傍
- 二峰なら: H2 (ボラ-トレンド直交分解) の前提 + OPERATING_MODEL の「volatile レジーム」概念が揺らぐ

**検証方法**:
- 各ペア × 各時間軸で、YZ_vol を 20 分位に区切り、各分位の |future_return| 中央値をプロット
- 単峰 (中央値が単調変化) vs 二峰 (低/高ボラで上昇、中ボラで凹む)
- ついでに CHOP も同じ手法で確認 (CHOP は <25 で安定 → 単峰の可能性高い)

**期待される結論パターン**:
- (a) 単峰 → グリッド設計を見直す、SPEC v2 § 2-1 数値の方向性は維持
- (b) 二峰 → 指標選定そのものを再考、OPERATING_MODEL § 2-1 の概念設計まで波及
- (c) ペア依存 (USD/EUR/GBP で違う) → H4 (ペア別閾値) の根拠が深まるが、複雑化

**実装**: `scripts/_spec_2_1_indicator_return_curve.py` (未作成)

### Q2. H1 YZ_vol グリッド拡張 — 優先度: 高

**問い**: H1 YZ_vol の真の最適閾値は SPEC × どこにあるか?

**背景**: P1-1b で H1 YZ_vol 3ペアが Mode B で **グリッド上限 (SPEC × 1.5) を全 fold で選定**。
- SPEC 値 (0.00143-0.00175) が最適でない
- 真の最適は SPEC × 2 か × 3 か、それとも × 5 か不明
- グリッド外を探さないと結論できない

**検証方法**:
- 各ペアで YZ_vol abs グリッドを SPEC × {1.0, 1.5, 2.0, 2.5, 3.0, 4.0, 5.0, 6.0} で生成
- 各 fold IS で最良閾値を選定し、OOS で評価
- グリッド上限が再選定されるか、それとも内部に頭打ちが見つかるか確認

**期待される結論パターン**:
- (a) SPEC × 2.0-3.0 で頭打ち → 新 SPEC 値はそこ、Mode B 結果と整合
- (b) SPEC × 5.0 でも上限選定継続 → グリッド設計が大きく外れている (再々設計必要)
- (c) ペアごとに頭打ちが違う → ペア別グリッドが必要

**実装**: `scripts/_spec_2_1_h1_grid_extension.py` (未作成)
