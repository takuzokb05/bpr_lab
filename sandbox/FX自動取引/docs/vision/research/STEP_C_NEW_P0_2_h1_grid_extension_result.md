## 検証方法

### TR 評価式の改良

Q1 検証で「TR = median(high) / median(low) は low 群サイズ・ばらつきに敏感」と判明 (Mode B 二極化の真因)。
今回は **low 群を 25-50%ile 帯に固定** し、TR の比較性を確保した。

### グリッド

- abs: SPEC × {1.0, 1.5, 2.0, 2.5, 3.0, 4.0, 5.0, 6.0} (8 点)
- pct: {80, 85, 90, 92.5, 95, 97.5, 99, 99.5} (8 点)

### MIN_SAMPLE = 30

high / low 群とも 30 サンプル未満で TR を計算しない。

## 観察事項

### ① P1-1b の「グリッド上限選定」現象を完全説明

P1-1b では Mode B WFA で SPEC × 1.5 (グリッド上限) が全 fold 選定されていた。
本検証で ×1.5 (TR ≈ 2.4-2.7) は ×2.0 (TR ≈ 2.5-3.1) の手前 = **真の最適閾値はもっと高い帯**

→ Mode B での「グリッド上限選定」は、P1-1b の時点ですでに「真の最適がグリッド外」を示唆していた。

### ② 旧 SPEC (Mode A の 90%ile = ×1.0) は単峰の中段選択

Q1 で 12/12 件が単峰と確定した上で、Mode A の 90%ile は単に「中段」を取っていただけ。
×2.0 帯 (≈ 99%ile) のほうが TR は明らかに高い。

### ③ サンプル数とトレードオフ — 実用性は別問題

|ペア | ×1.0 n_high | ×2.0 n_high | 5 年比 |
|---|---|---|---|
| USD_JPY | 3,093 | 282 | 約 11x 減 |
| EUR_USD | 3,081 | 123 | 約 25x 減 |
| GBP_JPY | 3,106 | 318 | 約 10x 減 |

5 年で ×2.0 を採用すると n_high が 100-300 程度。WFA で 5 fold に分割すると 1 fold 20-60。
**Mode B WFA の MIN_SAMPLE_IS=50 制約に抵触する fold が発生する** 可能性が高い。

→ 実用性の確認は次のステップで Mode B WFA を SPEC × 2.0 で再実行する必要あり。

### ④ EUR_USD の特異性 (Handover 決定 C 関連)

EUR_USD は SPEC × 2.5 で n_high が 3 まで激減 — ボラ分布の上端が他 2 ペアよりタイト。
これは Q1 でも EUR_USD H1 の P/T 比 2.64 (USD_JPY 4.10, GBP_JPY 3.21 より低い) と整合。

→ EUR_USD は単峰だが「単峰の幅が狭い」= 他ペアと共通枠組みでは扱えるが、閾値は別個に最適化すべき (H4 ペア別閾値の根拠)

### ⑤ GBP_JPY ×3.0 の TR=4.64 は信用しない

n_high=50 の median 統計はノイズが大きい。
percentile 99.5%ile (×2.49, TR=3.19) で評価するほうが安全。
SPEC v2 § 2-1 の数値降ろしには **GBP_JPY ×2.0 (0.00350)** を採用候補とする。
