# Phase 1C-1 Gate FAILED — C案 v2 (MTFPullback トレンド判定堅牢化) 不採用 + GBP_JPY 撤退決定

**日付**: 2026-05-07
**判定**: GATE FAILED（PF≥1.5 全ペア未達）
**結論**: C案 v2 を全ペア一律適用は不採用 / GBP_JPY を運用ペアから撤退

---

## 1. 背景

5/6 USD_JPY/GBP_JPY/EUR_USD 全ペアで取引0件（pipeline 4309回中 EXECUTE 0）。
原因は MTFPullback の「戻り待ち」設計が強トレンド一方通行相場で機能しないこと。

改善3案を立案:
- **B案**: 朝レポ確信度連動の RSI 閾値動的緩和
- **C案**: トレンド判定堅牢化（MA200傾き + ADX）
- **A案**: ブレイクアウト戦略を排他切替で並走

エージェントチーム編成で検証:
- **analyst** 推奨: C → B → A
- **researcher** 知見: A案は「並走」より「Regime gating（排他切替）」が筋
- **karen** 反論: C単独は機会逸失悪化、5定数追加はカーブフィット製造機、Kill criteria 必須
- **pragmatist** 反論: 5定数→1定数で十分、フィーチャーフラグ不要、既存 adx_threshold 流用

→ **C案 v2（簡素化版 + 着手前BT検証 Gate）** に集約してユーザー承認、Gate 1 実施。

## 3. 重要発見（karen 予言の現実化）

1. **「C単独は機会逸失悪化」予言通り**: 全ペアで取引数減（合計 -41件 / 4ヶ月）
2. **「EUR_USD実績(+3029)を毀損するリスク」現実化**: バックテストでも PF 1.43 → 1.01 と劇的悪化
3. **バックテスト2倍乖離の前提崩壊**: ドキュメント記載の「PF 2.05 (EUR/USD M15) / 1.97 (USD/JPY M15)」と実測 PF 1.43 / 1.17 / 0.80 が大きく乖離。ドキュメント値の信憑性低下
4. **GBP_JPY は撤退レベル**: バックテストでも PF 0.80（=負け戦略）。本番実績 38件13勝 -2,004 と整合的

## 4. 決定事項

### 4.1 GBP_JPY 撤退（決定）

**根拠**:
- バックテストPF 0.80 = **どの改善案を当てても勝てない可能性が高い**
- 本番実績 38件中13勝、累計 -2,004 JPY（勝率 34%）
- C案 v2 でも改善せず（PF 0.69 に悪化）
- リスク集中懸念（USD_JPY/EUR_USD の2ペア運用）はあるが、負け戦略を残すよりマシ

**実装**:
- `src/config.py` `DEFAULT_INSTRUMENTS` から `GBP_JPY` 削除 → `["EUR_USD", "USD_JPY"]`
- `config/pair_config.yaml` `GBP_JPY` セクションに「停止中（PF 0.80, -2004 JPY, 2026-05-07撤退）」コメント
- VPS への反映は別 PR で

### 4.2 EUR_USD 現状維持（決定）

**根拠**:
- バックテストPF 1.43、本番 14件5勝 +3,029 JPY のプラス資産
- C案 v2 は PF を 1.43 → 1.01 に悪化させる（既存資産毀損）
- 触らない方が良い

### 4.3 USD_JPY: C案チューニング検討（保留）

**根拠**:
- C案 v2 で PF 1.17 → 1.47（+0.30）と改善方向
- Gate 1.5 まであと +0.03、`slope_min_pips` を 3.0→2.0 等に緩めれば達成可能性
- ただし PF 1.5 が「本番に投入する根拠」として十分かは別議論

**次アクション候補**:
- パラメータグリッド再検証（`slope_min_pips ∈ {1.0, 2.0, 3.0}`, `slope_bars ∈ {5, 10, 20}`）
- バックテスト2倍乖離の根本調査（スリッページ実測再注入、約定遅延再現）

### 4.4 B案・A案の位置づけ更新

- **B案**: GBP_JPY 撤退で対象が USD_JPY/EUR_USD のみに。EUR_USD は維持なら朝レポ連動の対象が狭い → 優先度低下
- **A案**: 排他切替型として中期検討。GBP_JPY 撤退で並走対象が減ったため、価値も減

## 5. 教訓（次回以降に効かせる）

1. **着手前BT Gate は機能した**: karen の Kill criteria 「過去データ再BTで PF<1.5 → 投入前 revert」が PR 作成前にC案を止めた。本番投入していたら EUR_USD の +3,029 を毀損していた可能性
2. **ドキュメント記載のPF値は信用しない**: PF 2.05/1.97 は実測 1.43/1.17 と大乖離。docs/strategy_validation_*.md の数値は再検証必要
3. **「全ペア一律」改善は危険**: USD_JPY 改善 / EUR_USD 悪化 / GBP_JPY 元々負け、と振る舞いがペア毎に違う。**ペア別最適化前提**の設計に切り替えるべき
4. **エージェントチーム編成は機能した**: researcher/analyst/Plan/karen/pragmatist の5本立てが、私単独では見落とした「並走→排他切替」「閾値緩和の三重悪化リスク」「BT2倍乖離の前提崩壊」を全て事前に補足
