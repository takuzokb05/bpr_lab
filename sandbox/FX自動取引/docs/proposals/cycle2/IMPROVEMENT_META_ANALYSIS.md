## リスク評価 (再現性・オーバーフィット)

### 1. 多重検定によるオーバーフィット疑い

同一データから 5 ビン × 7 閾値 × 5 セッション × 5 レジーム = **多数の切り口を試して最良を選んでいる**。素朴な単一仮説検定の p 値は使えない。

**緩和策**:
- IS/OOS 分割で評価し、**OOS で PF が IS と同等以上のもののみ採用** → Proposal 1, 3 はこの基準をクリア
- Proposal 2 は GBP_JPY の OOS n=0 が許容できないため次点扱い
- セッション × 閾値の組合せ最強値 (例: GBP_JPY × Any London or NY × conf≥0.70 PF 3.262) は n=23 で **採用しない**

### 2. LLM 自己評価バイアスの再現性

- `temperature=0` で固定されているが、Claude モデルバージョン (sonnet-4-6) が更新されると confidence の分布が変わる可能性
- 本番運用時はモデルバージョンを `claude-sonnet-4-20250514` 等で **固定** し、confidence のドリフトを月次モニタする運用が必要

### 3. シグナル分布の構造変化

- 直近 2025Q3 以降に CONFIRM only PF が劣化 (1.328 → 0.874) している。これは:
  - signal_v2 のエッジ自体が薄れている (環境変化)
  - LLM の confidence 0.5 帯の判定精度が下がっている
- Proposal 3 (高 confidence のみ) はこの劣化を回避できているが、**今後さらに劣化する可能性は否定できない**。
- **対策**: 半年運用後に PF 1.10 を下回ったら一旦停止し、新たに半年の判定データで閾値を再最適化

### 4. サンプル数評価

| 1 (Combined ≥0.65) | 319 | 2025Q1 で n=3 | ⚠️ 四半期スパース |

## 統合推奨

### 推奨アクション: **Phase 2'A 進行 (Proposal 3 採用)**

理由:
1. **diff +0.438 で CYCLE2 ゲート +0.30 を超過**
2. **OOS で 性能維持** (IS 1.335 → OOS 1.376) → 再現性確認
3. **直近 2025Q3+ で PF 1.338 維持** → 本番期待値として現実的
4. **n=672 (≈150件/年)** で運用機会十分、四半期サンプルも各≥30 維持
5. **実装はペア別パラメータ 1 つ追加するだけ** (`confidence_threshold_usd_jpy=0.65`, `confidence_threshold_gbp_jpy=0.60`)

### SPEC v3 への組み込み案

```python
class LLMFilterConfig:
    enabled_pairs = ["USD_JPY", "GBP_JPY"]
    confidence_thresholds = {
        "USD_JPY": 0.65,
        "GBP_JPY": 0.60,
    }
    accept_decisions = ["CONFIRM"]  # NEUTRAL/REJECT/CONTRADICT は全除外
    # base 戦略 (signal_v2) で signal -> LLM 判定 -> confirm かつ
    # ペアごとの閾値以上の confidence -> 発注、それ以外は見送り
```

### 二次推奨 (シンプル版): Proposal 1

Proposal 3 の実装が煩雑なら、**Combined × conf ≥0.65 (PF 1.586)** で進めても良い。ただし機会数が 319 件 → ≒70件/年に減るので、デモ期間中に十分なトレード数が稼げないリスクあり。

### 撤退の選択肢

撤退する場合の判断基準:
- Proposal 3 でデモ運用半年実施し、PF<1.10 なら撤退
- LLM API コストが月 5,000円 超過したら撤退 (現状コスト試算: 月 ≒2,000円 で許容範囲)

## 改善余地で見送るもの

| 切り口 | 採用しない理由 |
|---|---|
| セッション絞り込み (London+NY 重複等) | confidence 閾値の上に積んでも追加効果 +0.05 未満。サンプルが減るリスクが上回る |
| Low vol レジーム除外 | 効果あり (+0.295) だが、`abs(usd_jpy_24h)` のリアルタイム取得が必要で実装複雑度上昇。confidence 閾値で代替可能 |
| reasoning キーワードフィルタ | パターン差が +0.05〜+0.10 程度で、キーワードマッチに依存するため脆い |
| confidence ≥0.70+ (Proposal 2) | GBP_JPY OOS n=0 で再現性確認できず。サンプル不安定 |
| REJECT 基準緩和 | REJECT 群の if_taken PF が 0.849 で base より悪く、緩める動機なし |
