# プロポーザル: YZ_vol レジームルーター (戦略フィルターではなく戦略選択器として再利用)

## 1. 戦略仮説 (1段落)

SPEC v2 で「★★★★★ 確定」を取った YZ_vol 季節判定器は、**戦略フィルター (VOLATILE 中だけ打つ) として使うと PF 0.95 で失敗**した (Pragmatist BT)。これは「VOLATILE 期間にエッジ源があるシグナルを持っていなかった」だけで、季節判定器自体は **YZ_vol 高 vs 低 でリターン特性が違うこと自体は permutation p<0.001** で示されている。本提案は YZ_vol を「フィルター」ではなく「**戦略ルーター**」として再利用する: 高ボラレジームではモメンタム/ブレイクアウト系、低ボラレジームでは平均回帰系を発動させる。**Ultra 教訓 (CONTRARIAN_ULTRA.md) の三重定義 (分類器+戦略+発注 の同居)** を解消し、判定器は判定だけ、戦略選択は別モジュール、発注はさらに別、と完全分離する。

## 2. 想定エッジ源 [G1-1]

**構造的優位**:
- 平均回帰戦略は低ボラ期に強く、高ボラ期に弱い (BB幅が異常拡大すると 2σタッチが頻発し誤シグナルになる)
- モメンタム戦略は高ボラ期に強く、低ボラ期に弱い (ブレイク後の追従が消えてダマシになる)
- YZ_vol は実測 OHLC を Yang-Zhang 推定量で集約しており、ATR や std よりノイズ耐性が高い (Yang & Zhang, 2000)
- **同じ判定器を 2 戦略で活かす**ので、検証コストの償却が効く

**実取引データの裏付け** ([[feedback-deceased-world-data-inheritance]] 遵守):
- 亡き者は YZ_vol を使わず固定戦略で全レジームに突っ込み、低ボラ時の BollingerReversal 系で勝てる時もあったが、高ボラ時に MTFPullback で誤押し目買いをして負けた可能性 (詳細データは別途検証要)
- 「亡き者は分類器を持っていなかった」が負け原因の一つ。レジーム認識を加えることが純粋な追加情報

## 3. シグナル定義 (擬似コード)

```python
def route_strategy(data_m15, data_h1):
    # ステップ1: YZ_vol 計算 (M15 と H1)
    yz_m15 = yang_zhang(data_m15, window=20)
    yz_h1 = yang_zhang(data_h1, window=20)

    # ステップ2: レジーム判定 (rolling 30%ile, 3レジーム)
    yz_m15_p30 = rolling_quantile(yz_m15, window=5000, q=0.30)
    yz_m15_p70 = rolling_quantile(yz_m15, window=5000, q=0.70)

    if yz_m15.iloc[-1] > yz_m15_p70:
        regime = "HIGH_VOL"  # モメンタム適性
    elif yz_m15.iloc[-1] < yz_m15_p30:
        regime = "LOW_VOL"   # 平均回帰適性
    else:
        regime = "MID"       # 取引停止 (中途半端な期間は触らない)

    # ステップ3: 戦略選択
    if regime == "HIGH_VOL":
        return MomentumBreakout.generate_signal(data_m15)  # ATR breakout 改善版
    elif regime == "LOW_VOL":
        return BollingerReversal.generate_signal(data_m15)  # 既存 BB 逆張り
    else:
        return HOLD
```

**重要設計**: signal_v2 の placeholder 罠 ([[feedback-verification-scope-mismatch]]) を避けるため、**両戦略は独立に過去 BT で PF > 1.2 を予め確認したものしか繋がない**。Placeholder のままルーターに繋いではならない。

## 4. データ要件 [G1-2]

## 5. リスクモデル [G1-5]

- レジーム別に戦略を持つ = 同じ口座リスク 1%/trade、ただし**同時に複数レジーム戦略は起動しない** (排他)
- レジーム遷移時は in-flight ポジションをそのまま継続 (再判定で強制クローズしない)
- 想定 MaxDD: -10% (各戦略単独 BT の MaxDD × √2 で見積)
- テールリスク: レジーム判定が誤動作すると両戦略とも誤発火 → 月次 backtest シャドウ実行で齟齬検出

## 6. 自己改善メカニズム [G0-B] — 必須

**ドリフト検出 (毎週月曜)**:
- 直近30日のレジーム別 PnL を集計
- HIGH_VOL レジームの PF<1.0 / LOW_VOL レジームの PF<1.0 のどちらかでアラート
- レジーム判定そのもののドリフト: rolling p30/p70 閾値が直近 3ヶ月で過去 12ヶ月対比 ±30% 変動でアラート

**パラメータ自動再最適化 (月次)**:
- YZ_vol window (10/20/30)、rolling quantile window (3000/5000/8000)、quantile (0.25/0.30/0.35) のグリッド
- 各戦略個別のパラメータ (BB std / ATR mult) も並行再最適化
- 直近 12ヶ月 IS / 直近 3ヶ月 OOS で各レジーム別 PF を評価

**フォールバック**:
- 月次再最適化で全パラメータ組合せが PF<1.0 → 該当戦略のみ停止 (もう一方は継続)
- レジーム判定器が誤発火 (同じバーで HIGH と LOW が連続して切り替わる等) を検出したら、判定器をリセット (rolling window 再計算)

**トリガー条件**:
- 毎週月曜 06:00 JST ドリフトチェック
- 月初1日 全パラメータ再最適化
- 累計 -3,000 JPY で個別戦略停止、両方停止で全停止

## 7. 過去 BT 結果 [G0-A] — 必須

**現時点で示せる根拠**:
- BollingerReversal 単独 (LOW_VOL 想定): **PF 1.24 EUR_USD M15** 既存実績
- ATR breakout 単独 (HIGH_VOL 想定): signal_v2 で PF 0.95 (失敗実績、ただし**全レジーム適用**)
- YZ_vol 高ボラ期間だけに ATR breakout を絞った BT は未実施

**Phase 1 で行うべき追加 BT (宿題)**:
- EUR_USD M15 5年で HIGH_VOL レジームのみで ATR breakout PF 計測
- 同 LOW_VOL レジームのみで BollingerReversal PF 計測
- 期待値: HIGH_VOL/LOW_VOL 分離で各 PF が 1.3 以上に持ち上がる仮説
- もし HIGH_VOL での ATR breakout が PF<1.0 のまま → HIGH_VOL レジームでは取引しない設計に変更 (LOW_VOL 単独戦略になる)

**PF > 0.95 達成根拠**:
- LOW_VOL 単独運転に縮退しても素の BollingerReversal で PF 1.24 確保
- HIGH_VOL でのモメンタム成功は **gravy** であって、必須ではない

## 8. WFA / OOS [G1-7]

- レジーム判定 (rolling p30/p70) は anchored expanding で 5-fold WFA
- 各戦略個別の WFA を別途実施
- 統合 BT は walk-forward の各 fold で「判定器 + 戦略」を一体評価
- Deflated Sharpe を計算 (試行数: レジーム3グリッド × 戦略2 × 戦略パラメータ9 = 54)

## 11. リスク・既知の弱点

1. **判定の自己関与性**: YZ_vol で「HIGH_VOL」と判定した瞬間に既にボラ反転の兆しが始まっている可能性 (look-ahead 風のリスク)
2. **MID レジーム比率**: 30/70 で切ると中間 40% で取引停止になり、機会損失が大きい可能性 → グリッド調整必須
3. **両戦略の同時崩壊**: 2022年 Q3 のような全レジーム崩壊期に両方失敗
4. **三重定義罠の再演リスク**: ルーター + 戦略 + 発注の3層が密結合すると、また「何を観察しているかわからない」状態に。**設計時から明確分離**
5. **HIGH_VOL モメンタム部分のエッジ未確認**: 既存 signal_v2 PF 0.95 は HIGH_VOL に絞っても改善する保証なし
