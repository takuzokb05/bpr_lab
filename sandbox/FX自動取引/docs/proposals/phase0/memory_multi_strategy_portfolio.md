# プロポーザル: マルチ戦略ポートフォリオ (EUR_USD H1 RSI + USD_JPY M15 RSI を分散運用、優位な単一戦略の合成)

## 1. 戦略仮説 (1段落)

単一戦略の運命依存リスクを避けるため、**BT で OOS PF > 1.3 を実証した独立2戦略をポートフォリオ運用**する。具体的には (a) **EUR_USD H1 RSI Pullback** (BT で OOS PF 1.34-1.49、20中13が PF>1.0) と (b) **USD_JPY M15 RSI Pullback** (BT で OOS PF 1.94-2.46、20中14が PF>1.0)。両戦略は時間軸 (H1 vs M15) も通貨 (EUR vs USD-JPY) も独立。互いに低相関で、片方のレジーム崩壊を他方が補完。エッジ源は分散投資理論 (Markowitz 1952) + 各戦略の独立した実証エッジ。

## 2. 想定エッジ源 [G1-1]

- **構造的優位**: 2戦略は時間軸・通貨が独立 → Markowitz 分散効果でシャープレシオ最大化
- **個別エッジ源**: EUR_USD H1 = レンジ反転 + ECB/Fed 金利差ナラティブ、USD_JPY M15 = 流動性ピーク時の短期 micro reversal
- **過去BT共通**: 両戦略とも 20パラメータ中 70%超で PF>1.0、Pardo plateau 形状
- **亡き者DB**: EUR_USD は唯一プラス (+3,315 JPY)、USD_JPY は負け (-919 JPY) だが H1 timeframe での話 → M15 差別化

## 3. シグナル定義 (擬似コード)

```python
def signal_portfolio(bar_close_h1, history_h1, history_m15):
    # 並走する2戦略
    eur_signal = eur_usd_h1_rsi_signal(bar_close_h1, history_h1)
    usd_signal = usd_jpy_m15_rsi_signal(bar_close_m15, history_m15)

    signals = []
    # 各戦略の lot 配分 (50/50 等配分、後で動的調整可)
    if eur_signal:
        eur_signal.units = int(eur_signal.units * 0.5)  # 50% 配分
        signals.append(eur_signal)
    if usd_signal:
        usd_signal.units = int(usd_signal.units * 0.5)
        signals.append(usd_signal)

    return signals  # 同時実行可能

# 月次でリスクパリティ再配分
def rebalance_monthly():
    eur_sharpe = calc_rolling_sharpe('eur_usd_h1_rsi', window=60)
    usd_sharpe = calc_rolling_sharpe('usd_jpy_m15_rsi', window=60)

    # シャープ比率で配分
    total = eur_sharpe + usd_sharpe
    eur_alloc = eur_sharpe / total
    usd_alloc = usd_sharpe / total

    # ただし片方 Sharpe<0 ならその戦略は 0% (一時停止)
    if eur_sharpe < 0:
        eur_alloc = 0
    if usd_sharpe < 0:
        usd_alloc = 0
```

## 5. リスクモデル [G1-5]

- 各戦略の SL/TP を継承 (EUR_USD: ATR×1.5/×2.5, USD_JPY: ATR×2.0/×3.0)
- **同時最大ポジション**: 2 (片方ずつ)
- **戦略間相関監視**: 30日 rolling 相関 > 0.7 になったら片方を半量化
- **想定 MaxDD**: 個別 -5〜-15% を分散効果で **-3〜-8%** に縮小
- **テールリスク**: USD インデックスショック (例 Fed 緊急利下げ) で EUR_USD と USD_JPY 同時に動く可能性 → 30日相関で警告

## 6. 自己改善メカニズム [G0-B] — 必須

### ドリフト検出
- **戦略別 PF モニタ**: 各戦略の rolling 30件 PF を独立に追跡
- **相関監視**: 30日 rolling 相関で 0.7 超なら警告
- **総合シャープ**: ポートフォリオ全体のシャープレシオを月次測定、0.5 未満で全停止考慮

### 自動再最適化
- **月次リスクパリティ**: 過去60日のシャープ比率に応じて lot 配分を再計算 (上記擬似コード)
- **戦略個別の月次パラメータ再選定**: 各戦略の BT グリッド再走で最適パラメータを更新
- **戦略追加/削除**: クォーター毎に、ポートフォリオに加えるべき新候補 (例: GBP_USD RSI Pullback) を BT グリッドで評価し、PF>1.3 かつ既存戦略と相関<0.3 なら追加

### フォールバック
- **戦略単独で 30件連続 PF<0.8 → その戦略を1ヶ月停止**
- **ポートフォリオ全体 PF<0.95 が 60日継続 → 全停止 + 1ヶ月冷却**
- **配分履歴を `data/portfolio_history.json` に記録**

### 擬似コード (1段落)
> 2戦略を 50/50 で並走、毎月リスクパリティで配分再調整 (シャープ比率比例)。各戦略個別の月次パラメータ再選定。30日相関>0.7 で警告、片方半量化。戦略単独 30件連続 PF<0.8 で1ヶ月停止。ポートフォリオ全体 PF<0.95 が60日継続で全停止。クォータごとに新候補 (GBP_USD RSI等) を評価し追加検討。

### ポートフォリオ合成 (50/50, シミュレーション)

- 個別戦略のリターン系列を結合し、リスクパリティ調整後の総合 Sharpe を測定
- **想定**: 個別シャープ 0.7-1.0 → ポートフォリオシャープ **0.9-1.3** (相関 < 0.3 の場合)
- 想定 PF: **1.5-1.8** (個別を分散効果で底上げ)
- 想定 MaxDD: 個別 -10〜-15% → **-5〜-9%**

### Phase 1 で実装する BT
- 2戦略を同時実行した過去5年シミュレーション (50/50 + リスクパリティ両方)
- スクリプト: `scripts/_portfolio_bt.py` を新規作成 (個別 `_contrarian_bt.py` を拡張)

### PF > 0.95 を超える論証
- 個別戦略がそれぞれ単独で PF > 1.3 → ポートフォリオは個別の加重平均かそれ以上 → **必ず PF > 1.0**
- 相関が -1 でない限り (現実では +0.0〜+0.3)、分散効果で個別 PF を維持しつつ DD 縮小

## 11. リスク・既知の弱点

1. **戦略間相関の上昇リスク**: USD 共通要因で同時動き → 相関監視必須
2. **実装複雑度の増加**: 単一戦略より管理コスト高 (5-7日 vs 2-3日)
3. **OOS 検証データ不足**: USD_JPY M15 が OOS 11-15 件と少ない、ポートフォリオ全体も検証期間限定
4. **モニタリング負荷**: 30日相関 + 戦略別PF + ポートフォリオシャープの3層監視
5. **小口運用での不利**: 配分 50/50 で各戦略の最小ロット (0.01) を割れない口座規模が必要 → 100万円以上推奨

## 撤退条件 (事前明記)

1. ポートフォリオ全体 PF < 0.95 が 60日継続
2. 累計 PnL < -5%
3. 戦略間相関 > 0.7 が3ヶ月継続 (分散効果消失)
4. 個別戦略の片方が 90日 trades < 5 (バランス維持困難)
