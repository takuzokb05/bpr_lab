# プロポーザル: 亡き者の負けパターン逆張り (Loss Pattern Reverse-Engineering)

## 1. 戦略仮説 (1段落)

亡き者 (`fx_trading_prod_snapshot.db`) の closed trade 64件のうち **負けが 49件 (76.6%)**、勝ちが 15件のみ。**「亡き者が頻繁に取った逆方向を取れば勝てるのではないか」** という反転戦略仮説。本提案は、亡き者が打ったエントリー条件 (MA-crossover シグナル + 各種フィルタ) を**逆方向に変換**して BT する。具体的には:
- 亡き者が BUY を打ったタイミングで SELL
- 亡き者が SELL を打ったタイミングで BUY
- SL/TP のロジックはそのまま (反転トレード設計の鏡像)

**ただしこの仮説は脆い**。亡き者の負けが「シグナル誤り」だったのか「スプレッド負け」「タイミング遅延」「市場ノイズ」だったのかを峻別しなければ単純反転は機能しない。本提案は **「亡き者の負けがシグナル方向そのものに起因する」場合のみ採用**、それ以外は廃案とする。

## 2. 想定エッジ源 [G1-1]

**構造的優位** (仮説):
- もし亡き者の戦略がシグナル方向そのもので負けていたなら、**逆張りで構造的に勝てる**
- 「他人が損切りする所で買う」は古典 (Wyckoff 1934, "Composite Operator") の現代的版
- **個人投資家コホートの逆相関は実証研究あり** (Barber & Odean 2000: 個人の選定銘柄は市場平均を下回る)

**実取引データの裏付け** ([[feedback-deceased-world-data-inheritance]] 遵守):
- 亡き者 GBP_JPY: 勝率 35.1%、PF 0.87 → ベース勝率 50% に対し**有意に低い**
- 亡き者 USD_JPY: 勝率 14.3%、PF 0.48 → 極めて低い
- 亡き者 GBP_USD: 勝率 0%、PF 0.00 → 全敗 (3件サンプルだが)
- ただし EUR_USD は勝率 38.5% で **PF 2.37** → 勝率は低いが勝ち分が大きい平均回帰パターン
- **「負け = シグナル方向が逆」は EUR_USD では成立しない** (RR で取り戻している)
- **GBP_JPY/USD_JPY は反転候補**

## 3. シグナル定義 (擬似コード)

```python
def generate_signal(data_m15, data_h1):
    # 亡き者と同じシグナル生成 (MA crossover + RSI + ADX + MFI)
    raw_signal = OldMACrossover.generate_signal(data_m15)
    if raw_signal == HOLD:
        return HOLD

    # 反転
    if raw_signal == BUY:
        reversed_signal = SELL
    else:
        reversed_signal = BUY

    # SL/TP は鏡像
    return reversed_signal

# ペア制限: 亡き者で PF<0.5 だったペアのみ反転を試す
# = GBP_JPY (PF 0.87), USD_JPY (PF 0.48), GBP_USD (PF 0.00)
# EUR_USD は逆に PF 2.37 だったので順方向の方が良い → 除外
TARGET_PAIRS = ["GBP_JPY", "USD_JPY"]
# (GBP_USD は 3件サンプルで信頼性不足、除外)
```

## 6. 自己改善メカニズム [G0-B] — 必須

**ドリフト検出 (毎週)**:
- 直近30日の反転戦略 PF を集計
- 反転戦略の PF が <1.0 になったら即アラート (反転仮説の崩壊)
- 同期間の亡き者シグナル (順方向) の PnL もシャドウ計算し、両方とも負けていれば「シグナル自体が市場ノイズ」と判定

**パラメータ自動再最適化 (月次)**:
- 反転対象ペアの動的選定: 過去90日の順方向 BT PF<0.7 のペアだけを反転対象に
- もし全ペアの順方向 BT PF が 0.8 以上に回復したら、反転戦略は全停止

**最重要: 反転仮説そのものの自己検証**:
- 毎月、亡き者シグナル順方向 BT と反転 BT の両方を並行 BT
- 反転が常に順方向より優れているか?
- もし反転 PF<順方向 PF が2回連続発生 → 反転仮説崩壊 → **戦略全廃案**

**フォールバック**:
- 新パラメータで 30日 PF<0.9 → 旧パラメータリバート
- 反転仮説崩壊 → 全廃案 + Slack 通知 + 別戦略への切替推奨

**トリガー条件**:
- 毎週月曜 06:00 JST ドリフトチェック
- 月初1日 仮説検証 + 再最適化
- 累計 -3,000 JPY / 反転仮説崩壊 / 90日 trades<10 → 撤退

## 7. 過去 BT 結果 [G0-A] — 必須

**現時点で示せる根拠**:
- 亡き者の順方向: GBP_JPY PF 0.87 (実取引、37件)、USD_JPY PF 0.48 (実取引、7件)
- 単純反転すれば**理論上 PF が逆数になる**: GBP_JPY 1.15、USD_JPY 2.08
- **ただし**: スプレッド/スリッページは反転しても同じく差し引かれる → 実効 PF はもう少し低い
- 計算: スプレッド 3 pips、平均 trade size 18 pips の場合、コスト 3/18 = 17%
  - GBP_JPY 反転 実効 PF ≈ 1.15 × (1-0.17) / (1+0.17) ≈ 0.82 — **負ける**
  - USD_JPY 反転 実効 PF ≈ 2.08 × (1-0.17) / (1+0.17) ≈ 1.48 — 勝てる

**Phase 1 で行うべき追加 BT (宿題)**:
- GBP_JPY/USD_JPY M15 5年で亡き者シグナル順方向 と 反転 を BT
- スプレッド込みの実効 PF を確認
- USD_JPY は7件サンプルしかないので、**5年BTで100件+のサンプルで真の反転効果を測る**ことが必須

**PF > 0.95 達成根拠**:
- USD_JPY 反転は実取引データから PF 1.48 想定 (スプレッド込)
- ただし**7件→100件で PF が 1.0 付近に回帰する可能性大**
- GBP_JPY 反転は実効 PF 0.82 で**達成しない見込み**
- 採用するならば USD_JPY 限定

## 11. リスク・既知の弱点

1. **仮説の脆さ**: 亡き者の負けが「シグナル誤り」とは限らない。スプレッド/スリッページ/タイミング遅延が真因なら反転しても負ける
2. **サンプル過少**: USD_JPY 実取引 7件は仮説検証に不十分
3. **「反転」の本質的問題**: 過去の勝てない人の真逆をやることは「過去の損切りライン直前で買う」リスク — 大きく逆行する可能性
4. **EUR_USD で順方向が PF 2.37 だった事実**: 同じシグナルロジックが**ペアによって順 / 逆 が分かれる**= 単一の反転設計では不十分
5. **過剰最適化リスク**: ペア選定+反転の組合せが「亡き者データへのフィット」になる可能性
6. **倫理的問題**: 「他人が損切るタイミングで買う」設計は流動性供給側として正常だが、心理的に難しい
7. **karen 批判耐性**: 「亡き者で負けた仕様を逆にして勝つ」というのは [[feedback-deceased-world-data-inheritance]] の悪用と取られる懸念
