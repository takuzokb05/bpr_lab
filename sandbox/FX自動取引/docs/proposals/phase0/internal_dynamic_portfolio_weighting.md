# プロポーザル: 複数戦略の動的重み付けポートフォリオ (Online Adaptive Allocator)

## 1. 戦略仮説 (1段落)

亡き者は単一戦略 (RSI MA Crossover) を全ペア・全レジームに適用し、結果として **全 trade の PF 0.81 / 累計 -1,180 JPY** という単一失敗モードに陥った。本提案は**複数の独立な戦略を並列実行し、直近パフォーマンスに応じて自動的に資金を再配分する**メタ戦略。各戦略は**個別に PF>1.0 を確認済みのもの**のみ採用 (placeholder禁止)。online learning の枠組み (Cover 1991, "Universal Portfolios"; Helmbold et al. 1998) で、勝ち戦略に資金が集まり、負け戦略から自動的に撤退する。**戦略単独で失敗するリスクを構造的に分散**し、**自己改善が本質的に組み込まれる** (重みの再分配自体が継続学習)。

## 2. 想定エッジ源 [G1-1]

**構造的優位**:
- ポートフォリオ分散効果: 相関の低い複数戦略を組み合わせるとシャープレシオは個別戦略の最大値を超える (Markowitz 1952, Modern Portfolio Theory)
- Universal Portfolio (Cover 1991): 事後最適の単一戦略パフォーマンスに対しlog(N)/T で漸近的に追いつく証明あり
- Online learning (Multiplicative Weights, EXP3): regret bound が証明されており、worst-case でも単一最良戦略と大差ない収束保証
- 各戦略が独立な仮説で動くので、市場レジーム変化に**自然な robustness**

**実取引データの裏付け** ([[feedback-deceased-world-data-inheritance]] 遵守):
- 亡き者の単一戦略全ペア適用が PF 0.81 で失敗 → 「単一戦略 single-point-of-failure」が負け原因の構造
- 一方で EUR_USD のみ PF 2.37 と部分的勝利あり → **戦略×ペアの組合せに heterogeneity がある** = 動的重み付けが意味を持つ前提
- 「亡き者は重み付けを持たなかった」が負け原因のメタ要因 → 本提案で直接修正

## 3. シグナル定義 (擬似コード)

```python
# 複数戦略を並列保持
strategies = [
    ("BR_EUR_USD", BollingerReversal, "EUR_USD"),
    ("MTF_EUR_USD", MTFPullback, "EUR_USD"),
    ("BR_USD_JPY", BollingerReversal, "USD_JPY"),
    ("ATR_Breakout_GBP_JPY", AdaptiveATRBreakout, "GBP_JPY"),
    ("Confluence_EUR_USD", ConfluenceGateStrategy, "EUR_USD"),
]
# 各戦略は事前 BT で PF>1.0 を確認済みのみ採用

# 重み: 初期均等、累積パフォーマンスで更新
weights = {name: 1.0/len(strategies) for name, _, _ in strategies}

def update_weights(realized_pnl_by_strategy, eta=0.1):
    """Multiplicative Weights Update Method (MWUM)
    各戦略の直近 PnL に応じて重みを更新。"""
    for name in weights:
        recent_pnl = sum(realized_pnl_by_strategy.get(name, []))  # 直近30日
        weights[name] *= exp(eta * recent_pnl / normalizer)
    # 正規化
    total = sum(weights.values())
    weights = {k: v/total for k, v in weights.items()}

def allocate_position_size(strategy_name, base_size):
    """重みに応じてポジションサイズを調整"""
    return base_size * weights[strategy_name] * len(strategies)

def on_signal(strategy_name, raw_signal):
    if weights[strategy_name] < 0.05:
        return HOLD  # 重みが下限を下回った戦略は停止
    size = allocate_position_size(strategy_name, BASE_RISK_PCT)
    return execute(raw_signal, size)
```

## 5. リスクモデル [G1-5]

- SL/TP: 各戦略のデフォルト
- ポジションサイジング: 口座 0.6% リスク × 重み × 戦略数 (例: 5戦略均等で全体口座 3% リスク、最大単一戦略 0.6%)
- 想定 MaxDD: -8% (分散効果 + 重み低下による自動撤退)
- テールリスク: 5戦略同時崩壊 (2020年3月コロナ級ショック) では分散効果消失
- ハード上限: 全戦略合算で口座リスク 3% を超えない

## 6. 自己改善メカニズム [G0-B] — 必須

**ドリフト検出 (毎日)**:
- 各戦略の直近30日 PF を毎日計算
- 過去365日対比で PF が ±0.5 変動した戦略を個別アラート
- 重み分布の Gini 係数を監視 (1戦略集中 = 過剰収束のサイン)

**重み自動更新 (毎日 = continuous self-improvement)**:
- Multiplicative Weights Update で毎日重みを更新
- 学習率 eta は事前 BT で最適化 (典型値 0.05-0.1)
- 重み下限 0.05、上限 0.5 (1戦略集中の防止)

**戦略入れ替え (月次)**:
- 直近 90日 PF<0.9 の戦略は**候補プールに降格**、新戦略候補を試験投入
- 新戦略は最初 0.05 重みでテスト、3ヶ月 PF>1.2 で本格採用
- ヘッドカウント維持: 常に 3-7戦略を保持

**フォールバック**:
- 全戦略の重み合計が 1.0 から ±0.1 ずれたら正規化エラー → ログ + 均等再配分
- 連続2ヶ月で戦略入れ替えが2件以上 → ポートフォリオ不安定アラート + 手動レビュー

**トリガー条件**:
- 毎日 23:00 JST 重み更新
- 毎週月曜 ドリフトチェック
- 月初1日 戦略入れ替え判定
- 累計 -5,000 JPY / 戦略数<3 / 不安定アラート → 全停止

## 11. リスク・既知の弱点

1. **個別戦略への依存**: 採用候補戦略が事前 BT で PF>1.0 でないとそもそも始まらない → **他プロポーザル戦略の Phase 2 通過が前提条件**
2. **複雑度高**: 実装重量級、メンテナンスコスト大
3. **重み収束失敗**: eta が高すぎると振動、低すぎると追従しない
4. **戦略間相関の評価不足**: 「独立な戦略5つ」と称してても実質高相関だと分散効果なし
5. **online learning の遅延**: 重み更新は事後追従、リアルタイムでレジーム変化に追いつかない可能性
6. **過剰最適化リスク**: 戦略選定 + 重み更新パラメータ + 戦略個別パラメータの三重最適化
7. **karen 批判**: 「複雑なポートフォリオが単一戦略より勝つ証明はない」批判
8. **Goodhart の法則**: 「PF を最大化する重み更新」が「PF の指標自体を歪める」可能性
