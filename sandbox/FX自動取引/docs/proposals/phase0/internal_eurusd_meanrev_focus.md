# プロポーザル: EUR_USD 平均回帰フォーカス戦略 (Bollinger Reversal v2 + 自己改善)

## 1. 戦略仮説 (1段落)

亡き者の実取引データ (`fx_trading_prod_snapshot.db`) では、**EUR_USD だけが PF 2.37 / +3,315 JPY (13件、勝率 38.5%)** で唯一の勝ちペアだった。他全ペア (GBP_JPY/USD_JPY/GBP_USD/AUD_USD/EUR_JPY) が PF<1 で計-4,495 JPY と負けた中で、EUR_USD のみ平均勝ち+1,147 JPY / 平均負け-303 JPY の極端な非対称収益を示した。これは「EUR_USD は欧州時間中心に平均回帰しやすい構造」と仮説立てし、BollingerReversal (PF 1.24 EUR/USD BT 実績あり) を**唯一の本命ペアに絞り、他ペアを完全に捨てる**ことで、亡き者で実証された唯一の勝ちパターンに資源を集中する。

## 2. 想定エッジ源 [G1-1]

**構造的優位**:
- ECB / Fed のスケジュール化された金融政策アナウンスでオーバーシュート → 平均回帰が他ペアより素直
- 欧州・米国の双方向流動性 (世界最大の出来高ペア) で極端な値動きが statistical mean-reversion を呼ぶ
- 商品通貨や JPY クロスのような片側マクロ要因 (キャリートレード、原油等) が少ない、ピュアな相対金利ペア

**実取引データの裏付け** ([[feedback-deceased-world-data-inheritance]] 遵守):
- 亡き者で EUR_USD 13件 / 勝率 38.5% / **平均勝ち +1,147 / 平均負け -303** → 1勝で4負分を取り戻す典型的な平均回帰パターン
- 同期間 GBP_JPY は平均勝ち+627 / 平均負け-391 = 1.6 RR で対称的 (= JPYクロスのモメンタム性)
- 「亡き者で何が勝ち、何が負けたか」のデータが**唯一明示的に「これは続けろ」と言っている数少ない事実**

## 3. シグナル定義 (擬似コード)

```python
def generate_signal(data_m15, data_h1):
    # ベース: BollingerReversal (2σ タッチ + RSI 過熱)
    bb_upper, bb_lower = bollinger(close, length=20, std=2.0)
    rsi = RSI(close, length=14)

    # 改善1: H1 トレンドフィルタ (亡き者の負け原因対策)
    # 亡き者 EUR_USD の 8件負けの多くは強トレンド中の逆張りだった可能性
    h1_ema200 = EMA(data_h1.close, 200)
    h1_trend_strong = ADX(data_h1) > 30
    if h1_trend_strong and (close > h1_ema200 * 1.005 or close < h1_ema200 * 0.995):
        return HOLD  # 強トレンドの伸びきり初動では逆張りしない

    # 改善2: 欧州/NYセッション限定 (Tokyo 時間の薄商い回避)
    if not (7 <= utc_hour <= 19):
        return HOLD

    # シグナル本体
    if close >= bb_upper and rsi >= 70:
        return SELL
    if close <= bb_lower and rsi <= 30:
        return BUY
    return HOLD

# SL/TP: ATR × 1.2 (短め)、TP = SL距離 × 1.5
# 改善3: タイムストップ 4時間で強制クローズ (平均回帰は早く効く前提)
```

## 6. 自己改善メカニズム [G0-B] — 必須

**ドリフト検出 (毎日 06:00 自動実行)**:
- 過去30日の win rate / PF / 平均勝ち pips / 平均負け pips を計算
- **KL ダイバージェンス**: 直近30日と過去365日の trade outcome 分布を比較。KL > 0.5 でアラート
- 直近30日の PF が <1.0 / >1.0 でフラグ

**パラメータ自動再最適化 (月次、または PF<1.0 検出時)**:
- グリッド: BB std ∈ {1.8, 2.0, 2.2}、RSI threshold ∈ {65/35, 70/30, 75/25}、ATR mult ∈ {1.0, 1.2, 1.5}
- Walk-forward 直近 12ヶ月 IS / 直近 3ヶ月 OOS で最良パラメータ採択
- OOS PF ≥ 1.3 のパラメータがなければ**手動レビューフラグ** (自動切替しない)

**フォールバック**:
- 新パラメータ採用後 30日 PF<0.9 → 直前パラメータへ自動リバート
- 連続2回再最適化失敗 → 戦略停止 + Slack通知

**トリガー条件・頻度**:
- ドリフト検出: 毎日 06:00 JST cron
- パラメータ再最適化: 月初1日 / 直近30トレードで PF<1.0 検出時 (どちらか早い方)
- 撤退 hard stop: 累計 -5,000 JPY / 90日 trades<10 / 連続2回再最適化失敗

## 11. リスク・既知の弱点

1. **サンプルサイズ**: 亡き者の EUR_USD 13件はサンプル過少。13件 PF 2.37 は偶然の可能性が **棄却できない** (二項検定 p=0.3 程度)
2. **生存者バイアス**: 亡き者の損失全体を見ずに「勝ったペア」だけ抽出する形は cherry-pick
3. **平均回帰の死**: トランプ関税 / 円急騰 / ECB 政策変更等で EUR_USD がモメンタム支配下に入ると BollingerReversal は連敗する (旧 GBP_JPY と同じ轍)
4. **タイムストップの罠**: 4時間で強制クローズしたら勝てたはずのトレードを切る可能性。BT で要検証
5. **セッション制限の代償**: Tokyo 時間 (薄商い)を切ると BB ブレイクの取りこぼし発生
