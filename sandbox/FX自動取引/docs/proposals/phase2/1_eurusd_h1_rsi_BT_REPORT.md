## 1. 戦略概要 (Phase 0 確定仕様、改変なし)

### シグナル定義 (擬似コードそのまま実装)
```python
rsi_14 = RSI(close, 14)
ema50, ema200 = EMA(close, 50), EMA(close, 200)
atr_14 = ATR(high, low, close, 14)
trend_up   = ema50 > ema200
trend_down = ema50 < ema200

if rsi_14 < 30 and trend_up:
    long: sl = entry - atr_14 * 1.5, tp = entry + atr_14 * 2.5
if rsi_14 > 70 and trend_down:
    short: sl = entry + atr_14 * 1.5, tp = entry - atr_14 * 2.5
```
- 時間損切り: 24時間
- ペア / TF: EUR_USD / H1 (固定)
- リスク/取引: 口座 1% (Kelly 1/4)
- 同時保有: 1ポジション

### コスト想定 (PHASE_2_PLAN.md §C 準拠)
- スプレッド: 1.0 pip (EUR_USD)
- スリッページ: エントリー +1 pip、決済 +1 pip
- 合計 round-trip コスト: **3.0 pips**

## 7. 自己改善メカニズム動作

| 機構 | 5年間 作動回数 | 評価 |
|---|---:|---|
| ドリフト検出 (high_vol regime 突入) | 427 回 | 過敏 |
| ドリフト復帰 (p70 復帰) | 427 回 | バランス取れている |
| ローリング20PFモニタ 半量化トリガー | 1 回 | 機能 |
| 自動停止トリガー (PF<0.7×30連続) | 882 回 | **異常多発** |
| 撤退条件シミュレーション | **発動** (直近90日 trades 4 < 5) | **発動** |

### 重大警告
- `high_vol regime` が **5年間で 427 回出入り** = ほぼ常時抑制状態。シグナル本体 (long 428 + short 494 = 922) のうち、実トレードに繋がったのは **33件 (3.6%)**。
- 自動停止トリガが **882 回** = 連続条件のカウントが過敏に動作 (連続 30 件 PF<0.7 が累積で何度も再カウントされる実装上の挙動)。これは「self-improvement が機能して停止した」のではなく「実装が過敏」というシグナル。
- **撤退条件発動**: 過去5年中で「直近90日 trades<5」が真になっている = self-improvement の抑制効果でトレードが過疎化していた証左。

### 結論
プロポーザル §6 「自己改善メカニズム」の擬似コードは BT 上でも作動を確認したが、

1. high_vol regime の閾値設計 (ATR p90/p70) は **強すぎ** て戦略が機能しない
2. それでも漏れて取引した小数 (33件) が見かけ上 PF 1.09 を出したが、**素の戦略は明確に負け** (PF 0.79、-481k JPY)
3. **「自己改善が正しく機能した結果としてのプラス」ではなく「シグナルが大半抑制されたから損失が出なかった」だけ** = エッジを実証していない

## 10. 撤退理由 (重要な事実関係)

### プロポーザル記載との乖離

プロポーザル `memory_eur_usd_h1_rsi_pullback.md` での主張:
- 「BTグリッドで OOS PF>1.0 が 13/20、平均 OOS PF=1.06、最頻パラメータで PF 1.34-1.49」
- 「5年累計 OOS リターン (RSI 30/70, ATR 2.5、lot 1.0、100万円口座): 想定 +6-12%/年」

今回の精査 BT 実測:
- 同パラメータ (RSI 30/70 + ATR_SL 1.5 / ATR_TP 2.5) で **OOS PF 0.57、5年累計 +0.73%/年** (self-improvement あり) / **-9.6%/年** (self-improvement なし)
- 既存 `data/backtest_grid_h1_EUR_USD.csv` 結果との **乖離が極めて大きい**

### 乖離の可能性のある原因 (要検証、本BTでは未確定)

1. **既存BTグリッドのコスト想定が甘い**: プロポーザルでは 1.5pip スプレッド + 0.3pip スリッページ = 1.8pip だったが、本BTは保守の 3.0pip。0.5pip 差で PF が大きくぶれる可能性
2. **時間損切り未実装**: 既存BTには 24時間損切りが入っていなかった可能性。本BTでは 33件中 7件 (21%) が TIME 決済 → これが負け側に偏ると PF が下がる
3. **2024-2026 期間の構造変化**: ECB金利体制の変化でレンジ性が崩れた (プロポーザル §11 で起草者自身が想定リスクとして明記済み) → 実際にこれが起きている
4. **パラメータの過剰適合**: BTグリッド 20点中 13点で PF>1.0 だが、これは plateau ではなく **noise** だった可能性 (今回の感度 ±20% で全FAIL から)

### プロポーザル §11 起草者自身の警告が現実化
> 「2024-2026 期間限定の有効性: BTグリッドは5年だが、ECB金利体制の変化 (例 2027年以降の利上げサイクル終了) でレンジ性が崩れる可能性 → 自己改善メカニズムで月次再評価が必須」

→ 2025年以降のレジームで起こったのは「予期した崩壊」そのもの。月次再評価では救えなかった (WFA W5/W6/W7 で連続劣勢)。

## 11. 結論

### Phase 3 進出: 不可

理由:
- 必須 5 ゲートで FAIL
- OOS で完全に崩壊 (PF 0.57、Sharpe -2.47)
- WFA 7 windows のうち PF≥1.3 は 1 つだけ (W2 のみ、2022年末ECB局面)
- パラメータ感度ゼロ (近傍 6/6 で FAIL)
- 自己改善メカニズムは作動するが、抑制によってトレードを殺すことで「見かけ上の生存」を作っているだけで、エッジ実証になっていない

### 棚上げ判断
**廃案**。Phase 0 評価時の「6評価委員全員 PASS」「総合 87/100」は、**実測 BT を経ずに既存BTグリッド資料を信用したことによるエラー**であり、`docs/RETREAT_2026-05-26.md` で警告された「effort heuristic の罠」「自分が書いた memory を自分で破った」と同型の失敗。

### 救済の可能性 (低い)
仮に救うとすれば:
1. パラメータを W2 期間 (2022-11〜2023-05) で再フィットすれば PF 2.25 が出る = **過剰適合の典型**
2. 別ペア (EUR_CHF、EUR_GBP 等) への横展開 = プロポーザル §11 で記載済の Phase 3 ロードマップ
3. M15 や M5 と合成 = プロポーザル §11 で「Phase 2 で検討」と記載

ただし、**素の戦略が PF 0.79 / 年率 -9.6% である事実**は救済設計の前に直視すべき。EUR_USD H1 RSI Pullback というエッジ仮説自体を再検討する必要がある。
